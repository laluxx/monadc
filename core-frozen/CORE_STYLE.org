* Monad Core Style Specification

This file records the source style expected across the core and prelude.

** 1. Design-forward source

Core modules should be written in idiomatic Monad/Wisp, not lowered into old
compiler workarounds. If a feature is part of the intended language, prefer the
intended source form and let the compiler grow toward it.

Use top-level Wisp syntax:

#+begin_src monad
define square :: Int -> Int
  :doc "Return X squared."
  x -> x * x
#+end_src

Use small inner Lisp forms only when they are the clearest local expression,
such as a short lambda in a test or a small higher-order helper.

** 2. Type notation

Monad type notation is not Haskell notation:

#+begin_src monad
[a]     ; any sequence / collection of a
(a)     ; a concrete list of a
((Int)) ; a concrete list of concrete lists of Int
[Int]   ; an array of Int
#+end_src

Public collection APIs should use =[a]= when they work over the sequence
abstraction. Use =(a)= only when the representation must be a concrete list.

** 3. Algebraic data types over encodings

Do not encode algebraic structure with lists or sentinels.

Correct:

#+begin_src monad
data Maybe a = Nothing | Just a
#+end_src

Incorrect:

#+begin_src monad
Nothing == []
Just x  == [x]
#+end_src

Once an ADT exists, every public API must use it. Optional results are
=Maybe a=; errors with detail are =Either e a=.

** 4. Typeclasses and instances

Write typeclasses as source-level language constructs:

#+begin_src monad
class Monad m where
  (>>=)  :: m a -> (a -> m b) -> m b
  return :: a -> m a
#+end_src

Instances use equation defaults or pattern matching:

#+begin_src monad
instance Monad Maybe where
  (ma >>= f) =>
    match ma with
      Nothing -> Nothing
      Just x  -> f x

  (return x) => Just x
#+end_src

Numeric classes belong in =prelude/Numeric.mon=; structural classes belong in
=prelude/Class.mon=; concrete algorithms belong in =Math.mon=.

** 5. Associated types

Associated result types are written inside the class and fixed in instances:

#+begin_src monad
class Pow a where
  type Result a
  (^) :: a -> Int -> Result a

instance Pow Float where
  type Result Float = Float
#+end_src

** 6. Aliases

A real alias is metadata on the canonical definition. Do not duplicate alias
function bodies.

Correct:

#+begin_src monad
define fanout :: (a -> b) -> (a -> b) -> a -> [b]
  :alias &&
  :doc "Apply F and G to X and return both results."
  f g x -> [(f x) (g x)]
#+end_src

Incorrect:

#+begin_src monad
define && :: ...
  f g x -> fanout f g x
#+end_src

** 7. Helper functions

Helper functions belong in `where` clauses attached to the definition that uses
them. Avoid top-level helper leakage.

#+begin_src monad
define sqrt :: Float -> Float
  x | <= 0.0    -> 0.0
    | otherwise -> until goodEnough? improve (x / 2.0)
    where
      improve    g -> (g + x / g) / 2.0
      goodEnough g -> (< (abs-float ((improve g) - g)) (epsilon * (maxFloat 1.0 g)))
  :doc "Approximate the square root of X."
#+end_src

** 8. Doc placement

If a definition uses only plain =->= clauses, put =:doc= before the clauses:

#+begin_src monad
define square :: Int -> Int
  :doc "Return X squared."
  x -> x * x
#+end_src

If a definition uses guarded | clauses, put =:doc= after the guarded clauses:

#+begin_src monad
define abs-int :: Int -> Int
  x | < 0       -> 0 - x
    | otherwise -> x
  :doc "Return the absolute value of integer X."
#+end_src

** 9. Guard shorthand

When a guarded definition has a single left-hand binder, bare operator/predicate
sections in guards are applied to that binder:

#+begin_src monad
define char-upcase :: Char -> Char
  c | >= 'a' and <= 'z' -> c - case-delta
    | otherwise         -> c
  :doc "Convert C to uppercase."
#+end_src

** 10. Constants

Typed constants use bracketed `define` forms. Symbolic aliases are metadata:

#+begin_src monad
define [pi :: Float] 3.141592653589793
  :alias π
#+end_src

Canonical infinite enumerations use range literals:

#+begin_src monad
define [naturals  :: List] ⌜0..  ⌝
define [positives :: List] ⌜1..  ⌝
define [evens     :: List] ⌜0,2..⌝
define [odds      :: List] ⌜1,3..⌝
#+end_src

** 11. Partiality

Partial functions may exist when they are conventional (=head=, =tail=,
=fromJust=), but each one must say =Partial= in its doc string. Prefer safe
variants returning =Maybe= for public APIs.

** 12. Module layering

The intended dependency direction is:

#+begin_src txt
prelude/Function
prelude/Class
prelude/Numeric
Data.Maybe / Data.Either / Data.Tuple
prelude/Coll
Data.List / Data.Set / Data.Json
Math
System / Term
#+end_src

Avoid importing concrete modules into foundational modules unless the typeclass
itself explicitly depends on the type.
