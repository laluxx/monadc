The =Char= type represents a single Unicode codepoint. At the type level it is a 32-bit codepoint; at the codegen level it is lowered to =i8= for ASCII-compatible values. The runtime tag =RT_CHAR= distinguishes it from other scalar types.

See [[file:strings.org][Strings]] for strings as character sequences and [[file:types.org][Types]] for the ground type hierarchy.

* 1. Character Literals ([[file:../reader.c::1369]])

Characters use single-quote delimiters (=TOK_CHAR=):

#+BEGIN_SRC monad
'a'                ;; Char 'a'
'\n'               ;; Newline
'\x41'             ;; Hex escape — 'A'
'\u03C0'           ;; Unicode — π
'\''               ;; Literal single quote
'\\'               ;; Literal backslash

#+BEGIN_SRC wisp
'a'                ;; Char 'a'
'\n'               ;; Newline
'\x41'             ;; Hex escape — 'A'
'\u03C0'           ;; Unicode — π
'\''               ;; Literal single quote
'\\'               ;; Literal backslash
#+END_SRC

Supported escape sequences:

| Sequence | Meaning           |
|----------+-------------------|
| =\n=       | Newline           |
| =\t=       | Tab               |
| =\r=       | Carriage return   |
| =\\=       | Backslash         |
| =\'=       | Single quote      |
| =\0=       | Null              |
| =\xNN=     | Hex codepoint     |
| =\uNNNN=   | Unicode codepoint |

Note: In [[file:wisp.org][Wisp]] syntax, ='\n'= may fail because the Wisp expander expands =\\n= to a real newline. Use S-expression syntax for escape sequences involving backslash (see [[file:strings.org][Wisp char escape limitation]]).

Key Points:
- Character literals use single-quote delimiters: ='a'=, ='\n'=, ='\x41'=, ='\u03C0'=
- Supported escapes: =\n=, =\t=, =\r=, =\\=, =\'=, =\0=, =\xNN=, =\uNNNN=
- In Wisp mode, backslash escapes may break; use s-expression syntax for literals with escapes

See also: [[file:wisp.org][Wisp]]

* 2. Char Type

The =Char= type is a primitive ground type ([[file:types.org][Types]]) defined as =TYPE_CHAR= in the type system ([[file:../types.h::8]]). Its kind is =TYPE_KIND_GROUND=.

- **Runtime representation:** 32-bit integer tagged as =RT_CHAR=
- **Codegen lowering:** =i8= for ASCII-compatible codepoints
- **LLVM global format:** stored as =i8= in string globals

This page covers the =Char= type: character literals, type
conversions, type predicates, and string iteration.

See [[file:../context/types.org][context/types.org]] for =TYPE_CHAR= and [[file:../context/language.org::runtime-value-system][context/language.org — Runtime Value System]].

* 3. Type Conversions

Character conversion operations dispatch through strcmp handlers ([[file:../codegen.c::13144]], [[file:../context/codegen-ops.org][context/codegen-ops.org]]):

#+BEGIN_SRC monad
(Char 65)           ;; 'A' — integer codepoint to Char
(Int 'A')           ;; 65  — Char to integer codepoint
(Hex 255)           ;; 0xFF — integer to Hex
(Bin 42)            ;; 0b101010 — integer to Binary
(Oct 63)            ;; 0o77 — integer to Octal
#+END_SRC

#+BEGIN_SRC wisp
Char 65           ;; 'A' — integer codepoint to Char
Int 'A'           ;; 65  — Char to integer codepoint
Hex 255           ;; 0xFF — integer to Hex
Bin 42            ;; 0b101010 — integer to Binary
Oct 63            ;; 0o77 — integer to Octal
#+END_SRC

All conversion paths are verified by test fixtures. See [[file:casting.org][Casting]] for the full conversion reference.

See also: [[file:casting.org][Casting]]

* 4. Type Predicate

#+BEGIN_SRC monad
(Char? 'a')         ;; True
(Char? 65)          ;; False
(Char? "hello")     ;; False
#+END_SRC

#+BEGIN_SRC wisp
Char? 'a'         ;; True
Char? 65          ;; False
Char? "hello"     ;; False
#+END_SRC

The =Char?= predicate checks =TYPE_CHAR= kind at the type level.

* 5. Character Operations

** 5.1. String Indexing ([[file:../codegen.c::9732]])

String indexing returns a =Char=:

#+BEGIN_SRC monad
("hello" 0)        ;; 'h'
("hello" 1)        ;; 'e'
#+END_SRC

#+BEGIN_SRC wisp
("hello" 0)        ;; 'h'
("hello" 1)        ;; 'e'
#+END_SRC

Postfix bracket syntax also works: =xs[0]= ([[file:../reader.c::6621]]). Both compile to a character load with bounds checking. See [[file:strings.org][String Indexing]].

** 5.2. Iteration

Strings iterate character-by-character via =for=:

#+BEGIN_SRC monad
for c -> "hello"
  (show c)
;; prints:
;; 'h'
;; 'e'
;; 'l'
;; 'l'
;; 'o'
#+END_SRC

#+BEGIN_SRC wisp
for c -> "hello"
  show c
;; prints:
;; 'h'
;; 'e'
;; 'l'
;; 'l'
;; 'o'
#+END_SRC

Or backwards:

#+BEGIN_SRC monad
for c <- "hello"
  (show c)
;; prints:
;; 'o'
;; 'l'
;; 'l'
;; 'e'
;; 'h'
#+END_SRC

#+BEGIN_SRC wisp
for c <- "hello"
  show c
;; prints:
;; 'o'
;; 'l'
;; 'l'
;; 'e'
;; 'h'
#+END_SRC

See [[file:iteration.org][Iteration]] for the full =for= loop reference.

See also: [[file:iteration.org][Iteration]]
