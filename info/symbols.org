Monad has two symbol-like types: =Symbol= (interned identifiers) and =Keyword= (colon-prefixed self-evaluating constants). Both are primitive ground types at the type level.

* Symbols (=TYPE_SYMBOL=)

Symbols are interned names used for identifiers, variable references, and quoted data:

#+BEGIN_SRC monad
'x                ;; symbol x (quoted)
'my-function      ;; symbol my-function
'π                ;; unicode symbol
#+END_SRC

#+BEGIN_SRC wisp
'x                ;; symbol x (quoted)
'my-function      ;; symbol my-function
'π                ;; unicode symbol
#+END_SRC

Symbols are interned: two symbols with the same name share the same runtime object. This enables efficient equality comparison via pointer identity.

** Symbol Intern Table ([[file:../env.c]])

The compiler maintains a symbol intern table for fast lookup:

| Aspect   | Detail            |
|----------+-------------------|
| Storage  | Hash map in =Env=   |
| Key      | String name       |
| Value    | =Symbol= pointer    |
| Lifetime | Compiler lifetime |

** Runtime Representation

| Tag  | =RT_SYMBOL=                    |
|------+------------------------------|
| Data | =char*= (interned name string) |

* Keywords (=TYPE_KEYWORD=)

Keywords are colon-prefixed self-evaluating constants used for named parameters, map keys, and metadata:

#+BEGIN_SRC monad
:key              ;; keyword :key
:alias            ;; keyword :alias
:x-y              ;; keyword :x-y
:monadc           ;; keyword :monadc
#+END_SRC

#+BEGIN_SRC wisp
:key              ;; keyword :key
:alias            ;; keyword :alias
:x-y              ;; keyword :x-y
:monadc           ;; keyword :monadc
#+END_SRC

Keywords evaluate to themselves — they do not need quoting:

#+BEGIN_SRC monad
:hello            ;; => :hello
(= :key :key)     ;; Tru)
#+END_SRC

#+BEGIN_SRC wisp
:hello            ;; => :hello
= :key :key     ;; Tru)
#+END_SRC

** Runtime Representation

| Tag  | =RT_KEYWORD=                          |
|------+-------------------------------------|
| Data | =char*= (keyword name, without colon) |

** Keyword Usage Patterns

Keywords are commonly used for:

- Map keys: =#{"key1" val1 :key2 val2}=
- Named arguments (planned): =(fn x :verbose True)=
- Metadata annotations: =:doc "description"=
- Enum-like constants: =:red :green :blue=

* Quoting

Symbols require quoting to prevent evaluation:

#+BEGIN_SRC monad
(x                 ;; evaluates to the value bound to x)
'x                ;; the symbol x itself
:key              ;; evaluates to itself (no quote needed)
#+END_SRC

#+BEGIN_SRC wisp
x                 ;; evaluates to the value bound to x
'x                ;; the symbol x itself
:key              ;; evaluates to itself (no quote needed)
#+END_SRC

See [[file:collections.org][Collections]] for quoted list syntax.
