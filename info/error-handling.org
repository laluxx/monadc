* Error (type)

=(data Error String)= — simple error type carrying a message.

* error (builtin)

=(error "message")= — halts the program at runtime. Use for
unrecoverable errors.

* ?? (null-coalescing)

=(?? expr default)= returns =expr= if not =nil=, =default= otherwise.

#+BEGIN_SRC monad
(?? (lookup key dict) "default")
(?? (safe-div a b) 0)
#+END_SRC

Implementation: evaluates =lhs=, checks nil via =rt_value_is_nil=,
branches to =rhs= or returns =lhs= unwrapped. Scalar types (Int, Float,
Bool, Char) are never nil and skip the check entirely.

* Pattern: Option type with match

#+BEGIN_SRC monad
(define (safe-div a b -> (Option Int))
  (if (= b 0) None
    (Some (/ a b))))
#+END_SRC

** Debug vs Release

- **Debug (fat pointer):** raw pointer + static string table baked into
  the binary. =nil= carries a source-location tag, enabling trace
  output without heap allocation.
- **Release (thin pointer):** =??= is a single null check; the string
  table is stripped entirely. No trace, no overhead.

* Parser Errors

The parser uses =Error:= prefix for user-facing messages and
=[internal]= for internal compiler errors. See [[file:errors.org][Errors]].
