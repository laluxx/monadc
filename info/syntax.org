Monad supports two surface syntaxes — S-expressions and Wisp — that produce identical ASTs and mix freely in the same file. This page covers atoms, lists, literals, comments, identifiers, operators, and all special forms.

* Syntax Overview

* 1. S-Expressions

S-expressions are Lisp-style parenthesized prefix notation. Every form
is either an /atom/ (a single value) or a /list/ (a sequence of atoms or
sublists enclosed in parentheses). The reader
([[file:../context/reader.org][context/reader.org]]) tokenizes and parses S-expr
source directly into the AST.

** 1.1 Atoms

Atoms are the atomic building blocks of the language — indivisible values
that cannot be broken into smaller expressions:

| Atom       | Example              | Description                      | Page                      |
|------------+----------------------+----------------------------------+---------------------------|
| Integer    | =42=, =-7=, =1_000=        | 63-bit signed                   | [[file:numeric.org]]          |
| Float      | =3.14=, =1e10=, =-0.5=     | 64-bit IEEE 754                 | [[file:numeric.org]]          |
| Hex        | =0xFF=                 | Hexadecimal                     | [[file:numeric.org]]          |
| Binary     | =0b1010=               | Binary                          | [[file:numeric.org]]          |
| Octal      | =0o777=                | Octal                           | [[file:numeric.org]]          |
| Ratio      | =1/2=, =3/4=             | Exact rational                  | [[file:rationals.org]]      |
| Percentage | =50%=                  | =50%= desugars to =1/2=         | [[file:rationals.org]]      |
| String     | ="hello\nworld"=       | C-escaped string                | [[file:strings.org]]        |
| Char       | ='a'=, ='\n'=            | Single character                | [[file:characters.org]]     |
| Symbol     | =x=, =my-fn?=, =π=         | Identifier                      | [[file:symbols.org]]         |
| Keyword    | =:key=, =:alias=         | Keyword                         | [[file:symbols.org]]         |
| Path       | =/tmp/file=, =~/.config= | File path                       | [[file:paths.org]]           |
| Boolean    | =True=, =False=          | Boolean literals                | [[file:booleans.org]]       |
| Nil        | =nil=                  | Empty list / null               | —                         |

The reader tokenizes atoms in =reader_tokenize= — see
[[file:../context/reader.org][context/reader.org — Tokenizer]] for the full grammar.

** 1.2 Lists

A list is a sequence of atoms or sub-lists enclosed in parentheses.
Lists form the backbone of S-expression syntax — function calls,
special forms, and data structures are all represented as lists:

#+BEGIN_SRC monad
(+ 1 2)                        ;; function call
(define x 42)                  ;; special form
(lambda (x) (+ x 1))           ;; nested lists
'((1 2) (3 4))                 ;; nested quoted list
#+END_SRC

Lists are the fundamental compound structure in S-expr syntax. The reader
produces =AST_LIST= nodes for parenthesized forms
([[file:../context/reader.org][context/reader.org — List Parsing]]).

** 1.3 Strings

Strings use C-style escape sequences:

#+BEGIN_SRC monad
"hello\nworld"                   ;; newline inside string
"tab\there"                      ;; tab character
"quote: \"hello\""               ;; escaped double quote
"\x41"                           ;; ASCII 65 = "A"
"\u03C0"                         ;; Unicode π
#+END_SRC

| Escape | Meaning     | Notes              |
|--------+-------------+--------------------|
| =\n=     | Newline     | Line feed (0x0A)   |
| =\t=     | Tab         | Horizontal tab     |
| =\"=     | Double quote| String delimiter   |
| =\\=     | Backslash   | Literal backslash  |
| =\0=     | Null        | Zero byte          |
| =\xNN=   | Hex byte    | 2-digit hex        |
| =\uNNNN= | Unicode     | 4-digit codepoint  |

Adjacent string literals are NOT concatenated (unlike C).

** 1.4 S-Expression Key Points

- S-expressions are parenthesized prefix notation: =(operator operand ...)=
- Atoms are indivisible values (numbers, strings, symbols, etc.)
- Lists are compound structures enclosing atoms or sublists
- The reader directly parses S-expr into AST nodes

* 2. Wisp Syntax

Wisp is an indentation-aware surface syntax
([[file:../wisp.c][wisp.c]], 4785+ lines) — see the dedicated page at
[[file:wisp.org][Wisp Syntax]] and
[[file:advanced-wisp.org][Advanced Wisp]].

It expands to standard S-expressions in a pre-pass before the reader (see
[[file:../context/wisp.org][context/wisp.org]]). Both produce identical ASTs and
mix freely in the same file.

** 2.1 Side-by-Side Comparison

| S-Expression                          | Wisp                              |
|---------------------------------------+-----------------------------------|
| =(define x 42)=                       | =define x 42=                        |
| =(lambda (x) (+ x 1))=               | =lambda x\n  + x 1=                  |
| =(if (> x 0) "pos" "neg")=           | =if\n  > x 0\n  "pos"\n  "neg"=    |
| =(let [x 1 y 2] (+ x y))=            | =let\n  x 1\n  y 2\n  + x y=        |
| =(data Option a None (Some a))=       | =data Option a\n  None\n  Some a=  |
| =(module Math [pi e])=                | =module Math [pi e]=                |
| =(begin (show "a") (show "b"))=       | =begin\n  show "a"\n  show "b"=    |
| =(match x (Some v) v None -1)=        | =match x\n  Some v -> v\n  None -> -1= |

** 2.2 How It Works

The wisp expander runs as a two-pass pre-pass before the reader
([[file:../context/wisp.org::wisp-architecture][context/wisp.org — Architecture]]):

1. Tokenization: split source into tokens (parens, brackets, symbols, strings,
   numbers, etc.). Balanced brackets are preserved as single tokens.
2. Layout expansion: use indentation and arity tables to insert parentheses
   where needed, converting indentation-driven blocks into S-expression trees.

Pipeline details at [[file:../context/wisp.org::wisp-pipeline][context/wisp.org — Pipeline]].

** 2.3 Wisp Key Points

- Wisp uses indentation instead of parentheses to show structure
- The Wisp expander runs as a pre-pass, converting to S-exprs before the reader
- Both syntaxes produce identical ASTs and mix freely
- The arity table tells the expander how many arguments each form expects

* 4. Comments

Comments are identical in both S-expr and Wisp syntax:

#+BEGIN_SRC monad
;; Line comment

;|
Block comment
spanning multiple
lines
|;
#+END_SRC

#+BEGIN_SRC wisp
;; Line comment

;|
Block comment
spanning multiple
lines
|;
#+END_SRC

| Comment Type  | Syntax    | Behavior                          |
|---------------+-----------+-----------------------------------|
| Line          | =;; ...=  | Runs from =;;= to end of line      |
| Block         | =;| ... |;=| | Delimited, can span multiple lines |

Comments are stripped during tokenization before parsing
([[file:../context/reader.org][context/reader.org — Comment Handling]]).

* 5. Identifiers

Identifiers in Monad follow Lisp conventions with Unicode support.

| Category      | Pattern                       | Examples                   | Page               |
|---------------+-------------------------------+----------------------------+--------------------|
| Symbol        | Alpha, digits, =?=, =!=, =-=      | =x=, =my-fn?=, =π=, =a-b=      | [[file:symbols.org]]  |
| Keyword       | Colon prefix                  | =:key=, =:alias=            | [[file:symbols.org]]  |
| Constructor   | Uppercase start               | =True=, =Some=, =Nil=        | [[file:data.org]]   |
| Type variable | Lowercase or ='= prefix        | =a=, ='a=, =t=               | [[file:types.org]]  |
| Operator      | Punctuation chars             | =+=, =*=, =<=, =>>=            | [[file:numeric.org]]|

The reader identifies symbols via =reader_read_symbol= in
[[file:../context/reader.org][context/reader.org — Symbol Parsing]].

* 6. Operators

Monad allows operator symbols as ordinary function identifiers. Operators are
not a separate syntactic class — they are symbols that happen to consist of
punctuation characters and are used in prefix position.

| Category       | Operators                              | Page                  |
|----------------+----------------------------------------+-----------------------|
| Arithmetic     | =+=, =-=, =*=, =/=, =%=, =^=                | [[file:numeric.org]]     |
| Comparison     | ===, =/=, =<=, =>=, =<, =>                 | [[file:equality.org]] |
| Logical        | =and=, =or=, =not=                         | [[file:logic.org]]    |
| Bitwise        | =&, |=, =^=, =~=, =<<=, =>>=                  | [[file:numeric.org]]     |
| Composition    | =..=, =||>=, =<$>=, =>=>=                    | [[file:functional-programming.org]] |

All operators are prefix (S-expr) or prefix with arity (Wisp):

#+BEGIN_SRC monad
(+ 1 2)                  ;; arithmetic
(= x 42)                 ;; equality
(and p q)                ;; logical
(& flags 0xFF)           ;; bitwise
(|= x f)                  ;; pipeline
#+END_SRC

#+BEGIN_SRC wisp
+ 1 2                     ;; arithmetic
= x 42                    ;; equality
and p q                   ;; logical
& flags 0xFF              ;; bitwise
|= x f                    ;; pipeline
#+END_SRC

** Complete example:

#+BEGIN_SRC monad
;; Save as syntax-demo.mon && monadc syntax-demo.mon && ./syntax-demo
(module Main)

;; All major special forms in one program
(define (greet [name :: String] -> String)
  (++ "Hello, " name "!"))

(define [x :: Int] 42)
(let [y (+ x 1)]
  (show (greet "World"))
  (show "x = " x ", y = " y))

(data Option a None (Some a))
(match (Some 10)
  (Some v) -> (show "Got: " v)
  None     -> (show "Nothing"))

(show "Done")
#+END_SRC

** Common Pitfalls

- **S-expr and Wisp cannot be in the same top-level form**: You cannot
  start a list with `(` in S-expr mode and close it with Wisp indentation.
  Each top-level form is exclusively one syntax.
- **Adjacent string literals are NOT concatenated**: Unlike C, `"hello"
  "world"` is a syntax error, not string concatenation. Use `++`.
- **Feature conditionals are parsed but inert**: `#+linux` / `#+darwin`
  blocks parse correctly but have no codegen effect yet.
- **Block comments cannot nest**: `;| ... |;` does not support nesting.
- **Operator symbols are just identifiers**: `+`, `*` etc. are symbols,
  not a separate operator class. They cannot be used in infix position
  (except through Wisp arity tables).

** 7.8 Special Forms Summary

- All special forms work identically in S-expr and Wisp syntax
- Binding forms (=define=, =lambda=, =let=) create variable scopes
- Conditional forms (=if=, =cond=, =match=) branch on values
- Type forms (=data=, =type=, =class=, =instance=) define types
- Module forms (=module=, =import=, =include=) organize namespaces
- Control forms (=begin=, =while=, =for=, =set!=) manage execution
