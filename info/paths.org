=Path= is a first-class literal type for filesystem paths. Paths are valid Unix path tokens that the reader recognizes as =AST_PATH= nodes ([[file:../reader.c]]).

* Path Literals

#+BEGIN_SRC monad
~/xos/projects/c/monadc/context/     ; absolute with home
/tmp/monadc-fuzz                     ; absolute
./context                            ; relative
../monadc                            ; parent
#+END_SRC

Path literals are valid as values in both S-expression and Wisp syntax:

#+BEGIN_SRC monad
(define context-dir ~/xos/projects/c/monadc/context/)
(show context-dir)                   ; prints directory listing
#+END_SRC

#+BEGIN_SRC wisp
define context-dir ~/xos/projects/c/monadc/context/
show context-dir
#+END_SRC

* Path Type

=Path= is a primitive ground type (=TYPE_PATH=) in the type system ([[file:../types.h::8]]).

| Aspect         | Detail              |
|----------------+---------------------|
| Type kind      | =TYPE_KIND_GROUND=    |
| Type enum      | =TYPE_PATH=           |
| Runtime tag    | =RT_PATH=             |
| Representation | =char*= (path string) |

* Path Operations

Paths support =show= for display and comparison via ==== and =!=:

#+BEGIN_SRC monad
(= ./context ./context)             ; True
(!= ./context /tmp)                 ; True
#+END_SRC

#+BEGIN_SRC wisp
= ./context ./context
!= ./context /tmp
#+END_SRC

* Path vs String

| Aspect         | Path             | String               |
|----------------+------------------+----------------------|
| Type           | =Path= (=TYPE_PATH=) | =String= (=TYPE_STRING=) |
| Literal syntax | =~/path=, =./path=   | ="text"=               |
| Quote needed   | No               | No                   |
| Use case       | Filesystem paths | General text         |

A path literal is not a string — it has its own type and cannot be used where a =String= is expected without explicit conversion.
