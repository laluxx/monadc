=Ratio= is an exact rational number type. It represents a numerator/denominator pair with no floating-point rounding. Unlike =Float= (64-bit IEEE 754), =Ratio= preserves precision through arithmetic operations.

* Ratio Literals ([[file:../reader.c::265]])

#+BEGIN_SRC monad
(1/2           ;; one half)
(3/4           ;; three quarters)
(7/3           ;; seven thirds)
(-5/8          ;; negative ratio)
#+END_SRC

#+BEGIN_SRC wisp
1/2           ;; one half
3/4           ;; three quarters
7/3           ;; seven thirds
-5/8          ;; negative ratio
#+END_SRC

The reader tokenises =1/3= as a single =AST_RATIO= token, not a division expression. The ratio syntax is: =<integer>/<positive-integer>=.

* Percentage Syntax

Percentages desugar to ratios at parse time:

#+BEGIN_SRC monad
(50%           ;; = 1/2)
(25%           ;; = 1/4)
(100%          ;; = 1/1)
(150%          ;; = 3/2)
#+END_SRC

#+BEGIN_SRC wisp
50%           ;; = 1/2
25%           ;; = 1/4
100%          ;; = 1/1
150%          ;; = 3/2
#+END_SRC

The =%= suffix is recognised by the reader and converted to an =AST_RATIO= node with denominator 100, reduced.

* Ratio Type

The =Ratio= type is a primitive ground type ([[file:types.org][Types]]) defined as =TYPE_RATIO= in the type system. Its runtime representation is a pair of 64-bit integers (numerator, denominator) tagged as =RT_RATIO=.

| Aspect         | Detail                               |
|----------------+--------------------------------------|
| Type kind      | =TYPE_KIND_GROUND=                     |
| Type enum      | =TYPE_RATIO=                           |
| Runtime tag    | =RT_RATIO=                             |
| Representation | struct { int64_t num; int64_t den; } |
| Codegen        | LLVM struct ={i64, i64}=               |

* Ratio Arithmetic

Ratios support full arithmetic with automatic reduction:

#+BEGIN_SRC monad
(+ 1/2 1/3)       ;; 5/)
(- 3/4 1/4)       ;; 1/)
(* 2/3 3/4)       ;; 1/)
(/ 1/2 1/4)       ;; 2/)
#+END_SRC

#+BEGIN_SRC wisp
+ 1/2 1/3       ;; 5/)
- 3/4 1/4       ;; 1/)
* 2/3 3/4       ;; 1/)
/ 1/2 1/4       ;; 2/)
#+END_SRC

Mixed-type operations with =Int= and =Float=:

#+BEGIN_SRC monad
(+ 1/2 3)         ;; 7/2  — Int promoted to rati)
(+ 1/2 0.5)       ;; 1.0  — promoted to Floa)
#+END_SRC

#+BEGIN_SRC wisp
+ 1/2 3         ;; 7/2  — Int promoted to rati)
+ 1/2 0.5       ;; 1.0  — promoted to Floa)
#+END_SRC

* Ratio as Function

=Ratio= values can be used as functions:

#+BEGIN_SRC monad
(1/2 100)         ;; 50 — multiplies numerator by arg, divides by denominato)
(1/3 9)           ;;)
#+END_SRC

#+BEGIN_SRC wisp
1/2 100         ;; 50 — multiplies numerator by arg, divides by denominato)
1/3 9           ;;)
#+END_SRC

This works via the codegen dispatch for =AST_RATIO= in function position ([[file:../codegen.c]]). The ratio takes a single =Int= argument and returns the integer result of =(/ (* num arg) den)=.

* Comparison

#+BEGIN_SRC monad
(= 1/2 2/4)       ;; True  — structural equalit)
(< 1/3 1/2)       ;; Tru)
(> 3/4 1/2)       ;; Tru)
#+END_SRC

#+BEGIN_SRC wisp
= 1/2 2/4       ;; True  — structural equalit)
< 1/3 1/2       ;; Tru)
> 3/4 1/2       ;; Tru)
#+END_SRC

Ratios are compared by cross-multiplying to avoid floating point.
