This page covers compiler diagnostics: error format, error layers,
common error patterns, diagnostic fixtures, emit flags for debugging,
and REPL error recovery.

Compiler errors are structured as semantic relations between invalid
source forms and explicit diagnostics. Each error reports the source
location (=file:line:column=) with a descriptive message and a source
pointer showing the offending span.

Error output format (reader.c:246-281):

Test coverage: every stable codegen error path has a focused
expected-failure fixture in =tests/codegen/rt_codegen_err_*.mon=
(43+ fixtures). The diagnostic fixture pattern is documented at
[[file:../context/language.org::codegen-diagnostics][context/language.org — Codegen Diagnostics]].
Emit flags for debugging (--emit-json, --emit-ir, --emit-asm) are
handled in cli.c:234-269 and main.c:774-1259.

For runtime error handling patterns (error type, ?? operator, etc.),
see [[file:error-handling.org][Error Handling]].

#+BEGIN_EXAMPLE
file:line:column: error: <message>
  line_no | <source line>
          | ^              # points at error column
#+END_EXAMPLE

The =compiler_error= function (reader.c:288-314) formats the
=file:line:column= prefix, prints the relevant source line, and
positions a caret (^) at the error column. For multi-character spans,
=compiler_error_range= (reader.c:242-286) uses =^~~= to show the full
extent.

1. Error Layers

| Layer | Source | Phase |
|-------------+-----------+------------------------------------------|
| Reader | =reader.c= | S-expression / Wisp tokenization & parse |
| Wisp | =wisp.c= | Indentation → S-expression desugaring |
| Inference | =infer.c= | HM constraint generation & unification |
| Dep checker | =dep.c= | Bidirectional NbE type checking |
| Codegen | =codegen.c= | LLVM IR generation and lowering |
| Runtime | =repl.c= | JIT execution (signal handlers) |

For the compilation pipeline that produces these errors, see [[file:build.org][Build System # Compilation Pipeline]].
For type inference error diagnostics, see [[file:type-inference.org][Type Inference]].
For dependent type checking diagnostics, see [[file:dependent-types.org][Dependent Types]].

Key Points:
- Errors are structured as =file:line:column= with a source pointer (=^= or =^~~=)
- Six error layers: Reader → Wisp → Inference → Dep checker → Codegen → Runtime
- Reader errors cover syntax issues (unbalanced parens, unterminated strings)
- Codegen errors cover semantic issues (unbound variables, arity mismatches)
- Runtime errors use signal handlers and longjmp for recovery

See also: [[file:error-handling.org][Error Handling]]

2. Common Error Patterns by Layer

2.1 Reader Errors (=reader.c=)

- *"unexpected ')'"* — unbalanced closing parenthesis.
- *"unterminated string"* — string literal missing closing quote.
- *"unexpected end of input"* — file ends mid-expression.
- Unknown escape sequence in string/char literal.
- Invalid numeric literal prefix.

For reader syntax reference, see [[file:syntax.org][Syntax]].

2.2 Wisp Errors (=wisp.c=)

- Indentation mismatch (inconsistent dedent/indent levels).
- Empty =@= indexing target.
- Char escape in wisp mode (= '\n' = fails because =\\n= becomes
  newline, splitting the literal — use s-expression syntax for char
  literals with escapes).

For Wisp syntax reference, see [[file:wisp.org][Wisp]].

2.3 Codegen Errors (=codegen.c=)

- *Unbound variable* — reference to a name not in scope:

#+BEGIN_SRC monad
(show missing)  ; error: unbound variable 'missing'
#+END_SRC

#+BEGIN_SRC wisp
show missing  ; error: unbound variable 'missing'
#+END_SRC

- *Unknown function* — call to an undefined function:

#+BEGIN_SRC monad
(missing 1)     ; error: unknown function 'missing'
#+END_SRC

#+BEGIN_SRC wisp
missing 1     ; error: unknown function 'missing'
#+END_SRC

- *Arity mismatch* — wrong number of arguments to =quote=, =if=,
  =while=, =unless=, =for=, =set!=, =head=, =tail=, etc.
- *Type mismatch* — arithmetic on strings, array size mismatch.
- *Sized-array mismatch* — array value does not match type annotation.
- *Layout errors* — unknown field, lowercase name, unknown field type.
- *Address-of errors* — taking address of non-lvalue (literal, builtin).
- *Mutation errors* — =set!= on non-assignable target.
- *Cast arity* — type conversion with wrong argument count.
- *Refinement naming* — invalid refinement type name or base type.
- *Empty list forms* — =()= in call position.
- *Non-callable head* — number or other non-function in call position.
- *Runtime collection operation arities* — =head=, =tail=, =pair?=,
  =++=, =count= with wrong argument count.

For variables and scope, see [[file:define.org][Define]].
For type errors, see [[file:types.org][Types]].
For layout types, see [[file:layouts.org][Layouts]].
For refinement types, see [[file:refinements.org][Refinements]].

2.4 Runtime Errors (=repl.c=)

- *SIGSEGV* — from bad pointers (e.g., accessing elements of a
  mis-typed array). Caught by =repl_signal_handler= (signal.c:79)
  which longjmps back to the REPL via =g_repl_escape=.
- *SIGBUS* — from JIT-compiled code.
- *Assertion failure* — =(assert ...)= triggers a longjmp through
  =g_assert_jmp_ptr= back to the eval loop.

For REPL error recovery, see [[file:repl.org][REPL # Signal Handling]].
For runtime error handling, see [[file:error-handling.org][Error Handling]].

3. Diagnostic Fixture Pattern

Each stable =codegen.c= error path has a focused expected-failure
fixture in =tests/codegen/rt_codegen_err_*.mon=. The pattern is one
error per fixture:

#+BEGIN_SRC monad
;; TEST-ID: tests.codegen.rt-codegen-err-unbound-variable
;; TEST-SECTION: diagnostic
;; TEST-EXPECT: fail:compile

(module Main)
(show missing)
#+END_SRC

#+BEGIN_SRC wisp
;; TEST-ID: tests.codegen.rt-codegen-err-unbound-variable
;; TEST-SECTION: diagnostic
;; TEST-EXPECT: fail:compile

module Main
show missing
#+END_SRC

4. Emit Flags for Debugging

These CLI flags let you inspect intermediate compiler output:

| Flag | Output | Use Case |
|-----------------+------------------------+--------------------------------------|
| =--emit-json= | AST as JSON | Inspect parsed form, verify reader |
| =--emit-ir= | LLVM IR text (.ll) | Inspect generated LLVM IR |
| =--emit-asm= | Assembly text (.s) | Inspect generated assembly |
| =--emit-bc= | LLVM bitcode (.bc) | Pass to =llc=, =opt=, or other tools |

These flags are handled in =cli.c:234-269= and =main.c:774-1259=.
Usage:

#+BEGIN_EXAMPLE
monadc --emit-json program.mon   # dump AST to stdout as JSON
monadc --emit-ir  program.mon    # dump LLVM IR to stdout
monadc --emit-asm program.mon    # dump assembly to stdout
#+END_EXAMPLE

For the full CLI flag reference, see [[file:cli.org][CLI Reference]].

5. REPL Recovery

The REPL uses three mechanisms to survive errors without crashing:

| Mechanism | Scope |
|-----------------------+------------------------------|
| =setjmp/=longjmp= | Parse errors in =REPL_PARSE= macro (repl.c:52) |
| =g_repl_escape= | Runtime errors / crashes (repl.c:65) |
| =repl_signal_handler= | SIGSEGV / SIGBUS from JIT (repl.c:77-88) |

For detailed REPL error recovery, see [[file:repl.org][REPL # Signal Handling]].

See also: [[file:repl.org][REPL]]
