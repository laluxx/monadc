#+TITLE: Inline Assembly
#+STARTUP: showeverything
#+OPTIONS: toc:nil

[[file:index.org][<- Back to Index]]

Monad provides inline x86-64 assembly blocks for low-level control
through the =asm.c= preprocessing and codegen pipeline
([[file:../asm.c]], [[file:../asm.h]]). The reader recognizes =(asm
mnemonic operand ...)= inside an S-expression and creates an
=AST_ASM= node ([[file:../reader.c::3889]]). Wisp-style definitions
expand to the same form.

* Architecture ([[file:../asm.h]])

The assembly pipeline is:

1. =preprocess_asm= ([[file:../asm.h::41]]) takes an =AST_ASM= node
   and an =AsmContext=, splits the flat token stream into individual
   instructions by mnemonic recognition, maps parameter symbols to
   registers or LLVM placeholders, escapes numeric immediates, and
   preserves string operands as raw text.
2. =codegen_inline_asm= ([[file:../asm.h::46]]) lowers the preprocessed
   instruction array to LLVM IR via =LLVMGetInlineAsm=.

** Key Data Structures

- =AsmInstruction= ([[file:../asm.h::24]]) — one instruction: a
  mnemonic string (e.g. "add", "mov") and a dynamic array of operand
  strings (registers, immediates, or raw text)

- =AsmContext= ([[file:../asm.h::31]]) — carries the function's
  =EnvParam= array, parameter count, optional =RegisterAllocator=,
  and a boolean =naked= flag controlling prologue/epilogue

- =RegisterAllocator= ([[file:../asm.h::10]]) — maps function
  parameters to SysV AMD64 ABI registers:
  - =int_regs[6]=: =%rdi=, =%rsi=, =%rdx=, =%rcx=, =%r8=, =%r9=
  - =float_regs[8]=: =%xmm0=–=%xmm7=

* Basic Syntax

=(asm mov 1 %rax add %rbx %rax ret)= — multiple instructions in one
block. The preprocessor splits them by mnemonic boundaries.

* Operand Types

| Syntax   | Description                                   |
|----------+-----------------------------------------------|
| =%reg=   | Named register (=%rax=, =%rdi=, =%rbx=, ...)  |
| =$N=     | Immediate integer value (escaped as raw text)  |
| =name=   | Monad function parameter (LLVM placeholder)     |
| ="str"=  | Raw string operand (passed through as-is)      |

Parameter-name symbols in non-naked functions are mapped to LLVM
constraint placeholders (=$0=, =$1=, …). In naked functions they are
mapped to SysV ABI registers instead. Numeric immediates (e.g. =1=)
are escaped (prefixed with =$=). String operands (e.g. ="syscall"=)
are preserved verbatim.

* Naked Functions

Use =:naked True= to suppress function prologue/epilogue. A naked body
must contain at least one =(asm ...)= block. Parameters map directly
to SysV ABI registers:

#+BEGIN_SRC monad
(define [sys-write :: Int -> String -> Int -> Int]
  :naked True
  fd buf len -> (asm mov 1 %rax mov fd %rdi
                      mov buf %rsi mov len %rdx
                      syscall ret))
#+END_SRC

Parameter mapping for naked functions follows the =RegisterAllocator=
(%rdi for arg 0, %rsi for arg 1, %rdx for arg 2, %rcx for arg 3,
%r8 for arg 4, %r9 for arg 5). Float-typed parameters use %xmm0–%xmm7
([[file:../asm.h::10]]).

KNOWN BUG ([[file:../context/language.org::317]]): Naked asm compiles
but does not propagate results through LLVM constraints. The raw asm
output has no constraint wiring — =add y %rax= returns the first
parameter only, not the sum. Naked asm is suitable for syscall
invocations where the return value comes from %rax but not for
arithmetic that needs result propagation.

* Non-naked Functions

Non-naked asm lowers through =LLVMGetInlineAsm= with LLVM constraint
strings ([[file:../asm.h::46]]):

- Return value: ="=r"= for integer-like, ="=x"= for float
- First input: tied to output with =,0=
- Remaining inputs: =,r=

#+BEGIN_SRC monad
(define [add-one :: Int -> Int]
  x -> (asm inc x %rax
            mov %rax x))
#+END_SRC

The first parameter is tied to the output (=,0=), later parameters use
=,r=. Two-operand arithmetic/logical instructions with placeholder
destinations are reordered for AT&T syntax.

** Size-suffix Inference

Bare integer mnemonics in non-naked mode infer size suffixes from LLVM
operand widths ([[file:../context/language.org::328]]):

| Operand Width | Suffix | Example    |
|---------------+--------+------------|
| 8 bits        | =b=    | =addb=     |
| 16 bits       | =w=    | =addw=     |
| 32 bits       | =l=    | =addl=     |
| 64 bits       | =q=    | =addq=     |

For =Int= parameters (64-bit on x86-64) this emits =q=, so =(asm add
x y)= lowers to =addq $2, $1= under constraints =r,0,r=
([[file:../context/language.org::331]]). Verified by
=rt_asm_function_nonaked.mon= printing 30 for =(add 10 20)=.

* Multi-instruction Splitting

The preprocessor splits flat =(asm ...)= token streams into individual
instructions by recognizing mnemonic boundaries. The 60 test fixtures
in [[file:../tests/codegen/asm/]] cover:

- Syscall-shaped =String= operands
- =Int= placeholder arithmetic
- Immediate escaping
- Multi-instruction splitting
- Unary =inc=/=dec=/=neg=/=not=
- Shifts and rotates with immediate counts
- Six-argument placeholder constraints
- =Char= and =Float= return types
- SSE scalar add/multiply
- Conditional moves
- Signed division
- =xchg=
- No-op placement

* Pointer Sugar for Syscall Buffers

=type_from_name= ([[file:../context/types.org::1547]]) accepts
star-prefixed pointer sugar such as =*U8= and converts it to
=Pointer :: U8=:

#+BEGIN_SRC monad
(define [sys-write2 :: Int -> *U8 -> Int -> Int]
  :naked True
  fd buf len -> (asm mov 1 %rax mov fd %rdi
                      mov buf %rsi mov len %rdx
                      syscall ret))
#+END_SRC

String literals still type-check as =String=, so direct
String-to-=*U8= call coercion remains a separate FFI bridge concern
([[file:../context/language.org::397]]).

* Wisp Form

#+BEGIN_SRC monad
define [add-one :: Int -> Int]
  x -> asm inc x %rax
           mov %rax x
#+END_SRC

* Known Gaps

- Naked asm does not propagate results (no LLVM constraint wiring)
- String-to-=*U8= call coercion for syscall convenience is not
  implemented — use an explicit FFI bridge
- No support for =asm goto= or =asm volatile= qualifiers
