The =Coll= module ([[file:../core/prelude/Coll.mon]], ~100 lines) provides
higher-order collection operations. Import with =(import Coll)=. See
[[file:../context/core-libs.org][context/core-libs.org]] for the library architecture.

1. Functions

| Function     | Example                           | Description                   |
|--------------+-----------------------------------+-------------------------------|
| =take=       | =(take 3 '(1 2 3 4 5))=           | First n elements              |
| =drop=       | =(drop 2 '(1 2 3 4 5))=           | Drop n elements               |
| =takeWhile=  | =(takeWhile even? '(2 4 5 7))=    | Take while predicate holds    |
| =dropWhile=  | =(dropWhile even? '(2 4 5 7))=    | Drop while predicate holds    |
| =map=        | =(map (lambda [x] (+ x 1)) '(1 2 3))= | Transform each element   |
| =all?=       | =(all? even? '(2 4 6))=           | All elements satisfy predicate|
| =foldl=      | =(foldl + 0 '(1 2 3 4))=          | Left fold                     |
| =foldr=      | =(foldr + 0 '(1 2 3 4))=          | Right fold                    |

#+BEGIN_SRC monad
(import Coll)

(show (take 3 '(1 2 3 4 5)))        ;; (1 2 3)
(show (drop 2 '(1 2 3 4 5)))        ;; (3 4 5)
(show (all? even? '(2 4 6)))        ;; True
(show (foldl + 0 '(1 2 3 4)))       ;; 10
(show (map (lambda [x] (+ x 1)) '(1 2 3))) ;; (2 3 4)
#+END_SRC

#+BEGIN_SRC wisp
import Coll

show (take 3 '(1 2 3 4 5))        ;; (1 2 3)
show (drop 2 '(1 2 3 4 5))        ;; (3 4 5)
show (all? even? '(2 4 6))        ;; True
show (foldl + 0 '(1 2 3 4))       ;; 10
show (map (lambda [x] (+ x 1)) '(1 2 3)) ;; (2 3 4)
#+END_SRC

2. Predicate Usage

The =all?= function works with any predicate returning Bool:

#+BEGIN_SRC monad
(import Coll)
(import Math)

(show (all? even? '(2 4 6 8)))        ;; True
(show (all? prime? '(2 3 5 7 11)))    ;; True
(show (all? (lambda [x] (> x 0)) '(1 2 3))) ;; True
#+END_SRC

#+BEGIN_SRC wisp
import Coll
import Math

show (all? even? '(2 4 6 8))        ;; True
show (all? prime? '(2 3 5 7 11))     ;; True
show (all? (lambda [x] (> x 0)) '(1 2 3)) ;; True
#+END_SRC

3. Folding Patterns

**3.1 Summing with foldl**

#+BEGIN_SRC monad
(import Coll)
(show (foldl + 0 '(1 2 3 4 5)))      ;; 15
#+END_SRC

#+BEGIN_SRC wisp
import Coll
show (foldl + 0 '(1 2 3 4 5))      ;; 15
#+END_SRC

**3.2 Reversing with foldl**

#+BEGIN_SRC monad
(import Coll)
(define (reverse xs)
  (foldl (lambda [acc x] (cons x acc)) '() xs))

(show (reverse '(1 2 3)))             ;; (3 2 1)
#+END_SRC

#+BEGIN_SRC wisp
import Coll
define (reverse xs)
  foldl (lambda [acc x] (cons x acc)) '() xs

show (reverse '(1 2 3))             ;; (3 2 1)
#+END_SRC

**3.3 Mapping with foldr**

#+BEGIN_SRC monad
(import Coll)
(define (add1-list xs)
  (foldr (lambda [x acc] (cons (+ x 1) acc)) '() xs))

(show (add1-list '(1 2 3)))           ;; (2 3 4)
#+END_SRC

#+BEGIN_SRC wisp
import Coll
define (add1-list xs)
  foldr (lambda [x acc] (cons (+ x 1) acc)) '() xs

show (add1-list '(1 2 3))           ;; (2 3 4)
#+END_SRC

4. Known Issues

- =import Coll= fails while compiling =takeWhile=
- Functions that take function arguments (=map=, =filter=, =all?=,
  =foldl=, =foldr=) are affected by the closure codegen gap: lambdas
  cannot be passed as arguments from paren-mode
- Module requires explicit =import Coll= — not automatic

See [[file:../context/language.org::287][context/language.org — Known codegen gaps]] for the closure gap
status and [[file:functional-programming.org][Functional Programming]] for workarounds.

5. Workaround

When the closure gap blocks higher-order use, use manual recursion or
list-comprehension patterns instead:

#+BEGIN_SRC monad
;; Instead of (map f xs):
(define (add1 xs -> [Int])
  ((match xs)
    []      -> []
    [x|xs]  -> (++ (+ x 1) (add1 xs))))
#+END_SRC

#+BEGIN_SRC wisp
;; Instead of (map f xs):
(define (add1 xs -> [Int])
  (match xs)
    []      -> []
    [x|xs]  -> (++ (+ x 1) (add1 xs)))
#+END_SRC

#+BEGIN_SRC monad
;; Instead of (filter pred xs):
(define (evens xs -> [Int])
  ((match xs)
    []      -> []
    [x|xs]  -> (if (even? x)
                 (++ x (evens xs))
                 (evens xs))))
#+END_SRC

#+BEGIN_SRC wisp
;; Instead of (filter pred xs):
(define (evens xs -> [Int])
  (match xs)
    []      -> []
    [x|xs]  -> (if (even? x)
                 (++ x (evens xs))
                 (evens xs)))
#+END_SRC

See [[file:patterns.org][Pattern Matching]] and [[file:recursion.org][Recursion]] for manual alternatives.
When closures are implemented, higher-order functions from Coll will work
directly ([[file:../context/language.org::287][context/language.org — gap: closures]]).

Implementation Notes

- The Coll module lives in =core/prelude/Coll.mon= (~100 lines) and
  provides higher-order list operations
  ([[file:../context/core-libs.org][context/core-libs.org — Coll module]]).
- =map=, =foldl=, =foldr=, and =all?= all take function arguments; their
  utility is limited by the closure codegen gap — lambdas cannot
  currently be passed as arguments from paren-mode
  ([[file:../context/language.org::287][context/language.org — closure gap]]).
- =takeWhile= fails inference during compilation due to type-level
  complexity in the recursive predicate signature.

Key Points

- =(import Coll)= provides =take=, =drop=, =map=, =foldl=, =foldr=,
  =all?=, =takeWhile=, =dropWhile=.
- Higher-order functions (=map=, =foldl=, =foldr=) are affected by the
  closure codegen gap — use manual recursion as a workaround.
- =takeWhile= compilation fails; use =take= with a manual predicate
  guard instead.
- Folding patterns work for summing, reversing, and mapping when
  closures are not required.
- Module requires explicit =import Coll= — not automatic.
