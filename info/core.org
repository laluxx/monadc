* Implementation Notes

The core library is implemented through a three-layer dispatch
architecture in =codegen_expr=
([[file:../context/codegen-ops.org::dispatch-architecture][context/codegen-ops.org — Dispatch Architecture]]):

1. *AST type switch* — handles leaf nodes (literals, symbols, booleans)
2. *strcmp dispatch* — compares the head symbol string against 83+ known
   operations; this is how most core functions are reached from paren-mode
3. *register_builtins* — 94 env entries for wisp-mode symbol resolution

| Layer            | Code Location               | Purpose                        |
|------------------+-----------------------------+--------------------------------|
| AST type switch  | [[file:../codegen.c][codegen.c]]:3382-3988        | Leaf AST nodes                 |
| strcmp dispatch  | [[file:../codegen.c][codegen.c]]:3889+            | 83+ known operations           |
| register_builtins| [[file:../codegen.c][codegen.c]]:13144            | 94 env entries                 |

Functions listed as "NOT callable" are only in =register_builtins= and are
reachable only from wisp-mode (env lookup), not from paren-mode (strcmp).

** 4.6 Collections Key Points

- Lists are immutable linked cons cells
- Arrays come in 3 variants: stack, fat-pointer, heap-growable
- Sets and maps are hash-based with O(1) average operations
- Mutable variants use =!=-suffixed names (=conj!=, =assoc!=)
- String operations are UTF-8 aware

* Common Pitfalls

- **cond, let, let*, letrec are special forms, not callable functions**:
  They are handled by the reader/compiler and have no strcmp dispatch.
  They work as syntactic forms but NOT via =(apply cond ...)=.
- **map, fn, defn, concat live in Coll Module, not core**: These are
  NOT strcmp dispatachable from paren-mode. Import them from the
  [[file:coll-module.org][Coll Module]] or use the prelude.
- **tail is partially broken on lists**: =tail= segfaults on some list
  inputs. Prefer =cdr= for list tail access.
- **list is partially broken as a variadic constructor**: =(list 1 2 3)=
  works in zero-arg case but the variadic form is unreliable. Use
  =cons= or quoted syntax ='(1 2 3)=.
- **Runtime construction of maps/sets broken without annotation**:
  =(#{:a 1})= without =:: Map= annotation fails. Always annotate.
- **Superscript sugar (=^=) has no strcmp handler**: =x²= desugars at
  reader time but =^= is not callable from paren-mode. Use =(pow x 2)=
  from the [[file:math-module.org][Math Module]] instead.

* NOT Callable Summary

The following are recognized by the compiler but are /not/ callable via
strcmp dispatch from paren-mode (they are =register_builtins=-only or
handled as special forms):

| Name        | Reason                                                          |
|-------------+-----------------------------------------------------------------|
| =map=         | Lives in Coll Module or prelude                                  |
| =fn=          | Short-form lambda (reader sugar, not strcmp)                     |
| =defn=        | Function definition sugar                                        |
| =concat=      | Lives in Coll Module or prelude                                  |
| =substring=   | Lives in Coll Module or prelude                                  |
| =reverse=     | Lives in Coll Module or prelude                                  |
| =nth=         | Lives in Coll Module or prelude                                  |
| =length=      | Lives in Coll Module or prelude                                  |
| =reduce=      | Lives in Coll Module or prelude                                  |
| =cond=        | Special form handled by reader/compiler                          |
| =let=         | Special form handled by reader/compiler                          |
| =let*=        | Special form handled by reader/compiler                          |
| =letrec=      | Special form handled by reader/compiler                          |
| =^=           | Superscript desugaring only                                      |

Note: =cond=, =let=, =let*=, =letrec= are handled as special forms by
the reader/compiler (not via register_builtins or strcmp). =map=, =fn=,
=defn=, =concat=, =substring=, =reverse=, =nth=, =length=, =reduce= live in
[[file:coll-module.org][Coll Module]] or the prelude / core library.

** Complete example:

#+BEGIN_SRC monad
;; Save as core-demo.mon && monadc core-demo.mon && ./core-demo
(module Main)

;; Core library functions — always available without import
(define [x :: Int] 42)
(define [y :: Int] 10)

;; Arithmetic
(show (+ x y 3))     ;; 55
(show (* 2 3 4))     ;; 24
(show (/ 17 5))      ;; 3 (integer division)

;; Comparison
(show (= x 42))      ;; True
(show (< y x))       ;; True

;; Logic
(show (and True (< y x)))    ;; True
(show (or False (= x 0)))    ;; False

;; List operations
(define xs '(1 2 3 4 5))
(show (car xs))      ;; 1
(show (cdr xs))      ;; (2 3 4 5)
(show (count xs))    ;; 5

;; String operations
(show (++ "hello" " " "world"))
(show (count "hello"))
#+END_SRC
