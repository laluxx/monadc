A step-by-step walkthrough from zero to working Monad programs. Covers bindings, functions, data types, control flow, collections, FFI, modules, and advanced topics — with tested examples in both S-expression and Wisp syntax.

Test coverage: tutorial examples are tested by =rt_tutorial_*.mon= fixtures in [[file:../tests/codegen/branches/][tests/codegen/branches/]]. See [[file:../context/tests-arch.org][context/tests-arch.org]] for the full test architecture.

This tutorial walks through Monad from zero to working programs. All
examples use real, tested syntax from the compiler's own codebase.
Every example is drawn from codegen runtime fixtures in
[[file:../tests/codegen/][tests/codegen/]] or from the Main.mon standard library
([[file:../Main.mon][Main.mon]], 1244 lines).

For an overview of the language's design and goals, see
[[file:intro.org][Introduction]]. For the full grammar reference, see
[[file:syntax.org][Syntax]].

* 1 Hello World

A Monad program is a module with top-level expressions. The entry point
is declared with =(module Main)=. Everything that follows is evaluated in
order when the program runs:

#+BEGIN_SRC monad
(module Main)
(show "hello world")
#+END_SRC

#+BEGIN_SRC wisp
module Main
show "hello world"
#+END_SRC

Compile and run:

#+BEGIN_SRC sh
./monad hello.mon -o hello && ./hello
#+END_SRC

The =(module Main)= declaration marks the entry module. Each top-level
form is compiled to native code and executed sequentially. The =(show)=
function prints any value to stdout.

Same program in Wisp syntax (see [[file:wisp.org][Wisp Syntax]]). Notice how Wisp
removes the outer parentheses and uses line breaks + indentation to
structure the code:

#+BEGIN_SRC monad
module Main
show "hello world"
#+END_SRC

#+BEGIN_SRC wisp
module Main
show "hello world"
#+END_SRC

* 2 Basic Expressions

** 2.1 Variables and Bindings

Use =define= to bind a name to a value. Types are inferred
automatically (see [[file:type-inference.org][Type Inference]]):

#+BEGIN_SRC monad
define i 20          ; Inferred Int
define f 40.0        ; Inferred Float
define s "string"    ; Inferred String
define c 'A'         ; Inferred Char
define h 0xFF        ; Inferred Hex
define b 0b1010      ; Inferred Bin
define o 0o755       ; Inferred Oct
#+END_SRC

#+BEGIN_SRC wisp
define i 20          ; Inferred Int
define f 40.0        ; Inferred Float
define s "string"    ; Inferred String
define c 'A'         ; Inferred Char
define h 0xFF        ; Inferred Hex
define b 0b1010      ; Inferred Bin
define o 0o755       ; Inferred Oct
#+END_SRC

Notice that =define= works the same in both syntaxes — the only
difference is the absence of parentheses in Wisp. Comments use
semicolons (=;=).

** 2.2 Type Annotations

Type annotations are optional — Hindley-Milner inference fills in types
automatically ([[file:../context/infer.org::constraint-generation][context/infer.org — Constraint Generation]]).
When you want explicit types, place =:: Type= after the name in brackets:

#+BEGIN_SRC monad
(define [x :: Int] 42)
(define [f :: Float] 40.0)
#+END_SRC

#+BEGIN_SRC wisp
define [x :: Int] 42
define [f :: Float] 40.0
#+END_SRC

The double-colon (=::=) and single-colon (=:=) forms are equivalent.
Both desugar to the same AST. See [[file:define.org][Define]] for more.

** 2.3 Mutation

Variables are immutable by default. To reassign, use =set!=:

#+BEGIN_SRC monad
(define [x :: Int] 1)
(set! x (+ x 1))   ; x is now 2
#+END_SRC

#+BEGIN_SRC wisp
define [x :: Int] 1
set! x (+ x 1)   ; x is now 2
#+END_SRC

The =set!= form is the only way to mutate a variable binding. See
[[file:mutation.org][Mutation]] for more.

For more on bindings, see [[file:define.org][Define]] and [[file:let.org][Let Bindings]].

** 2.4 Numeric Types and Literals

Monad supports multiple numeric literal forms, all inferred by the
type checker:

| Literal | Type    | Example   | Notes                      |
|---------+---------+-----------+----------------------------|
| =42=      | Int     | Integer   | Default for whole numbers  |
| =3.14=    | Float   | Float     | IEEE 754 double precision  |
| =0xFF=    | Int     | Hex       | Starts with =0x=           |
| =0b1010=  | Int     | Binary    | Starts with =0b=           |
| =0o755=   | Int     | Octal     | Starts with =0o=           |
| =1/2=     | Ratio   | Rational  | Exact arithmetic           |
| =50%=     | Ratio   | Percentage| Desugars to =1/2=          |

See [[file:numeric.org][Numeric Types]], [[file:rationals.org][Rationals]].

** 2.5 Arithmetic and Operators

Standard arithmetic uses prefix notation — the operator comes first,
followed by operands. Operators are variadic where meaningful:

#+BEGIN_SRC monad
(show (+ 1 2 3))     ; 6 — variadic: sum all arguments
(show (- 10 3))      ; 7
(show (* 4 5))       ; 20
(show (/ 10 3))      ; 3 — integer division truncates
(show (% 10 3))      ; 1 — modulo
#+END_SRC

#+BEGIN_SRC wisp
show + 1 2 3     ; 6 — variadic
show - 10 3      ; 7
show * 4 5       ; 20
show / 10 3      ; 3 — integer division
show % 10 3      ; 1 — modulo
#+END_SRC

Integer and float types can mix — the compiler promotes Int to Float:

#+BEGIN_SRC monad
(show (+ 1 2.5))     ; 3.5
(show (* 2 3.0))     ; 6.0
#+END_SRC

#+BEGIN_SRC wisp
show + 1 2.5     ; 3.5
show * 2 3.0     ; 6.0
#+END_SRC

** 2.6 Bitwise Operations

Bitwise operations (see [[file:arithmetic.org][Arithmetic]]) work on integer values:

#+BEGIN_SRC monad
(show (& 0xFF 0x0F))     ; 15 — bitwise AND
(show (| 0xF0 0x0F))     ; 255 — bitwise OR
(show (^ 0xFF 0x0F))     ; 240 — bitwise XOR
(show (<< 1 4))          ; 16 — shift left
(show (>> 16 2))         ; 4 — shift right
(show (>>> -16 2))       ;  — arithmetic shift right
#+END_SRC

#+BEGIN_SRC wisp
show & 0xFF 0x0F     ; 15 — bitwise AND
show | 0xF0 0x0F     ; 255 — bitwise OR
show ^ 0xFF 0x0F     ; 240 — bitwise XOR
show << 1 4          ; 16 — shift left
show >> 16 2         ; 4 — shift right
show >>> -16 2       ;  — arithmetic shift right
#+END_SRC

** 2.7 Comparisons and Logic

Comparisons return =Bool= (=True= or =False=):

#+BEGIN_SRC monad
(show (= 1 1))       ; True
(show (!= 1 2))      ; True
(show (< 1 2))       ; True
(show (>= 3 2))      ; True
#+END_SRC

#+BEGIN_SRC wisp
show = 1 1       ; True
show != 1 2      ; True
show < 1 2       ; True
show >= 3 2      ; True
#+END_SRC

Logic uses short-circuit evaluation — =and= and =or= stop evaluating
as soon as the result is determined:

#+BEGIN_SRC monad
(show (and True False))   ; False
(show (or True False))    ; True
(show (not False))        ; True
#+END_SRC

#+BEGIN_SRC wisp
show and True False   ; False
show or True False    ; True
show not False        ; True
#+END_SRC

See [[file:equality.org][Equality]], [[file:booleans.org][Booleans]], [[file:logic.org][Logic]].

| Numeric literal   | =42=                           | =42=                           |
| Arithmetic        | =(+ 1 2)=                      | =+ 1 2=                        |

* 3 Functions

** 3.1 Named Functions

Define functions with =define=. Parameter types and return type can be
annotated with =:: Type=:

#+BEGIN_SRC monad
(define (square [x :: Int] -> Int)
  (* x x))

(define (add [a :: Int] [b :: Int] -> Int)
  (+ a b))
#+END_SRC

#+BEGIN_SRC wisp
define square [x :: Int] -> Int
  * x x

define add [a :: Int] [b :: Int] -> Int
  + a b
#+END_SRC

Return type annotation is optional — the compiler infers it
(see [[file:type-inference.org][Type Inference]]). Without annotations:

#+BEGIN_SRC monad
(define (square x)
  (* x x))
#+END_SRC

#+BEGIN_SRC wisp
define square x
  * x x
#+END_SRC

When you omit type annotations, Hindley-Milner inference determines the
types from how the function is used
([[file:../context/infer.org::expression-inference][context/infer.org — Expression Inference]]).

** 3.2 Anonymous Functions (Lambdas)

Anonymous functions use =lambda= — useful for passing as arguments
to higher-order functions:

#+BEGIN_SRC monad
(define double (lambda [x :: Int] (* x 2)))
(show (double 5))    ; 10
#+END_SRC

#+BEGIN_SRC wisp
define double lambda [x :: Int] * x 2
show double 5    ; 10
#+END_SRC

Lambdas can capture variables from their enclosing scope (closures).
See [[file:functional-programming.org][Functional Programming]], [[file:purity.org][Purity]], and [[file:lambda.org][Lambda]].

** 3.3 Recursion

Functions can call themselves recursively. Here is factorial, the
canonical recursive example:

#+BEGIN_SRC monad
(define (factorial [n :: Int] -> Int)
  (if (<= n 1)
    1
    (* n (factorial (- n 1)))))
#+END_SRC

The base case (=n <= 1=) returns 1. The recursive case multiplies =n=
by the factorial of =n - 1=.

The compiler supports tail-call optimization (TCO) for properly
tail-recursive functions — when the recursive call is the last
expression evaluated. See [[file:recursion.org][Recursion]].

** 3.4 Functions Summary

| Concept           | S-expr Form                              | Wisp Form                        |
|-------------------+------------------------------------------+----------------------------------|
| Named function    | =(define (f x) body)=                    | =define f x\n  body=               |
| Typed function    | =(define (f [x :: Int] -> Int) body)=    | =define f [x :: Int] -> Int\n  body= |
| Lambda            | =(lambda [x] body)=                      | =lambda [x]\n  body=               |
| Recursion         | =(define (f n) (if (= n 0) 1 (* n (f (- n 1)))))= | =define f n\n  if = n 0 1\n    * n (f (- n 1))= |

* 4 Data Types

** 4.1 Strings and Characters

Strings are UTF-8 encoded and immutable. Common operations
(see [[file:strings.org][Strings]]):

#+BEGIN_SRC monad
(define s "hello world")
(show (count s))                ; 11
(show (starts-with? s "hello")) ; True
(show (ends-with? s "world"))   ; True
(show (contains? s "lo wo"))    ; True
(show (s 1))                    ; "e" — string indexing
#+END_SRC

Characters are literal with single quotes:

#+BEGIN_SRC monad
(define c 'A')
(show (Char c))    ; 65 — convert to Int
#+END_SRC

Strings support indexing via function-call syntax: =("hello" 0)=
returns =="h"==. See [[file:casting.org][Casting]] for type conversions.

** 4.2 Algebraic Data Types

Algebraic data types (ADTs) model sum and product types — values that
can be /one of several variants/ (sum) and /contain multiple fields/
(product):

#+BEGIN_SRC monad
(data Bool False True)
(data Option a None (Some a))
(data Tree a Leaf (Node Tree a a Tree))
#+END_SRC

The =Bool= type has two constructors: =False= and =True=. =Option= is
parameterized by type =a=: =None= or =Some a=. =Tree= is a recursive
data type: either a =Leaf= or a =Node= with left tree, value, and right
tree.

Pattern matching destructures ADTs:

#+BEGIN_SRC monad
(match (Some 42)
  case (Some x) -> (show x)
  case None -> (show "nothing"))
#+END_SRC

See [[file:data.org][Data Types]], [[file:patterns.org][Pattern Matching]], [[file:maybe.org][Maybe (Option)]].

** 4.3 Layout Types (C-Compatible Structs)

Layouts define memory layouts that match C structs — useful for FFI
and performance-critical code:

#+BEGIN_SRC monad
(layout Vec3
  [x :: Float]
  [y :: Float]
  [z :: Float]
  where
    method ->array :: Self -> Array
      s -> [s.x s.y s.z])

(define v Vec3 1.0 2.0 3.0)
(show v->array)
#+END_SRC

Layouts can include methods, type parameters, and inline/pointer/array
fields with packed or aligned options. See [[file:layouts.org][Layout Types]].

** 4.4 Refinement Types

Types can be constrained with predicates — subset types that carry
runtime checks:

#+BEGIN_SRC monad
(type Natural { x ∈ Int | x >= 0 })
(type Positive { x ∈ Int | x > 0 })
(type Even { x ∈ Int | x % 2 = 0 })
#+END_SRC

Each refinement generates a predicate function (e.g., =Positive?=)
that checks membership at runtime:

#+BEGIN_SRC monad
(define (safe-divide [n :: Int] [d :: Positive] -> Int)
  (/ n d))
#+END_SRC

The type checker applies the constraint statically when possible and
inserts runtime checks otherwise. See [[file:refinements.org][Refinement Types]].

** 4.5 Type System Overview

Monad has a dual type system: Hindley-Milner inference for the surface
language ([[file:type-inference.org][Type Inference]]) and a dependent type
checker based on Intuitionistic Type Theory ([[file:dependent-types.org][Dependent Types]]).
The two systems are bridged by a =TERM_EMBED= node type
([[file:../context/dep.org::dep-interop-bridge][context/dep.org — HM/ITT Bridge]]).

See [[file:types.org][Types]] for the full type reference.

* 5 Control Flow

** 5.1 Conditionals

Use =if= for simple two-branch conditionals:

#+BEGIN_SRC monad
(define (max [a :: Int] [b :: Int] -> Int)
  (if (> a b) a b))
#+END_SRC

#+BEGIN_SRC wisp
define max [a :: Int] [b :: Int] -> Int
  if > a b a b
#+END_SRC

The =if= form takes three arguments: condition, then-expression, and
else-expression. It always returns a value — all forms in Monad are
expressions.

Use =unless= as a statement form when there is only one branch:

#+BEGIN_SRC monad
(unless False (show "executed"))   ; prints "executed"
#+END_SRC

#+BEGIN_SRC wisp
unless False
  show "executed"   ; prints "executed"
#+END_SRC

See also [[file:conditionals.org][Conditionals]] for =cond=, =when=, and the =??= defaulting operator.

** 5.2 Iteration

Use =while= for mutable loops:

#+BEGIN_SRC monad
(define [x :: Int] 0)
(while (< x 3)
  (show x)
  (set! x (+ x 1)))
#+END_SRC

#+BEGIN_SRC wisp
define [x :: Int] 0
while < x 3
  show x
  set! x + x 1
#+END_SRC

Use =for= comprehensions over arrays, lists, sets, and strings:

#+BEGIN_SRC monad
(for i -> [1 2 3 4 5]
  (show i))
#+END_SRC

#+BEGIN_SRC wisp
for i -> [1 2 3 4 5]
  show i
#+END_SRC

See [[file:iteration.org][Iteration]], [[file:range.org][Ranges]], [[file:control.org][Control Flow]].

** 5.3 Sequencing

Use =begin= to execute multiple expressions in sequence, returning the
last value:

#+BEGIN_SRC monad
(define x (begin
  (show "computing...")
  (+ 1 2)))   ; x = 3
#+END_SRC

See [[file:sequencing.org][Sequencing]].

** 5.4 Control Flow Summary

| Form      | Purpose                        | Returns           |
|-----------+--------------------------------+-------------------|
| =if=      | Two-branch conditional         | Branch value      |
| =cond=    | Multi-branch conditional       | Matching branch   |
| =when=    | One-branch side-effect guard   | Last expression   |
| =unless=  | Negated one-branch guard       | Last expression   |
| =while=   | C-style loop                   | =nil=              |
| =for=     | Collection iteration           | Collection        |
| =begin=   | Expression sequencing          | Last expression   |

* 6 Collections

** 6.1 Lists

Lists are linked cons cells — immutable and recursive:

#+BEGIN_SRC monad
(define my-list '(1 2 3))
(show (car my-list))        ; 1
(show (cdr my-list))        ; (2 3)
(show (cons 0 my-list))     ; (0 1 2 3)
(show (++ my-list '(4 5)))  ; (1 2 3 4 5)
(show (count my-list))      ; 3
#+END_SRC

The =quote= syntax (='=) prevents evaluation of the list, creating a
literal linked list. =car= gets the first element, =cdr= gets the rest.
See [[file:list.org][Lists]].

** 6.2 Arrays

Arrays come in three forms with different memory characteristics
(see [[file:arrays.org][Arrays]]):

| Variant      | Syntax    | Memory      | Size              | Use Case                      |
|--------------+-----------+-------------+-------------------+-------------------------------|
| Stack        | =[1 2 3]= | Stack frame | Fixed at compile  | Small, known-size data        |
| Fat pointer  | =[16kb I32]= | Stack/heap | Runtime-sized     | Medium, runtime-sized data    |
| Heap-growable| =~[1 2 3]= | Heap        | Runtime-sized     | Large, dynamically-sized data |

#+BEGIN_SRC monad
(define stack-arr [1 2 3])           ; Stack array, known size at compile time
(define heap-arr ~[1 2 3])           ; Heap-allocated, runtime-sized
#+END_SRC

Array indexing uses function-call syntax:

#+BEGIN_SRC monad
(define arr [10 20 30])
(show (arr 0))    ; 10
(show (arr 1))    ; 20
#+END_SRC

See [[file:heap.org][Heap Allocation]], [[file:arena.org][Arena Allocation]].

** 6.3 Sets

Sets are open-addressing hash sets with O(1) average operations:

#+BEGIN_SRC monad
(define [s :: Set] {1 2 3})
(show (contains? s 1))     ; True
(show (conj s 4))          ; {1 2 3 4} — returns new set
#+END_SRC

Set operations return new sets (immutable by default). Mutable variants
(=conj!=, =disj!=) modify in place. See [[file:set.org][Sets]].

** 6.4 Maps

Maps are hash maps for keyword/value dictionaries:

#+BEGIN_SRC monad
(define [m :: Map] {:a 1 :b 2})
(show (m :a))              ; 1 — key lookup
#+END_SRC

Maps support immutable operations (=assoc=, =dissoc=) and mutable
variants (=assoc!=, =dissoc!=). See [[file:map.org][Maps]].

See [[file:collections.org][Collections]] for the full overview.

* 7 Advanced Topics

** 7.1 C FFI

Call C functions directly — no binding generators needed. Just
include the header:

#+BEGIN_SRC monad
(include "math.h")
(show (sqrt 9.0))     ; 3.0
(show (floor 3.7))    ; 3.0
(show (ceil 3.2))     ; 4.0

(include "stdlib.h")
(show (atoi "42"))    ; 42

(include "string.h")
(show (strlen "hello"))  ; 5
#+END_SRC

The FFI uses libclang to parse headers automatically
([[file:../context/ffi.org::ffi-parse-header][context/ffi.org — Header Parsing]]) and maps C types
to Monad types.

** 7.2 Modules and Imports

Modules help organize code into namespaces:

#+BEGIN_SRC monad
(module Main)  ; Exports everything by default
(import Math)
(import qualified Std.List :as L)
#+END_SRC

Import options:

| Form                          | Behavior                            |
|-------------------------------+-------------------------------------|
| =(import Math)=                 | Brings all symbols into scope       |
| =(import Math [sqrt log])=      | Imports only listed symbols         |
| =(import Math hiding [sqrt])=   | Imports everything except listed    |
| =(import qualified Math :as M)= | Access via =M:sqrt= (prefix required) |

See [[file:../context/module.org][context/module.org]] for the module resolution algorithm.

** 7.3 Inline Assembly

Assembly goes inside function bodies with =(asm ...)=:

#+BEGIN_SRC monad
(include "asm.h")

(define (sys-write [fd :: Int] [buf :: String] [len :: Int] -> Int)
  (asm
    mov 1 %rax
    mov fd %rdi
    mov buf %rsi
    mov len %rdx
    syscall
    ret))
#+END_SRC

Naked functions (no prologue/epilogue) use =:naked True=.
See [[file:../context/asm.org][context/asm.org]].

** 7.4 Testing

Tests are embedded directly in source files:

#+BEGIN_SRC monad
(tests
  (assert-eq (+ 1 2) 3 "addition works")
  (assert-eq (square 5) 25 "square works"))
#+END_SRC

Run with the test subcommand:

#+BEGIN_SRC sh
./monad test myfile.mon
#+END_SRC

See [[file:testing.org][Testing]], [[file:fuzzing.org][Fuzzing]].

** 7.5 REPL

Launch the interactive REPL for experimentation:

#+BEGIN_SRC sh
./monad -i
#+END_SRC

#+BEGIN_SRC monad
> (+ 1 2)
3
> (define greeting "hello")
> (++ greeting " world")
"hello world"
> (show greeting)
hello
#+END_SRC

The REPL uses LLVM JIT compilation via ExecutionEngine — each
expression is compiled and executed immediately
([[file:../context/repl.org::repl-evaluation][context/repl.org — REPL Evaluation]]).

* 8 Complete Example

A program that computes Fibonacci numbers:

#+BEGIN_SRC monad
(module Main)

(define (fib [n :: Int] -> Int)
  (if (<= n 2)
    1
    (+ (fib (- n 1)) (fib (- n 2)))))

(show (fib 10))   ; 55
#+END_SRC

This example compiles and runs successfully, verified by codegen runtime
fixtures (see [[file:../tests/codegen/rt_arith_all.mon][tests/codegen/rt_arith_all.mon]] for arithmetic
coverage).

For more complete programs, browse the [[file:../how_to/][how_to/]] directory and the
standard library at [[file:../Main.mon][Main.mon]].

* Common Pitfalls

- **cond, when, let, let*, letrec lack codegen dispatch**: These are
  parsed as special forms but the strcmp handler is missing — they work
  as built-in syntax but NOT as paren-mode function calls.
- **Map/set literals need explicit annotation**: `#{:a 1}` and `{1 2 3}`
  require `:: Map` / `:: Set` annotation or they break at runtime.
- **Closures are blocked**: Higher-order functions like =map=, =filter=,
  and =lambda= used as function arguments lack codegen dispatch.
- **String indexing returns String, not Char**: `("hello" 0)` returns
  `"h"` (a string), not a character. Use string-specific operations.
- **Integer division truncates**: `(/ 7 2)` gives `3`, not `3.5`. Use
  float literals for float division: `(/ 7.0 2)`.
- **set! requires prior define**: You must `define` a variable before
  using =set!= — there is no implicit declaration.
- **Tail-call optimization is fragile**: TCO only applies when the
  recursive call is in /tail position/ (the last expression in a function
  body). Calls in non-tail positions (e.g., inside `(+ 1 (f n))`) still
  consume stack.
