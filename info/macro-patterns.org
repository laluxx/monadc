Monad macros are functions that return =Syntax= ([[file:../macro.c::765][macro.c:765]]).
They receive their arguments unevaluated and produce a new s-expression
to compile. For the full macro system architecture — including reader
integration, hygiene renaming, variadic splicing, and the expander
implementation — see [[file:../context/macro.org][context/macro.org]].
For the user-facing macro API, syntax, and quasi-output mechanics, see
[[file:macros.org][Macros Reference]].

All examples below use syntax patterns tested by the compiler's own
codegen fixtures. The =swap!= macro pattern, for instance, is verified
in =rt_macro_swap.mon=.

* 1. Basics

#+BEGIN_SRC monad
;; A macro that always expands to 42
(define (unitmacro -> Syntax)
  42)

;; Pass-through macro
(define (identity-macro x -> Syntax)
  x)
#+END_SRC

#+BEGIN_SRC wisp
;; A macro that always expands to 42
define unitmacro -> Syntax
  42

;; Pass-through macro
define identity-macro x -> Syntax
  x
#+END_SRC

See [[file:macros.org][Macros — Defining Macros]] for the definition syntax and
[[file:lambda.org][Lambda]] for the =-> Syntax= return type convention.

* 2. Conditional Expansion

#+BEGIN_SRC monad
(define when :: condition . body -> Syntax
  (if condition (begin . body)))
#+END_SRC

#+BEGIN_SRC wisp
define when :: condition . body -> Syntax
  if condition (begin . body)
#+END_SRC

The =. body= captures all remaining arguments as a list. =when= wraps
them in =begin= inside an =if=. See [[file:conditionals.org][Conditionals — when]] for
the non-macro =when= form.

* 3. Negated Condition

#+BEGIN_SRC monad
(define (macro-unless condition . body -> Syntax)
  (if condition (begin) (begin . body)))
#+END_SRC

#+BEGIN_SRC wisp
define macro-unless condition . body -> Syntax
  if condition (begin) (begin . body)
#+END_SRC

This is the macro version of =unless= ([[file:conditionals.org][Conditionals — unless]]).

* 4. Mutation Helper

#+BEGIN_SRC monad
(define (swap! a b -> Syntax)
  (let [tmp a]
    (set! a b)
    (set! b tmp)))
#+END_SRC

#+BEGIN_SRC wisp
define swap! a b -> Syntax
  let [tmp a]
    set! a b
    set! b tmp
#+END_SRC

Usage:

#+BEGIN_SRC monad
(module Main)
(define [x :: Int] 1)
(define [y :: Int] 99)
(swap! x y)
(show x)  ;; 99
(show y)  ;; 1
#+END_SRC

#+BEGIN_SRC wisp
module Main
define [x :: Int] 1
define [y :: Int] 99
swap! x y
show x  ;; 99
show y  ;; 1
#+END_SRC

See [[file:mutation.org][Mutation]] for the =set!= form and [[file:let.org][Let Bindings]] for =let=.

* 5. Variadic Recursion

#+BEGIN_SRC monad
(define (my-and . args -> Syntax)
  []       -> True
  [x]      -> x
  [x|rest] -> (if x (my-and . rest) False))
#+END_SRC

#+BEGIN_SRC wisp
define my-and . args -> Syntax
  []       -> True
  [x]      -> x
  [x|rest] -> (if x (my-and . rest) False)
#+END_SRC

Pattern-matches on the argument list: empty =True=, single =x=, or
recurse. See [[file:recursion.org][Recursion]] for recursive patterns and
[[file:patterns.org][Pattern Matching]] for the destructuring syntax.

* 6. Variadic Macro Semantics

A rest parameter (=. name=) captures all remaining arguments as a list.
When spliced back with =. name= inside the macro body (e.g., =(begin . body)=),
the list is spliced into the output form. When used outside a list position
(e.g., as an argument to a non-variadic form), =body= is passed as a single
list value:

#+BEGIN_SRC monad
(define (example . args -> Syntax)
  (list . args))    ;; splices: (list 1 2 3)
(define (example . args -> Syntax)
  (list args))      ;; wraps: (list (1 2 3))
#+END_SRC

#+BEGIN_SRC wisp
define example . args -> Syntax
  list . args    ;; splices: (list 1 2 3)
define example . args -> Syntax
  list args      ;; wraps: (list (1 2 3))
#+END_SRC

See [[file:macros.org][Macros — Variadic Splicing]] for the mechanics and
[[file:../context/macro.org::macro-variadic-splicing][context/macro.org]] for implementation details.

* 7. Assertion Macro

#+BEGIN_SRC monad
(define (assert condition . body -> Syntax)
  (when condition
    (begin . body)))
#+END_SRC

#+BEGIN_SRC wisp
define assert condition . body -> Syntax
  when condition
    begin . body
#+END_SRC

See [[file:testing.org][Testing]] for the built-in =assert= form and
[[file:error-handling.org][Error Handling]] for error-reporting patterns.

* 8. #() Quasi-Output Syntax

Macro bodies can use =#(...)= to delimit the template output region.
Code outside =#()= is intended for compile-time computation (future work);
currently =#()= is stripped and only the inner subtree becomes output
([[file:../context/macro.org::marco-quasi-output][context/macro.org]]):

#+BEGIN_SRC monad
(define (assert condition message -> Syntax)
  (if condition
    #(when (not condition) (error message))
    #(begin)))
#+END_SRC

#+BEGIN_SRC wisp
define assert condition message -> Syntax
  if condition
    #(when (not condition) (error message))
    #(begin)
#+END_SRC

See [[file:macros.org][Macros — #() Quasi-Output Syntax]] for full details.

* 9. Hygiene

Macros are unhygienic. Variables in the expansion can capture
bindings at the call site:

#+BEGIN_SRC monad
(define (swap x y -> Syntax)
  (let [tmp x]
    (set! x y)
    (set! y tmp)))
#+END_SRC

#+BEGIN_SRC wisp
define swap x y -> Syntax
  let [tmp x]
    set! x y
    set! y tmp
#+END_SRC

If the caller has a =tmp= binding, it gets captured. Use gensymmed
names or rely on the compiler's hygiene ([[file:macros.org][Macros — Hygiene]])
to avoid this in complex macros.

* 10. Fixpoint Expansion

Macro expansion runs to a fixpoint (capped at 1024 rounds). A macro that
produces another macro call is expanded again in a subsequent round.
This enables recursive macros but requires caution: an infinite expansion
loop hits the cap and produces a compiler error
([[file:../context/macro.org::macro-fixpoint][context/macro.org]]):

#+BEGIN_EXAMPLE
Round 1: (my-macro x) → (another-macro x)
Round 2: (another-macro x) → (my-macro x)
... cap at 1024 → compiler error
#+END_EXAMPLE

See [[file:macros.org][Macros — How Macros Work]] for the expansion step details and
[[file:recursion.org][Recursion]] for general recursion patterns.

* 11. Key Points

- Macros are just functions returning =Syntax=
- No separate =macro= keyword, no quasiquotation
- Variadic args use =. name=
- Arguments are received unevaluated
- Return value must be a valid s-expression
- Rest params splice in list position, wrap in non-list position
- Expansion is fixpoint with a 1024-round cap
- Unhygienic by default; compiler hygiene available via rename
- =#()= marks quasi-output template regions

Implementation Notes

Macro patterns compile through the same expander infrastructure as macros
(=macro.c=, 765+ lines). Key implementation details for writers:

- **Variadic splicing**: a rest parameter (=. name=) in the macro's lambda
  captures trailing arguments as a list. When re-inserted with =. name=
  inside a list, the nodes are spliced flat; outside a list they wrap
  ([[file:../context/macro.org::macro-variadic-splicing][context/macro.org — variadic splicing]]).
- **Pattern matching on args**: =match= destructures the captured syntax
  list; the pattern-match compiler (=pmatch.c=) generates the case
  analysis ([[file:../context/pmatch.org][context/pmatch.org]]).
- **Fixpoint caution**: recursive macros that produce other macro calls
  are expanded in subsequent fixpoint rounds; infinite loops hit the
  1024-round cap and abort
  ([[file:../context/macro.org::macro-fixpoint][context/macro.org — fixpoint]]).
- **Hygiene**: by default macros are unhygienic (names can capture).
  The compiler's hygienic rename (=macroname__varname__N=) must be
  explicitly relied on or worked around with gensymmed names
  ([[file:../context/macro.org::macro-hygiene][context/macro.org — hygiene]]).

Key Points

- All patterns are =define= forms returning =Syntax= — no special macro
  keyword.
- Variadic macros use =. name= for rest args; =. name= splices in list
  position, wraps otherwise.
- =#()= quasi-output delimits the output template region.
- =match= destructures captured syntax lists for conditional expansion.
- Guard clauses (=| condition=) work inside =match= patterns.
- Fixpoint expansion enables recursive macros but requires termination
  guarantees.
- Unhygienic by default; compiler hygiene uses =__= separator.
