* Quick Reference

| Module     | Key Exports                                    |
|------------+------------------------------------------------|
| Core       | Built-in functions (always available)          |
| Math       | =gcd=, =lcm=, =factorial=, =sin=, =sqrt=, =pi= |
| Coll       | =map=, =foldl=, =foldr=, =take=, =drop=        |
| Test       | =assert-eq=, =deftest=, =run-tests=             |
| Ascii      | =upper=, =lower=                                 |

Test coverage: =make test-core= runs per-module tests via
=python3 tests/run_core.py=. Individual modules are tested by=rt_core_*.mon=
fixtures. Main.mon coverage is documented under [[file:../context/main-mon.org][context/main-mon.org]].

The Monad standard library is currently composed of two layers:

1. *Builtins* — functions dispatched directly in =codegen.c= (always
   available, no import needed). See [[file:core.org][Core Library]].

2. *Core modules* — source files under =core/= that contain reusable
   definitions compiled as needed.

* Core Modules

The =core/= directory contains per-module library sources:

| Module | File                              | Working Status                                                                                                                                                                                                                                                |
|--------+-----------------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Math   | =core/Math.mon= (400 lines)         | Partially working. Exports 30+ functions/constants including =gcd=, =lcm=, =factorial=, =fib=, =prime?=, =clamp=, =sin=, =sqrt=, =pow=, =floor=, =ceiling=, =round=, constants =pi=, =e=, =tau=, =phi=, infinite sequences =naturals=, =positives=, =evens=, =odds=. ** Usage examples

#+BEGIN_SRC monad
(import Math)

(show (gcd 12 8))       ; 4
(show (even? 42))       ; True
(show (factorial 5))    ; 120
(show pi)               ; 3.14159...
(show (clamp 0 100 150)); 100
#+END_SRC

#+BEGIN_SRC wisp
import Math

show (gcd 12 8)       ; 4
show (even? 42)       ; True
show (factorial 5)    ; 120
show pi               ; 3.14159...
show (clamp 0 100 150); 100
#+END_SRC

#+BEGIN_SRC monad
(import Coll)

(show (take 3 '(1 2 3 4 5)))       ; (1 2 3)
(show (drop 2 '(1 2 3 4 5)))       ; (3 4 5)
(show (all? even? '(2 4 6)))       ; True
(show (foldl + 0 '(1 2 3 4)))      ; 10
#+END_SRC

#+BEGIN_SRC wisp
import Coll

show (take 3 '(1 2 3 4 5))       ; (1 2 3)
show (drop 2 '(1 2 3 4 5))       ; (3 4 5)
show (all? even? '(2 4 6))       ; True
show (foldl + 0 '(1 2 3 4))      ; 10
#+END_SRC

Note: Core prelude modules are not yet automatically imported at
startup. Explicit =import= statements are required. Known compilation
issues exist (context/language.org:417-424): =import Coll= currently
fails while compiling =takeWhile= in =Coll.mon=; variadic =gcd= in
=Math.mon= fails.

** Test module

The =core/prelude/Test.mon= module provides =assert-eq=, =assert-true=,
=assert-false=, =deftest=, and =run-tests= for embedded testing:

#+BEGIN_SRC monad
(import Test)

(deftest arithmetic
  (assert-eq (+ 1 1) 2 "1+1=2")
  (assert-true (= 2 (+ 1 1)) "2=1+1"))
#+END_SRC

#+BEGIN_SRC wisp
import Test

deftest arithmetic
  assert-eq (+ 1 1) 2 "1+1=2"
  assert-true (= 2 (+ 1 1)) "2=1+1"
#+END_SRC

See [[file:testing.org][Testing]] for the full test framework reference.

* Main.mon

The file =Main.mon= (1244 lines) serves as both the compiler's primary
test surface and a de facto standard library reference. It demonstrates
real usage of every language feature:

- **Variables and types**: =define= with inferred and annotated types
  (=Int=, =Float=, =Char=, =String=, =Hex=, =Bin=, =Oct=).
- **Arithmetic**: mixed-type operations across all numeric types.
- **Functions**: typed function definitions with multi-param arrows,
  documentation strings, aliases.
- **Lists**: =list=, =make-list=, =append!=, =list-length=, =list-empty?=,
  =car=, =cons=, =++=.
- **Arrays**: fixed-size arrays (=Arr :: T :: N=), heap arrays (=~[]=),
  typed and inferred element types.
- **Layouts**: =layout Point= with construction and access, =where=
  clause methods.
- **Inline assembly**: naked (=:naked True=) and non-naked (=:naked
  False=) asm functions with SysV register usage.
- **Maps and sets**: =set=, =conj=, =disj=, =contains?=, map construction
  with =#{"key" val}= notation, =assoc=, =dissoc=, =keys=, =vals=,
  =merge=, =find=, =get= with default, map-as-function.
- **Control flow**: =for= loops with and without step, =if= expressions,
  =quote=, =show=.
- **Conditional compilation**: =#+LINUX=, =#+X86-64= feature flags.
- **Type aliases**: =type Code List=.
- **Ratios**: =1/3=, rational arithmetic.
- **Casting**: type-as-function casting across all builtin types.
- **Infix sugar**: =`into`= backtick infix notation.

* old-core/ (Aspirational / Pre-Rewrite)

The =old-core/= directory contains aspirational library modules from a
previous compiler era. They are NOT currently importable:

| File               | Contents                             |
|--------------------+--------------------------------------|
| =old-core/List.mon=   | List functions (aspirational)        |
| =old-core/Set.mon=    | Set functions (wisp =tests= block parsing issue) |
| =old-core/text/=      | Ascii, String modules (old =::= syntax)          |
| =old-core/Math.mon=   | Math functions (superseded by =core/Math.mon=)   |
| =old-core/prelude/    | Coll, Test modules (old versions)                |
| =old-core/json.mon=   | JSON parser/emitter                  |
| =old-core/Random.mon= | Random number generation             |
| =old-core/Assert.mon= | Assertion utilities                  |

These files use older syntax and are not compiled by the current
compiler. They serve as design references for future library work.
