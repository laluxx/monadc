Monad provides the =Error= type and the =Result= pattern for fallible operations.

=Result= represents a computation that can fail with an error message.
It is the standard pattern for fallible operations where a bare =nil=
is insufficient — the error carries a =String= message.

For the optional-value variant (no error message), see
[[file:maybe.org][Maybe (Option)]]. For runtime error mechanisms, see
[[file:error-handling.org][Error Handling]].

* 1. Error Type Definition

#+BEGIN_SRC monad
(data Error String)
#+END_SRC

#+BEGIN_SRC wisp
data Error String
#+END_SRC

#+BEGIN_SRC monad
(define parse-error (Error "invalid input"))
(define file-error  (Error "file not found"))
#+END_SRC

#+BEGIN_SRC wisp
define parse-error (Error "invalid input")
define file-error  (Error "file not found")
#+END_SRC

=Error= wraps a diagnostic =String= message. It is a single-constructor
ADT with one field. See [[file:data.org][Data Types]] for ADT definition syntax.

*** Key Points
- =Error= is a simple ADT wrapping a =String= message
- No special compiler support — standard tagged union
- Prefer =Error= over bare strings for type-safe error handling

See also: [[file:data.org][Data Types]]

* 2. Usage Pattern: Result

#+BEGIN_SRC monad
(define (parse-int s -> (Result Int String))
  ...)
#+END_SRC

#+BEGIN_SRC wisp
define parse-int s -> (Result Int String)
  ...
#+END_SRC

Where =Result= is used as a general type for "ok or error" returns.
The ok variant carries the success value; the error variant carries
a diagnostic message.

For pattern matching on Result types, see [[file:patterns.org][Pattern Matching]].

*** Key Points
- =Result a String= is the standard fallible-return pattern
- =Error String= carries the message, ok variant carries the value
- Type-safe: the success type and error type are both explicit

See also: [[file:patterns.org][Pattern Matching]]

* 3. error (Builtin)

The =(error "message")= builtin halts the program at runtime:

#+BEGIN_SRC monad
(define (assert-nonzero n)
  (if (= n 0) (error "assertion failed: n is zero")
    n))
#+END_SRC

#+BEGIN_SRC wisp
define assert-nonzero n
  if (= n 0) (error "assertion failed: n is zero")
    n
#+END_SRC

Codegen emits a call to =rt_error= which prints the message and calls
=exit(1)= ([[file:../codegen.c::6604]]). See
[[file:../context/codegen-ops.org][context/codegen-ops.org]] for the error codegen.

*** Key Points
- =(error msg)= is an unrecoverable halt — like Rust's =panic!= or Python's =raise=
- Use =Error String= ADT for recoverable errors; =error= for unrecoverable assertions
- Codegen calls =rt_error= which prints then calls =exit(1)=

See also: [[file:error-handling.org][Error Handling]]

* 4. undefined (Builtin)

=(undefined)= is a placeholder for unimplemented branches:

#+BEGIN_SRC monad
(match x
  None    -> (undefined)
  (Some v)-> v)
#+END_SRC

#+BEGIN_SRC wisp
match x
  None    -> (undefined)
  (Some v)-> v
#+END_SRC

Codegen emits an LLVM =unreachable= instruction or a call to
=rt_undefined= depending on context
([[file:../codegen.c::6685]]). Non-exhaustive pattern matches emit
=(undefined)= as their fallback. See [[file:patterns.org][Pattern Matching]].

*** Key Points
- =(undefined)= marks unreachable or unimplemented code paths
- Codegen uses LLVM =unreachable= (UB) or =rt_undefined= (runtime trap)
- Non-exhaustive =match= uses =(undefined)= as implicit fallback

See also: [[file:patterns.org][Pattern Matching]]

* 5. ?? With Result

The null-coalescing operator =??= provides a default:

#+BEGIN_SRC monad
(?? (parse-int "abc") 0)           ;; 0 on parse failure
#+END_SRC

#+BEGIN_SRC wisp
?? (parse-int "abc") 0
#+END_SRC

See [[file:conditionals.org][Conditionals]] for the =??= operator reference.

*** Key Points
- =??= unwraps a successful result or returns the default
- Works identically for =Option= and =Result=
- Discards the error message — use =match= when you need to inspect the error

See also: [[file:conditionals.org][Conditionals]]

* 6. Related: Result in Other Languages

| Language   | Type                             |
|------------+----------------------------------|
| Monad      | =(Result a String)= or equivalent |
| Rust       | =Result<T, E>=                  |
| Haskell    | =Either e a=                    |
| OCaml      | =('a, 'b) result=               |
| Java       | No direct equivalent             |

*** Key Points
- =(Result a String)= is the Monad equivalent of Rust's =Result<T, E>= or Haskell's =Either e a=
- Error type is always =String= currently — future versions may generalise the error type
