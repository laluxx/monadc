* Deep Nesting

Parentheses override wisp indentation:

#+BEGIN_SRC wisp
case x
  (Just v)
    show "got"
    show v
  Nothing
    show "nothing"
#+END_SRC

Each =case= clause's first line has the pattern in parens (overrides
wisp grouping). The function call body then starts at the next level.

* Mixing Wisp and S-exprs

Any line can use S-expression syntax. The wisp expander only cares
about lines that look like =prefix args ...=.

* Known Bug: Metadata in Wisp

Metadata keywords (=:alias=, =:doc=, =:deprecated=) in wisp-syntax
=define= forms are emitted as separate top-level forms instead of
being included inside the =define= s-expression. Use S-expression
syntax for reliable metadata.

* Platform Notes

Wisp expander is a simple line-based preprocessor. It handles basic
cases well but may have edge cases with complex nesting or mixed
indentation. Test your wisp code before relying on it.
