Test coverage: use of =(Option a)= pattern in codegen fixtures is
verified indirectly by =rt_match_*.mon= and =rt_bind_*.mon=.

=(Option a)= represents an optional value: either =None= (no value) or
=(Some x)= (a value of type =a=). It is the standard safe alternative
to bare =nil= returns.

For the error-carrying variant, see [[file:result.org][Result]]. For the design
rationale, see [[file:../context/language.org][context/language.org]].

* 1. Type Definition

#+BEGIN_SRC monad
(data Option a
  None
  (Some a))
#+END_SRC

#+BEGIN_SRC wisp
data Option a
  None
  (Some a)
#+END_SRC

=Option= is a regular algebraic data type — defined in the surface
language, no special compiler support. It participates fully in pattern
matching, type inference, and monomorphization. See [[file:data.org][Data Types]]
and [[file:patterns.org][Pattern Matching]].

*** Key Points
- =Option= is an ordinary ADT — no magic, no special compiler treatment
- Two constructors: =None= (absent) and =(Some a)= (present)
- Fully generic over any type =a= via Hindley-Milner polymorphism

See also: [[file:data.org][Data Types]], [[file:patterns.org][Pattern Matching]]

* 2. Construction

#+BEGIN_SRC monad
(define no-value None)
(define some-value (Some 42))
(define maybe-string (Some "hello"))
#+END_SRC

#+BEGIN_SRC wisp
define no-value None
define some-value (Some 42)
define maybe-string (Some "hello")
#+END_SRC

*** Key Points
- Construct =None= for absence, =(Some value)= for presence
- Type is inferred from the wrapped value
- Works with any type — Int, String, ADTs, layouts, etc.

See also: [[file:patterns.org][Pattern Matching]]

* 3. Consumption With Pattern Matching

The primary way to work with =Option= is =match=:

#+BEGIN_SRC monad
(define (safe-head xs -> (Option a))
  (if (nil? xs) None
    (Some (car xs))))

(define (show-option x -> String)
  (match x
    None      -> "nothing"
    (Some v)  -> (++ "got " (show v))))
#+END_SRC

#+BEGIN_SRC wisp
define safe-head xs -> (Option a)
  if (nil? xs) None
    Some (car xs)

define show-option x -> String
  match x
    None      -> "nothing"
    (Some v)  -> ++ "got " (show v)
#+END_SRC

See [[file:patterns.org][Pattern Matching]] for full match expression
reference and [[file:../context/pmatch.org][context/pmatch.org]] for the pattern
match compiler.

*** Key Points
- =match= destructures =Option= by constructor
- Exhaustiveness checking ensures no case is forgotten
- Pattern variables bind the unwrapped value

See also: [[file:../context/pmatch.org][context/pmatch.org]]

* 4. Pattern: Safe Division

#+BEGIN_SRC monad
(define (safe-div a b -> (Option Int))
  (if (= b 0) None
    (Some (/ a b))))

(define result (safe-div 10 2))     ;; (Some 5)
(define bad    (safe-div 10 0))     ;; None
#+END_SRC

#+BEGIN_SRC wisp
define safe-div a b -> (Option Int)
  if (= b 0) None
    Some (/ a b)

define result (safe-div 10 2)
define bad    (safe-div 10 0)
#+END_SRC

See [[file:error-handling.org][Error Handling]] for more error-handling
patterns.

*** Key Points
- =Option= replaces sentinel values (nil, -1, etc.) with a type-safe alternative
- The return type =(Option Int)= documents that the operation can fail
- Callers must handle both =None= and =(Some v)= — no silent nil propagation

See also: [[file:error-handling.org][Error Handling]]

* 5. ?? With Option

The null-coalescing operator works with =Option= values:

#+BEGIN_SRC monad
(?? (safe-div 10 0) 0)              ;; 0 — default on None
(?? (safe-div 10 2) 0)              ;; 5 — unwrapped value
#+END_SRC

#+BEGIN_SRC wisp
?? (safe-div 10 0) 0
?? (safe-div 10 2) 0
#+END_SRC

See [[file:conditionals.org][Conditionals]] for the =??= operator reference.

*** Key Points
- =??= unwraps =Some= or falls back to a default on =None=
- Type-safe: the default must match the wrapped type
- Shorter than a full =match= when a fallback value suffices

See also: [[file:conditionals.org][Conditionals]]

* 6. Related: Option in Other Languages

| Language   | Type                          |
|------------+-------------------------------|
| Monad      | =(Option a)= or =(Maybe a)=    |
| Haskell    | =Maybe a=                     |
| Rust       | =Option<T>=                   |
| OCaml      | ='a option=                  |
| F#         | =Option<'a>=                 |
| Java       | =Optional<T>=                |
| Swift      | =T?= (Optional)              |

*** Key Points
- =(Option a)= is the Monad equivalent of Rust's =Option<T>= or Haskell's =Maybe a=
- No null pointer exceptions — =None= is a first-class value, not a special sentinel
