Lists in Monad are linked cons cells at runtime — a chain of pairs where
each cell holds a value and a pointer to the rest. They are the classic
Lisp list: O(1) prepend, O(n) random access. The empty list is =nil= (also
written ='()=).

See [[file:pair.org][Pairs]] for the underlying cons-cell structure and
[[file:collections.org][Collections]] for comparison with other types.

* 1. Literal Syntax

Quoted S-expression lists are the primary construction form:

#+BEGIN_SRC monad
'(1 2 3)                ;; quote shorthand
(quote (1 2 3))         ;; explicit quote
'()                     ;; empty list (same as nil)
#+END_SRC

#+BEGIN_SRC wisp
'(1 2 3)
quote (1 2 3)
'()
#+END_SRC

The reader treats =(quote ...)= specially, producing a chain of
=AST_CONS= nodes terminating in =AST_NIL=. The entire list is allocated
at read time.

See [[file:quote.org][Quote]] for the quoting mechanism and
[[file:../context/reader.org][context/reader.org]] for AST_CONS construction.

Key Points:
- Lists are chains of cons cells terminated by =nil=
- Quoted syntax ='(1 2 3)= is the primary constructor
- Empty list ='()= is equivalent to =nil=

* 2. Core Operations

| Function  | Example                 | Result    |
|-----------+-------------------------+-----------|
| cons      | =(cons 1 '(2 3))=         | (1 2 3)   |
| car       | =(car '(1 2 3))=          | 1         |
| cdr       | =(cdr '(1 2 3))=          | (2 3)     |
| nil       | =nil=                     | ()        |
| ++        | =(++ '(1 2) '(3 4))=      | (1 2 3 4) |
| count     | =(count '(1 2 3))=        | 3         |
| empty?    | =(empty? '(1))=           | False     |
| nil?      | =(nil? nil)=              | True      |
| pair?     | =(pair? '(1 2))=          | True      |
| list      | =(list)=                  | ()        |
| make-list | =(make-list 3 'x)=        | (x x x)   |
| reduce    | =(reduce + 0 '(1 2 3))= | 6         |

See [[file:../context/codegen-ops.org][context/codegen-ops.org]] for the
operation dispatch catalogue and [[file:core.org][Core Library]] for builtins.

** 2.1 cons

Prepend an element to a list:

#+BEGIN_SRC monad
(cons 1 '(2 3))         ;; (1 2 3)
(cons 'x '())            ;; (x)
#+END_SRC

#+BEGIN_SRC wisp
cons 1 '(2 3)
cons 'x '()
#+END_SRC

Allocates a new cons cell in the arena. The original list is unchanged
— the new cell points to it (persistent data structure semantics).

See [[file:arena.org][Arena]] for allocation details.

** 2.2 ++ (append/prepend)

The =++= operator work on any collections, even lists:

#+BEGIN_SRC monad
(++ 1 '(2 3))           ;; (1 2 3) — prepend element
(++ '(1 2) '(3 4))      ;; (1 2 3 4) — append lists
#+END_SRC

#+BEGIN_SRC wisp
++ 1 '(2 3)
++ '(1 2) '(3 4)
#+END_SRC

When the first argument is not a list, it is consed onto the front of
the second. When both are lists, the first is copied and the last cdr
points to the second.

** 2.3 car / cdr

Destructure a non-empty list:

#+BEGIN_SRC monad
(car '(1 2 3))          ;; 1
(cdr '(1 2 3))          ;; (2 3)
(car (cons 'a '(b c)))  ;; a
#+END_SRC

#+BEGIN_SRC wisp
car '(1 2 3)
cdr '(1 2 3)
car (cons 'a '(b c))
#+END_SRC

Known bug: =tail= on lists segfaults due to a runtime list
implementation issue
([[file:../context/language.org::codegen-gaps][context/language.org]]).

** 2.4 nil? / pair? / empty?

Predicates for list structure:

#+BEGIN_SRC monad
(nil? '())              ;; True
(nil? '(1 2))           ;; False
(pair? '(1 2))          ;; True
(pair? 42)              ;; False
(empty? '(1))           ;; False
(empty? '())            ;; True
#+END_SRC

#+BEGIN_SRC wisp
nil? '()
nil? '(1 2)
pair? '(1 2)
pair? 42
empty? '(1)
empty? '()
#+END_SRC

=empty?= delegates to =(= count x 0)=.

See [[file:booleans.org][Booleans]] for boolean predicate semantics.

Key Points:
- =cons= allocates a new cell without modifying the original (persistent)
- =car= and =cdr= are the fundamental list destructors
- =nil?= tests for the empty list; =pair?= tests for a cons cell
- =tail= on lists has a known segfault bug (tracked in codegen-gaps)

* 3. Pattern Matching With Cons

List destructuring uses the =[x|xs]= cons-pattern:

#+BEGIN_SRC monad
(define take :: Int -> [a] -> [a]
  _ []          -> []
  n [x|xs]
    | n > 0     -> (x ++ (take (n - 1) xs))
    | otherwise -> [])

(define (length xs -> Int)
  (reduce (lambda [acc _] (+ acc 1)) 0 xs))
#+END_SRC

#+BEGIN_SRC wisp
define take :: Int -> [a] -> [a]
  _ []          -> []
  n [x|xs]
    | n > 0     -> x ++ (take (n - 1) xs)
    | otherwise -> []

define length xs -> Int
  reduce (lambda [acc _] + acc 1) 0 xs
#+END_SRC

The =[x|xs]= pattern binds the head and tail of the list. The empty
list pattern =[]= matches ='()=. List patterns can also be fixed-length:
=[a b c]= matches a list of exactly three elements.

See [[file:patterns.org][Pattern Matching]] for full pattern language and
[[file:recursion.org][Recursion]] for recursive list operations.

Key Points:
- =[x|xs]= destructures head and tail (cons pattern)
- =[]= matches the empty list
- Fixed-length =[a b c]= matches lists of exact length
- =take= example shows combined pattern + guard usage

* 4. List-as-Function Indexing

Lists can be used as functions for lookup:

#+BEGIN_SRC monad
('(10 20 30) 1)         ;; 20 — index access
(my-list 0)             ;; first element
#+END_SRC

#+BEGIN_SRC wisp
('(10 20 30) 1)
my-list 0
#+END_SRC

Returns the i-th element or =nil= if out of bounds
([[file:../codegen.c::9880]]). Indexing is O(n) — the list is traversed
from the head.

See [[file:collections.org][Collections]] for the collection-as-function protocol.

Key Points:
- Lists support O(n) index-based access via function-call syntax
- Out-of-bounds returns =nil= instead of throwing
- Prefer pattern matching for idiomatic list destructuring

* 5. Known Gap: Variadic list

=(list 1 2 3)= with multiple arguments is broken — =(car (list 1 2 3))=
returns =nil=. Only =(list)= (zero-arg) and =(make-list n val)= work
correctly
([[file:../context/language.org::codegen-gaps][context/language.org]]).

* 6. Runnable Example

#+BEGIN_SRC monad
;; run with: monadc list-demo.mon

(define (length [xs :: [a]] -> Int)
  (match xs with
    []       -> 0
    [_|rest] -> (+ 1 (length rest))))

(define (sum [xs :: [Int]] -> Int)
  (reduce + 0 xs))

(show (car '(10 20 30)))          ;; prints 10
(show (cdr '(10 20 30)))          ;; prints (20 30)
(show (length '(1 2 3 4 5)))      ;; prints 5
(show (sum '(1 2 3 4 5)))         ;; prints 15

(define (take : Int -> [a] -> [a]
  _ []          -> []
  n [x|xs]
    | n > 0     -> (x ++ (take (- n 1) xs))
    | otherwise -> []))

(show (take 3 '(a b c d e)))      ;; prints (a b c)
(show (++ '(1 2) '(3 4)))         ;; prints (1 2 3 4)
#+END_SRC

* 7. Gotchas

Common mistakes and surprising behaviours with lists:

- **=tail= on lists segfaults** — known bug tracked in codegen-gaps ([[file:../context/language.org::codegen-gaps]]).

- **=(list 1 2 3)= is broken** — the variadic =list= function with multiple arguments returns garbage. Use ='(1 2 3)= or =(cons 1 (cons 2 (cons 3 '())))= instead.

- **List indexing is O(n)** — =('(1 2 3 4 5) 3)= traverses from the head. For O(1) random access, use arrays.

- **=++= on lists copies the first list** — when both arguments are lists, the first list is copied and its last cdr is pointed at the second.

- **=empty?= is O(n)** — it delegates to =(= (count x) 0)=, which traverses the entire list. Use =nil?= for O(1) emptiness check.
