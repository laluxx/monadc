** Compilation Pipeline

#+BEGIN_SRC
source → wisp → reader → macro → infer → dep → codegen → emit
#+END_SRC

| Stage   | Source             | Description                              |
|---------+--------------------+------------------------------------------|
| Wisp    | [[file:../wisp.c]]     | Indentation-based syntax → S-expressions |
| Reader  | [[file:../reader.c]]   | S-expressions → AST nodes                |
| Macro   | [[file:../macro.c]]    | Macro calls → expanded AST               |
| Infer   | [[file:../infer.c]]    | Constraint generation and unification    |
| Dep     | [[file:../dep.c]]      | Bidirectional dependent type checking    |
| Codegen | [[file:../codegen.c]]  | AST → LLVM IR                            |
| Emit    | LLVM TargetMachine | LLVM IR → =.o=                             |

For details on each stage, see [[file:../context/build.org][context/build.org]] and [[file:../context/makefile.org][context/makefile.org]].

** Three Build Modes

The Makefile at the project root defines three build modes via target-specific
CFLAGS overrides. All three build the =monad= binary and =libmonad.a= static
archive.

#+CAPTION: Build Modes
| Mode    | Make Target      | CFLAGS                                                | Use Case                                                         |
|---------+------------------+-------------------------------------------------------+------------------------------------------------------------------|
| Debug   | =make= or =make all= | =-g -DDEBUG=                                            | Development, debugging (default)                                 |
| ASan    | =make asan=        | =-g -fsanitize=address -fno-omit-frame-pointer -DDEBUG= | Memory error detection (-fsanitize=address added to LDFLAGS too) |
| Release | =make release=     | =-DNDEBUG -O2=                                          | Production, benchmarks                                           |

- **Debug** (default): Full DWARF symbols, assertions enabled, no optimisation.
  Fastest edit-compile-test cycle, slowest runtime.
- **ASan**: AddressSanitizer catches buffer overflows, use-after-free, and memory
  leaks at runtime (~2× overhead). Run before release, especially important for
  a compiler using arena allocation (see [[file:../context/arena.org][context/arena.org]]).
- **Release**: Optimised binary with =-O2=, no debug symbols, no assertions.
  Use for production installation and benchmarking.

For the full LLVM analysis of these trade-offs, see
[[file:../context/build.org][context/build.org]].

** Verification Ladder

Every install should follow this sequence:

| Step    | Command           | Privilege | Required Before |
|---------+-------------------+-----------+-----------------|
| Clean   | =make clean=        | user      | compile         |
| Compile | =make=              | user      | install         |
| Install | =sudo make install= | root      | system use      |

Always run =make clean= after modifying compiler headers. The Makefile does not
track header dependencies automatically (see
[[file:../context/makefile.org][context/makefile.org]]).

** Test Targets

#+CAPTION: Make Test Targets
| Target         | Command                   | What It Tests                         |
|----------------+---------------------------+---------------------------------------|
| =make test=      | =python3 tests/run.py=      | Codegen + reader (atoms, sugar, lang) |
| =make test-core= | =python3 tests/run_core.py= | Core module per-module tests          |

Both targets build the compiler first (=all=) then run the test harness.
Every test fixture is compiled to a binary and executed — these are codegen
tests that exercise the full compilation pipeline.

For the architecture of the test suite, see
[[file:../context/tests-arch.org][context/tests-arch.org]].

** Linking

The =monad= binary is linked with =gcc= (CC in the Makefile). Linker flags:

#+BEGIN_SRC makefile
LDFLAGS = -lm -lreadline -lgmp $(shell llvm-config --ldflags --libs core) -lclang
#+END_SRC

- =-lm= — math library
- =-lreadline= — line editing for the REPL
- =-lgmp= — GNU Multiple Precision Arithmetic Library
- =$(shell llvm-config --ldflags --libs core)= — LLVM core libraries (version
  detected at build time)
- =-lclang= — libclang for FFI header parsing
- =-rdynamic= — runtime symbols available for FFI =dlopen=

The static runtime library =libmonad.a= (from =runtime.c= and =arena.c=,
compiled with =-fPIC=) is absorbed directly into the binary — no shared
library dependency at runtime.

For details on LLVM discovery and linker fallback behaviour, see
[[file:../context/makefile.org][context/makefile.org]].

** Install Layout

=make install= copies files under =PREFIX= (default =/usr/local=):

| Path                             | Content                   |
|----------------------------------+---------------------------|
| =PREFIX/bin/monad=                 | Compiler binary           |
| =PREFIX/lib/libmonad.a=            | Static runtime library    |
| =PREFIX/include/monad/runtime.h= | Runtime C header          |
| =PREFIX/lib/monad/core/=           | Core =.mon= library modules |

=make uninstall= reverses all these operations — removes the binary, static
library, include directory, and core module tree.

** Object Files and Caching

Compiler object files (all =.c= except =runtime.c= and =arena.c=) are compiled
via a generic pattern rule and placed alongside their sources. The =filter-out=
pattern prevents parallel-make race conditions on the runtime sources, which are
compiled with =-fPIC= for the static archive.
