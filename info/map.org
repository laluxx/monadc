Maps (hash maps / dictionaries) store key-value pairs in an
open-addressing hash table. O(1) average lookup/insert/delete. Keys are
hashed for distribution; values are boxed and stored alongside their keys.

See [[file:collections.org][Collections]] for an overview of all collection
types and [[file:../context/codegen-ops.org][context/codegen-ops.org]] for
operation dispatch.

* 1. Literal Syntax

#+BEGIN_SRC monad
#{:a 1 :b 2}            ;; (keyword <-> Int) map
#{}                     ;; empty map
#+END_SRC

#+BEGIN_SRC wisp
#{:a 1 :b 2}
#{}
#+END_SRC

Key-value pairs alternate in the literal (even node count). The reader
parses =#{...}= specially, creating the map at read time.

See [[file:symbols.org][Symbols and Keywords]] for keyword syntax and
[[file:../context/reader.org][context/reader.org]] for map literal parsing.

** 1.1 Known gap: untyped map construction

Literal maps =#{:a 1}= produce garbage at runtime without an explicit
=:: Map= annotation. This is the most significant runtime bug
([[file:../context/language.org::codegen-gaps][context/language.org]]).
Functional operations (=assoc=, =dissoc=, =merge=) receive garbage
inputs in untyped contexts. Only =assoc!= and =find= with typed
variables work correctly.

The root cause is that map literals bypass the type-directed boxing
path during codegen. A type annotation (=:: Map Key Val=) forces the
correct boxing. This is tracked as a known gap in codegen.

Key Points:
- Map literal syntax: =#{key val key val ...}=
- Empty map: =#{}=
- Untyped map literals produce garbage at runtime — always annotate =:: Map=
- The =#{...}= reader syntax is distinct from set ={}= and refiment ={x ∈ ...}=

* 2. Core Operations

| Function    | Example                  | Result       |
|-------------+--------------------------+--------------|
| assoc       | =(assoc #{:a 1} :b 2)=     | #{:a 1 :b 2} |
| assoc!      | =(assoc! m :b 2)=          | mutates m    |
| dissoc      | =(dissoc #{:a 1 :b 2} :a)= | #{:b 2}      |
| dissoc!     | =(dissoc! m :a)=           | mutates m    |
| find        | =(find #{:a 1} :a)=        | (:a 1)       |
| keys        | =(keys #{:a 1 :b 2})=      | (:a :b)      |
| vals        | =(vals #{:a 1 :b 2})=      | (1 2)        |
| merge       | =(merge #{:a 1} #{:b 2})=  | #{:a 1 :b 2} |
| contains?   | =(contains? #{:a 1} :a)=   | True         |
| count       | =(count #{:a 1 :b 2})=     | 2            |
| Map?        | =(Map? #{:a 1})=           | True         |
| collection? | =(collection? #{:a 1})=    | True         |

See [[file:core.org][Core Library]] for builtin implementations.

** 2.1 assoc / dissoc

#+BEGIN_SRC monad
(assoc #{:a 1} :b 2)            ;; #{:a 1 :b 2}
(dissoc #{:a 1 :b 2} :a)        ;; #{:b 2}
#+END_SRC

#+BEGIN_SRC wisp
assoc #{:a 1} :b 2
dissoc #{:a 1 :b 2} :a
#+END_SRC

=assoc= and =dissoc= copy the map before modifying (pure).
=assoc!= and =dissoc!= mutate in place.

See [[file:mutation.org][Mutation]] for mutable vs immutable semantics.

** 2.2 find / get

#+BEGIN_SRC monad
(find #{:a 1} :a)               ;; (:a 1) — found as pair
(find #{:a 1} :z)               ;; nil — not found
#+END_SRC

#+BEGIN_SRC wisp
find #{:a 1} :a
find #{:a 1} :z
#+END_SRC

=find= returns the key-value pair as a list or =nil= on miss.

See [[file:pair.org][Pairs]] for pair representation.

** 2.3 Map-as-Function

Maps are callable by key:

#+BEGIN_SRC monad
(define directions #{:north 0 :east 90 :south 180 :west 270})
(directions :north)             ;; 0
(directions :northwest -1)      ;; -1 with default on miss
#+END_SRC

#+BEGIN_SRC wisp
define directions #{:north 0 :east 90 :south 180 :west 270}
directions :north
directions :northwest -1
#+END_SRC

The second argument provides a default when the key is absent. Without
a default, a missing key returns =nil=.

See [[file:collections.org][Collections]] for the collection-as-function protocol.

** 2.4 keys / vals

#+BEGIN_SRC monad
(keys #{:a 1 :b 2})             ;; (:a :b)
(vals #{:a 1 :b 2})             ;; (1 2)
#+END_SRC

#+BEGIN_SRC wisp
keys #{:a 1 :b 2}
vals #{:a 1 :b 2}
#+END_SRC

Iteration order is not guaranteed (hash table traversal).

** 2.5 merge

#+BEGIN_SRC monad
(merge #{:a 1} #{:b 2})        ;; #{:a 1 :b 2}
(merge #{:a 1} #{:a 99})       ;; #{:a 99} — last wins
#+END_SRC

#+BEGIN_SRC wisp
merge #{:a 1} #{:b 2}
merge #{:a 1} #{:a 99}
#+END_SRC

Last-write-wins semantics: when duplicate keys exist, the rightmost
map's value is kept.

Key Points:
- Pure operations (=assoc=, =dissoc=, =merge=) copy the map
- Mutable operations (=assoc!=, =dissoc!=) modify in place
- =find= returns a pair =(key val)= or =nil= on miss
- Maps accept an optional default argument in function-call syntax
- =merge= uses last-wins semantics for duplicate keys

* 3. Iteration

Maps are iterable with =for=:

#+BEGIN_SRC monad
(for [[k v] #{:a 1 :b 2}]
  (show k v))                   ;; prints pairs
#+END_SRC

#+BEGIN_SRC wisp
for [[k v] #{:a 1 :b 2}]
  show k v
#+END_SRC

The destructuring pattern =[k v]= binds each key-value pair. Iteration
order is not guaranteed.

See [[file:iteration.org][Iteration]] for for-loop semantics and
[[file:../context/language.org][context/language.org]] for map iteration
dispatch.

Key Points:
- =for= over maps destructures key-value pairs with =[k v]=
- Iteration order is unspecified (hash table dependent)
- Use =keys= or =vals= to extract collections of keys or values

* 4. Runnable Example

#+BEGIN_SRC monad
;; run with: monadc map-demo.mon

;; Always annotate map literals to avoid garbage
(define m :: (Map :a Int) #{:a 1 :b 2 :c 3})

(show (count m))                  ;; prints 3
(show (find m :a))                ;; prints (:a 1)
(show (find m :z))                ;; prints nil

(define m2 (assoc m :d 4))
(show (count m2))                 ;; prints 4

(define m3 (dissoc m2 :a))
(show (count m3))                 ;; prints 3
(show (keys m3))                  ;; prints (:b :c :d)

;; Map-as-function with default
(show (m :a))                     ;; prints 1
(show (m :missing -1))            ;; prints -1

(define merged (merge m #{:d 4}))
(show (count merged))             ;; prints 4
#+END_SRC

* 5. Gotchas

Common mistakes and surprising behaviours with maps:

- **Untyped map literal =#{}= produces garbage** — always annotate =:: (Map K V)=. This is the most significant runtime bug in the compiler ([[file:../context/language.org::codegen-gaps]]).

- **=find= returns a pair, not a value** — =(find m :a)= returns =(:a 1)=, not =1=. Use =snd= or pattern matching to extract the value.

- **Iteration order is unspecified** — hash table traversal means =keys=, =vals=, and =for= produce no guaranteed ordering.

- **=merge= uses last-wins semantics** — duplicate keys keep the rightmost map's value, which may differ from other languages' behaviour.

- **Mutable ops (=assoc!=, =dissoc!=) do not copy** — aliasing the map before mutation affects all references.
