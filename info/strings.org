This page covers strings in Monad: literals, comparison, operations,
indexing, and the =Char= type.

Strings are UTF-8 encoded and immutable. Characters (see [[file:characters.org][Characters]]) are Unicode codepoints of type =Char=. The LLVM representation of a string is a global constant pointer (=i8*=) with an associated length. The runtime tag is =RT_STRING= wrapping a C =char*= and length ([[file:../context/language.org::runtime-value-system][context/language.org]]).

* 1. String Literals

#+BEGIN_SRC monad
"hello"  "line1\nline2"  "unicode: \u03C0"  "multi-line
string inside quotes"
#+END_SRC

Multi-line strings work: a newline character inside quoted text is part of the string. No line-continuation escape is needed.

Adjacent string literals (=="hello""world"=) are a compile-time syntax error — Monad does not support implicit concatenation. Use one literal or an explicit =++= operation.

See [[file:syntax.org][Syntax]] for the full string literal grammar.

* 2. Character Literals ([[file:../reader.c::1365]])

#+BEGIN_SRC monad
'a'                ;; Char 'A'
'\n'               ;; Char newline (TOK_CHAR)
'\x41'             ;; Hex escape (A)
'\u03C0'           ;; Unicode code point (π)
#+END_SRC

Delimited by single quotes (=TOK_CHAR=). Supported escape sequences: =\n=, =\t=, =\r=, =\\=, =\'=, =\0=, =\xNN=, =\uNNNN=.

See [[file:characters.org][Characters]] for full character literal documentation.

See also: [[file:characters.org][Characters]]

* 3. Char Type

The =Char= type is a 32-bit Unicode codepoint at the type-and-semantic level. Codegen lowers =TYPE_CHAR= to =i8= for ASCII-compatible codepoints. The runtime tag =RT_CHAR= distinguishes it from other scalar types. Type predicate =Char?= checks the type kind.

See [[file:../context/types.org][context/types.org]] for TYPE_CHAR, TYPE_STRING and [[file:types.org][Types]] for the ground type hierarchy.

* 4. Char Conversions

Type conversions dispatch through strcmp handlers ([[file:../codegen.c::13144]]):

#+BEGIN_SRC monad
(Char 65)           ;; 'A' — integer codepoint to Char
(Int 'A')           ;; 65  — Char to integer codepoint
(Hex 255)           ;; #xFF — integer to Hex literal
(Bin 42)            ;; #b101010 — integer to binary literal
(Oct 63)            ;; #o77 — integer to octal literal
#+END_SRC

#+BEGIN_SRC wisp
Char 65
Int 'A'
Hex 255
Bin 42
Oct 63
#+END_SRC

All five conversion targets (Int, Float, Char, Hex, Oct, Bin) are verified by test fixtures (=tests/codegen/rt_arithmetic.mon= and type-conversion corpus). See [[file:casting.org][Casting]] for the full conversion reference.

* 5. String Comparison

Strings support = and != with full value comparison (not pointer identity):

#+BEGIN_SRC monad
(= "abc" "abc")     ;; True
(!= "abc" "def")   ;; True
(= "abc" "def")    ;; False
#+END_SRC

#+BEGIN_SRC wisp
= "abc" "abc"
!= "abc" "def"
= "abc" "def"
#+END_SRC

Test fixtures: =rt_string_equality_true.mon=, =rt_string_equality_false.mon=. See [[file:equality.org][Equality]] for comparison semantics.

* 6. Operations

String operations have strcmp dispatch handlers and are callable from paren-mode ([[file:../context/language.org::working-dispatch][context/language.org]]):

| Function     | Example                       | Returns               |
|--------------+-------------------------------+-----------------------|
| show         | =(show 42)=                   | "42"                  |
| make-string  | =(make-string 5 'a')=         | "aaaaa"               |
| count        | =(count "hello")=             | 5                     |
| starts-with? | =(starts-with? "hi" 'h')=     | True                  |
| ends-with?   | =(ends-with? "hi" 'i')=       | True                  |
| contains?    | =(contains? "hello" "ell")=   | True                  |
| ++           | =(++ "hello" " world")=       | "hello world"         |
| head         | =(head "hello")=              | 'h'                   |
| tail         | =(tail "hello")=              | "ello"                |
| take         | =(take 3 "hello")=            | "hel"                 |
| drop         | =(drop 2 "hello")=            | "llo"                 |

#+BEGIN_SRC monad
(show 42)              ;; "42"
(make-string 5 'a')    ;; "aaaaa"
(count "hello")        ;; 5
(starts-with? "hi" 'h') ;; True
(ends-with? "hi" 'i')  ;; True
(contains? "hello" "ell") ;; True
(++ "hello" " world")  ;; "hello world"
(head "hello")         ;; 'h'
(tail "hello")         ;; "ello"
(take 3 "hello")       ;; "hel"
(drop 2 "hello")       ;; "llo"
#+END_SRC

#+BEGIN_SRC wisp
show 42
make-string 5 'a'
count "hello"
starts-with? "hi" 'h'
ends-with? "hi" 'i'
contains? "hello" "ell"
++ "hello" " world"
head "hello"
tail "hello"
take 3 "hello"
drop 2 "hello"
#+END_SRC

Test fixtures: =rt_string_helpers.mon= (=count=), =rt_string_concat.mon= (=++=), =rt_string_prefix_suffix.mon= (=starts-with?=, =ends-with?=).

The =count= function works uniformly on strings, lists, arrays, sets, and maps (see [[file:collections.org][Collections]]).

Key Points:
- Strings are UTF-8 encoded, immutable, with LLVM =i8*= global constants
- =show= converts any value to its string representation
- =++= concatenates strings; =head=, =tail=, =take=, =drop= treat strings as sequences
- =count= works uniformly on all collection types
- String indexing via =("str" i)= or =xs[i]= returns a =Char= with bounds checking

See also: [[file:collections.org][Collections]]

* 7. Substring

=substring= is registered as a builtin (=env_insert_builtin= with arity 3) but has no codegen strcmp handler yet — it is in the same gap category as =concat=, =reverse=, and =nth= ([[file:core.org][Core Library — NOT callable]]). Use =take= and =drop= for substring-like operations in the meantime.

* 8. String Indexing ([[file:../codegen.c::9732]])

Uses function-call syntax =("string" i)=. Returns a =Char= at index =i=.
Also postfix bracket: =xs[0]= ([[file:../reader.c::6621]]). Both forms compile to a character load from the string's internal buffer with bounds checking.

#+BEGIN_SRC monad
("hello" 0)   ;; 'h'
("hello" 1)   ;; 'e'
#+END_SRC

#+BEGIN_SRC wisp
("hello" 0)
("hello" 1)
#+END_SRC

See [[file:characters.org][Character Operations — String Indexing]].

* 9. String Interpolation

Planned but not implemented. Future syntax may embed values inside string literals using =_= placeholders or =${}= syntax.

* 10. Strings as Collections

Strings participate in the collection-as-function model. Operations like =head=, =tail=, =count=, =++=, =take=, =drop= work on strings uniformly. Iteration via =for= is supported:

#+BEGIN_SRC monad
(for [c "hello"]
  (show c))               ;; prints 'h', 'e', 'l', 'l', 'o'
#+END_SRC

#+BEGIN_SRC wisp
for [c "hello"]
  show c
#+END_SRC

See [[file:collections.org][Collections]] for the collection protocol and [[file:iteration.org][Iteration]] for =for= loops.

* 11. Wisp Char Escape Limitation

In Wisp syntax, ='\n'= fails because the Wisp expander expands =\\n= to a real newline, splitting the literal across lines. Use S-expression syntax for character escape sequences involving backslash.

See [[file:wisp.org][Wisp]] for details on the Wisp expander.

See also: [[file:wisp.org][Wisp]]
