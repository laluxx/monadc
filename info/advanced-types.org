* Type Classes

Monad provides dictionary-passing type classes ([[file:../typeclass.c]]).
A =class= declaration defines method signatures; an =instance= provides
concrete implementations. Each instance compiles to an LLVM global
dictionary struct:

#+BEGIN_SRC monad
(class Eq a (== (a a -> Bool)))
(instance Eq Int
  (define (== a b -> Bool) (builtin-== a b)))
#+END_SRC

#+BEGIN_SRC wisp
class Eq a
  == (a a -> Bool)
instance Eq Int
  define == a b -> Bool
    builtin-== a b
#+END_SRC

Deriving auto-generates instances for algebraic data types:

#+BEGIN_SRC monad
(data Suit Hearts Diamonds Clubs Spades deriving Eq)
#+END_SRC

#+BEGIN_SRC wisp
data Suit Hearts Diamonds Clubs Spades deriving Eq
#+END_SRC

* Dependent Types (ITT/NbE)

Intensional Type Theory with Normalization by Evaluation runs as a
shadow pass (Phase 6.5) between feature setup and LLVM codegen. It
validates dependent annotations and back-propagates ground types to
the AST:

#+BEGIN_SRC monad
(define Vec :: Π (n : Nat) → Type → Type ...)
#+END_SRC

#+BEGIN_SRC wisp
define Vec :: Π (n : Nat) → Type → Type ...
#+END_SRC

The NbE kernel provides =dep_eval=, =dep_quote=, =dep_normalise=, and
=dep_conv= (definitional equality). Bidirectional checking follows
Löh/McBride/Swierstra (2010).

→ [[file:dependent-types.org][Full Dependent Types Reference]]

* Type Inference (Hindley-Milner)

Algorithm W with let-polymorphism and monomorphization. Type variables
are implicitly quantified:

#+BEGIN_SRC monad
(define (id x -> a) x)             ;; ∀a. a → a
#+END_SRC

#+BEGIN_SRC wisp
define id x -> a
  x
#+END_SRC

Polymorphic functions specialize at each use site via the
=mono_cache=, generating zero-overhead native code.

→ [[file:type-inference.org][Full Type Inference Reference]]

* Refinement Types

Subset types with predicate constraints:

#+BEGIN_SRC monad
(type Pos { x ∈ Int | (> x 0) })
(type Even { x ∈ Int | (= (mod x 2) 0) })
#+END_SRC

#+BEGIN_SRC wisp
type Pos { x ∈ Int | > x 0 }
type Even { x ∈ Int | = (mod x 2) 0 }
#+END_SRC

Checked at compile time for literal arguments, deferred to runtime
otherwise.
