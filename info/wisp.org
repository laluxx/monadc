[[file:index.org][<- Back to Index]] · [[file:advanced-wisp.org][Advanced Wisp]] · [[file:syntax.org][S-expression Syntax]]

Wisp is an indentation-aware surface syntax for Monad
([[file:../wisp.c]]). It expands to standard S-expressions in a pre-pass before
the reader — both produce identical ASTs and mix freely in the same file
([[file:../context/wisp.org::wisp-architecture]]).

* How It Works

The wisp expander ([[file:../wisp.c]]) runs as a two-pass pre-pass before the
reader:

1. Tokenization: split source into tokens (parens, brackets, symbols, strings,
   numbers, etc.). Balanced brackets are preserved as single tokens.
2. Layout expansion: use indentation and arity tables to insert parentheses
   where needed, converting indentation-driven blocks into S-expression trees.

Pipeline within =wisp_parse_all= ([[file:../wisp.c::4696]]):

1. Import/builtin arities registered with =wisp_register_arity=
   ([[file:../context/wisp.org::wisp-api-functions]]).
2. Source stripped of comments preserving line/column positions.
3. 2D fraction desugaring (=desugar_fractions=).
4. Heuristic scan: if source contains no wisp-style lines (non-paren leading
   tokens), short-circuit directly to =parse_all=.
5. Build a local ArityTable seeded from the global FFI arity registry.
6. Arity pre-scan registers user-defined =data=/=type=/=layout=/=define= arities
   ([[file:../wisp.c::107]]).
7. Token stream built line-by-line via =build_token_stream= with indent tracking.
8. All top-level expressions parsed into S-expression strings.
9. Param-kind hook and known-function hook installed for the reader.
10. =parse_all= + =macro_expand_all= called on the transformed output.

The output is a flat list of standard S-expressions that feed directly into the
reader. The reader has no awareness that the source was originally wisp.

Arity registration order matters: FFI arities from =include= headers are
registered in Phase 0 (FFI pre-pass) before Phase 1 (wisp expansion), so that
imported C functions get correct layout inference
([[file:../context/wisp.org::inf.wisp.ffi-arity-flow]]).

* Basic Rules

Indentation determines grouping:

| Indent   | Meaning              |
|----------+----------------------|
| Same     | Sibling forms        |
| Deeper   | Child form           |
| Shallower| End of current form  |

No outer parens: =show 42= expands to =(show 42)=.

Mix at any line: lines starting with =(=, =[=, ={=, =#{} pass through verbatim
as S-expressions.

* Function Calls

#+BEGIN_SRC wisp
show 42                    ;; (show 42)
+ 1 2                      ;; (+ 1 2)
x < 10                     ;; (< x 10)
#+END_SRC

Infix operators auto-convert via arity-driven Pratt parsing:

|  Precedence | Operators                    |
|-------------+------------------------------|
|  1 (lowest) | =or=                           |
|           2 | =and=                          |
|           3 | ==== =!= =<= =<== =>== =>=               |
|           4 | =+= =-=                          |
|           5 | =*= =/=                          |
| 6 (highest) | Everything else (calls, =^=, =%=) |

Group with parens: =x * + 1 2= → =(* x (+ 1 2))=.

* Arity Table

The arity table maps function names to parameter counts, enabling the layout
expander to parenthesize indented arguments correctly
([[file:../context/wisp.org::wisp-arity-table]]). A known function with arity N
consumes the next N indented forms as its arguments; unknown forms fall back to
generic layout rules.

The arity table is a fixed-size hash map with 128 buckets, using the djb2 hash
function ([[file:../context/wisp.org::wisp-arity-table-internal]]). Two
instances exist:

- Global =g_ffi_arities=: persistent across modules, populated by FFI pre-pass
  and env-builtin registration.
- Local table in =wisp_parse_all=: seeded from the global table plus builtins
  and the arity pre-scan. Newly discovered arities are merged back into the
  global table at the end of =wisp_parse_all= to benefit subsequent modules.

Arity sources ([[file:../context/wisp.org::obs.wisp.arity-sources]]):

- Builtin functions: =wisp_register_arities_from_env= scans the env for
  registered functions and inserts their param counts.
- FFI C headers: libclang-parsed function signatures from =include= directives
  register their arities during the Phase 0 pre-pass.
- Layout constructors: struct field counts from =layout= declarations and FFI
  structs registered as constructor arities.

* Definitions

** Simple value

#+BEGIN_SRC wisp
define x 42                 ;; (define x 42)
#+END_SRC

** Typed value (two forms)

#+BEGIN_SRC wisp
define x :: Int 42          ;; wisp sugar: (define [x :: Int] 42)
define [x :: Int] 42        ;; bracketed (passes through)
#+END_SRC

** Function with params (no type signature)

#+BEGIN_SRC wisp
define square x -> * x x    ;; (define (square x) (* x x))
define twice f x -> f (f x) ;; higher-order
#+END_SRC

** Typed function (signature + clause)

#+BEGIN_SRC wisp
define square :: Int -> Int
  x -> x * x                ;; (define (square [x :: Int] -> Int) (* x x))
#+END_SRC

The expander desugars simple indented arrow clauses (like =x -> x * x=) to a
named-parameter function header and a direct body, binding =x= as the function
parameter ([[file:../context/wisp.org::obs.wisp.define-signature-clause-fixed]]).
Non-simple clauses containing =_=, literals, collection patterns like =[x|xs]=,
or guards keep the pattern-clause path. In some cases, multi-line =define= with
=->= arrows and guard conditions can produce incorrect lambda bindings because
the simple-clause heuristic misclassifies the parameter.

** Parenthesized signature + clause

#+BEGIN_SRC wisp
define (f [n :: Int] [xs :: Arr] -> Int)
  (f (dec n) xs)            ;; full lisp-style header, wisp body
#+END_SRC

** Zero-param function

#+BEGIN_SRC wisp
define greet -> String "hello"  ;; (define (greet -> String) "hello")
#+END_SRC

** Array declarations

#+BEGIN_SRC wisp
define buf :: [512]         ;; (define [buf :: Arr :: 512 :: U8] [])
define buf :: [16kb I32]    ;; (define [buf :: Arr :: 4096 :: I32] [])
#+END_SRC

* Arrow Clauses

=->= introduces the body. =→= (U+2192) and =↦= (U+21A6) also work.

#+BEGIN_SRC wisp
define fact
  0 -> 1
  n -> * n (fact (- n 1))
#+END_SRC

Guards:

#+BEGIN_SRC wisp
(define sign
  n | > 0 -> "pos"
  n | < 0 -> "neg"
  _       -> "zero")
#+END_SRC

Inline lambda sugar:

#+BEGIN_SRC wisp
[x y] => + x y             ;; (lambda ([x] [y]) (+ x y))
#+END_SRC

* Type Annotations

Both =::= and =:= work as annotation separators:

| Form                             | Result                                     |
|----------------------------------+--------------------------------------------|
| =define x :: Int 42=               | =(define [x :: Int] 42)=                     |
| =define [x :: Int] 42=             | =(define [x :: Int] 42)=                     |
| =define f :: Int -> String -> Ret= | registers arity 2, each arrow adds a param |
| =*U8=                              | =[Pointer :: U8]=                            |
| =**U8=                             | =[Pointer :: Pointer :: U8]=                 |
| ==(a -> b)= inside signature       | marks param as function type               |

* Type Annotations in Brackets

=define [x :: Int] 42= — typed value with bracket annotation.
=define f [a :: Int] [b] -> ...= — typed function params.

* Special Syntactic Sugar

** Pipe

#+BEGIN_SRC wisp
5 |> double |> add1          ;; (add1 (double 5))
#+END_SRC

Pipe chains use the arity table to determine correct grouping.

** For loops

#+BEGIN_SRC wisp
for c -> iterable            ;; forward: (for [i 0 (count iterable)] (define c (iterable i)) ...)
  body
for c <- iterable            ;; backward: (for [i (- (count iterable) 1) -1 -1] ...)
  body
#+END_SRC

** Where clauses

#+BEGIN_SRC wisp
define f x -> body
  where result = expr        ;; (let ([result expr]) body)
#+END_SRC

** if expressions (three forms)

#+BEGIN_SRC wisp
;; Form 1 — indented, else keyword
if (> x 0)
  "pos"
else
  "neg"

;; Form 2 — then/else at same indent
if (> x 0)
then "pos"
else "neg"

;; Form 3 — fully inline
if (> x 0) then "pos" else "neg"
#+END_SRC

Multi-line bodies wrap in =(begin ...)=. Missing branches fill with
=(undefined)=.

** ?-sugar (method-call style)

#+BEGIN_SRC wisp
(contains? s 2)               ;; when s is untyped → (s contains? 2)
#+END_SRC

Only fires when the first argument is a bare untyped symbol
([[file:../context/wisp.org::obs.wisp.qmark-sugar]]). Literals and function
calls as the first argument bypass the transform.

** !-sugar (mutation-call style)

#+BEGIN_SRC wisp
(conj! s 99)                  ;; when s is untyped → (s conj! 99)
#+END_SRC

Applies to =conj!=, =disj!=, =append!=, and any future !=-suffixed functions
([[file:../context/wisp.org::obs.wisp.bang-sugar]]). Same heuristic as
?-sugar: fires only when the first argument is a bare, untyped symbol.
Type-annotate to suppress: =define [s :: Set] (conj! s 99)=.

** @-sugar (indexing shorthand)

Array/list indexing via =@= syntax
([[file:../context/wisp.org::obs.wisp.at-sugar]]).

** Fraction sugar (2D)

#+BEGIN_SRC wisp
 1                            ;; ─ line between numerator and denominator
───                           ;; becomes (/ 1 2)
 2
#+END_SRC

* Data Types, Type Classes, and More

=layout=, =data=, =type=, =class=, =instance=, =match= all work as wisp
blocks, collected as single S-expression tokens:

#+BEGIN_SRC wisp
layout Point
  x :: Float
  y :: Float

data Option a
  None
  Some a

type Positive { x ∈ Int | (> x 0) }

match x with
  (Just v) -> show v
  Nothing  -> show "none"
#+END_SRC

* Mixing Wisp and S-expressions

Any line can switch styles mid-stream:

#+BEGIN_SRC wisp
define add x y -> x + y
(show (add 1 2))             ;; raw s-expr where it's clearer
define double x -> * x 2     ;; back to wisp
#+END_SRC

Comments work in both:

#+BEGIN_SRC wisp
;; this is a line comment
;|
multi-line
block comment
|;
#+END_SRC

* Known Limitations

| Bug                                                       | Workaround                                |
|-----------------------------------------------------------+-------------------------------------------|
| Char literals with escapes ='\n'= split across lines      | Use S-expression for escaped chars        |
| =:alias= / =:doc= metadata emitted as separate forms       | Use S-expression for metadata             |
| Wisp =->= =define= + guards (=| condition=) not supported | Use S-expression for guarded defines      |
| =..= path prefix breaks path literal (range operator)     | Use absolute =/= or home =~/= paths        |
| Multi-line =define= with =->= and guards can produce      | Use parenthesized Lisp-style define header|
| incorrect lambda bindings (simple-clause heuristic        | or S-expression for complex clauses       |
| misclassifies parameters)                                  |                                           |
| ?-sugar/!-sugar fires on untyped vars causing unintended  | Type-annotate the variable                 |
| method-call transforms                                     |                                           |

See [[file:advanced-wisp.org][Advanced Wisp]] for details and more edge cases.
