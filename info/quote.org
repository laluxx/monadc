Test coverage: quote and quasiquote are exercised by codegen fixtures
=rt_quote_*.mon= and =rt_quasiquote_*.mon=. The test architecture is documented at
[[file:../context/tests-arch.org][context/tests-arch.org]].

Quote and quasiquote prevent evaluation, treating the expression as
literal data. They are the foundation of code-as-data macros
([[file:macros.org][Macros Reference]]).

* 1. quote

Prevents evaluation of the following expression:

#+BEGIN_SRC monad
(quote (1 2 3))                  ;; '(1 2 3) ‚Äî list literal
'(1 2 3)                         ;; same, shorthand syntax
'x                               ;; symbol x (not variable lookup)
'()                              ;; empty list
#+END_SRC

#+BEGIN_SRC wisp
quote (1 2 3)
'(1 2 3)
'x
'()
#+END_SRC

The shorthand ='expr= is reader sugar for =(quote expr)=. The reader
produces =AST_QQUOTE= nodes for quote ([[file:../reader.h][reader.h]],
[[file:../context/reader.org][context/reader.org]]).

** 1.1 Use Cases

- Create list literals: ='(a b c)=
- Refer to symbols as data: ='my-symbol=
- Prevent macro expansion: ='(if x y z)=

#+BEGIN_SRC monad
(define my-list '(1 2 3))
(car my-list)                    ;; 1
(define s 'hello)                ;; symbol, not variable reference
#+END_SRC

#+BEGIN_SRC wisp
define my-list '(1 2 3)
car my-list
define s 'hello
#+END_SRC

See [[file:list.org][Lists]] for quoted list literals and [[file:symbols.org][Symbols]]
for symbol type details.

* 2. quasiquote

Template with unquote holes. Uses corner bracket syntax:

#+BEGIN_SRC monad
‚åú1 2 3‚åù                       ; quasiquote = (quasiquote 1 2 3)
‚åú1 2 ‚åûx‚åü 3‚åù                   ; unquote: ‚åûexpr‚åü = (unquote expr)
#+END_SRC

#+BEGIN_SRC wisp
‚åú1 2 3‚åù
‚åú1 2 ‚åûx‚åü 3‚åù
#+END_SRC

=‚åú= / =‚åù= delimit the quasiquote (U+231C / U+231D).
=‚åû= / =‚åü= delimit unquote holes (U+231E / U+231F).

No backtick syntax ‚Äî use corner brackets.

** 2.1 Unquote Semantics

Inside a quasiquote, =‚åûexpr‚åü= evaluates =expr= and substitutes its
value into the template:

#+BEGIN_SRC monad
(define x 42)
‚åú1 ‚åûx‚åü 3‚åù                    ;; (1 42 3)
#+END_SRC

#+BEGIN_SRC wisp
define x 42
‚åú1 ‚åûx‚åü 3‚åù
#+END_SRC

* 3. unquote-splicing

Splice a list into the surrounding template:

#+BEGIN_SRC monad
(define xs '(2 3))
‚åú1 ‚åû,@xs‚åü 4‚åù                   ;; (1 2 3 4)
#+END_SRC

#+BEGIN_SRC wisp
define xs '(2 3)
‚åú1 ‚åû,@xs‚åü 4‚åù
#+END_SRC

=‚åû,@xs‚åü = =(unquote-splicing xs)= ‚Äî flattens the list into the
surrounding template. The =,@= prefix distinguishes splicing from
plain unquote.

* 4. Macro Usage

Quasiquote is the primary macro output template
([[file:macros.org][Macros Reference]]):

#+BEGIN_SRC monad
(define (when condition . body -> Syntax)
  ‚åúif condition (begin ‚åû,@body‚åü)‚åù)
#+END_SRC

#+BEGIN_SRC wisp
define (when condition . body -> Syntax)
  ‚åúif condition (begin ‚åû,@body‚åü)‚åù
#+END_SRC

Inside the quasiquote, unquoted expressions (=‚åûexpr‚åü=) are evaluated
at macro-expansion time and spliced into the output. Compare with the
=#()= quasi-output syntax ([[file:macros.org][Macros ‚Äî #() Quasi-Output Syntax]]),
which uses a different mechanism:

- =‚åú‚Ä¶‚åûexpr‚åü‚Ä¶‚åù= ‚Äî quasiquote: evaluates at expansion time and substitutes.
- =#(‚Ä¶)= ‚Äî quasi-output: marks template region; code outside is future compile-time eval.

#+BEGIN_SRC monad
(define (greet name -> Syntax)
  ‚åú(string-append "Hello, " ‚åûname‚åü)‚åù)   ;; string-append at runtime
#+END_SRC

#+BEGIN_SRC wisp
define greet name -> Syntax
  ‚åú(string-append "Hello, " ‚åûname‚åü)‚åù
#+END_SRC

See [[file:macro-patterns.org][Macro Patterns]] for more macro templates using quasiquote
and [[file:../context/macro.org][context/macro.org]] for implementation details.

* 5. Reader Implementation

The reader handles quote and quasiquote in
[[file:../reader.c][reader.c]] ([[file:../reader.c::1828][reader.c:1828-1839]]):

- Quote shorthand ='x= produces =AST_QQUOTE= nodes.
- Quasiquote corner brackets =‚åú‚Ä¶‚åù= produce =AST_QQUOTE= with sub-nodes.
- Unquote =‚åûexpr‚åü= produces =AST_UNQUOTE=.
- Unquote-splicing =‚åû,@expr‚åü= produces =AST_UNQUOTE_SPLICE=.

Reader token types are defined in [[file:../reader.h][reader.h]]. See
[[file:../context/reader.org][context/reader.org]] for the full reader architecture.

Implementation Notes

Quote and quasiquote are handled primarily in the reader and macro
expander:

- **Reader nodes**: the shorthand ='x= produces =AST_QQUOTE= with the
  quoted expression as a child node. Corner brackets =‚åú‚Ä¶‚åù= produce
  =AST_QQUOTE= as well, with =AST_UNQUOTE= and =AST_UNQUOTE_SPLICE= for
  the unquote and splice forms
  ([[file:../context/reader.org][context/reader.org ‚Äî quote reader nodes]]).
- **Reader parsing**: quote is parsed at =reader.c:1828-1839= where the
  reader recognises ='= and wraps the next expression in a quote node
  ([[file:../reader.c::1828][reader.c ‚Äî quote parsing]]).
- **Quasiquote unicode**: corner brackets =‚åú‚åù= (U+231C/U+231D) and
  =‚åû‚åü= (U+231E/U+231F) are recognised as multi-character delimiters by
  the reader, not single tokens
  ([[file:../context/reader.org][context/reader.org ‚Äî corner bracket handling]]).
- **Macro integration**: quasiquote templates in macro bodies are
  expanded by the macro expander which evaluates =‚åûexpr‚åü= forms and
  substitutes their values into the output
  ([[file:../context/macro.org][context/macro.org ‚Äî quasiquote in macros]]).
- **#() quasi-output difference**: =#(...)= (macro quasi-output) is a
  separate mechanism from =‚åú‚Ä¶‚åû‚Ä¶‚åü‚Ä¶‚åù= (reader quasiquote). The former
  marks a template region; the latter reconstructs S-expressions with
  evaluated holes.

Key Points

- ='expr= is reader sugar for =(quote expr)= ‚Äî prevents evaluation.
- =‚åútemplate ‚åûhole‚åü rest‚åù= is quasiquote with unquote holes.
- =‚åû,@list‚åü= splices a list into the surrounding template.
- Corner brackets (=‚åú‚åù=, =‚åû‚åü=) are the quasiquote syntax ‚Äî no backtick.
- Quasiquote is the primary macro output template mechanism; =#()= is an
  alternative macro-specific syntax.
- At runtime, quoted data produces literal symbols, lists, and atoms.
- Macro integration: unquoted holes in macro templates are evaluated at
  expansion time.
