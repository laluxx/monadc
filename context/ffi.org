[OBS id:obs.context.ffi.context-struct src:ffi.h:64 conf:high]
  =FFIContext= holds growable arrays of =FFIFunction= (=function_count= /
  =function_cap=), =FFIConstant= (=constant_count= / =constant_cap=),
  =FFIStruct= (=struct_count= / =struct_cap=).  Also tracks already-included
  headers (=included= / =included_count= / =included_cap=) to prevent
  re-parsing, =strip_prefixes= for the =:unprefix= option, and per-module
  injection watermarks (=injected_into=, =wm_functions=, =wm_structs=,
  =wm_constants=) to support incremental injection across compilation units.

  at the given absolute path.  =system_include= distinguishes <=header.h>=

[THINK id:think.context.ffi.parse-header date:2026-06-13 by:codex]
  Implementation uses libclang (or an embedded C preprocessor) to walk the
  AST and extract function declarations, macro definitions, enum values,
  and struct definitions.  The =system_include= flag influences include-path
  resolution and may affect which declarations are visible.  Deduplication
  against the =included= array prevents re-parsing the same header.

[THINK id:think.context.ffi.map-type-precision date:2026-06-13 by:codex]

[THINK id:think.context.ffi.cache-invalidation date:2026-06-13 by:codex]
  Cache validity is determined by comparing the header file's mtime against
  the cache file's mtime.  If the header is newer the cache is invalidated
  and =ffi_parse_header= is re-run.  The cache format is a binary dump of
  the serialised =FFIFunction=/=FFIConstant=/=FFIStruct= arrays.

[DOC id:doc.ffi.category-contract src:context/ffi.org conf:high]
  This section declares the category contract for =ffi=. It is the
  routing target for failures and future test/context links touching C FFI
  header parsing, type mapping, arity registration, and cache management.

#+CAPTION: C FFI Category Contract
| Field | Contract |
|-------+----------|
| Owned phase | C header parsing and FFIContext construction (Phase 0 pre-pass), FFI type mapping from C types to Monad types, function arity registration for Wisp, cache serialization/deserialization. |
| Inputs | C header files via (include ...) directives, FFIContext for parsed header state, header file modification times for cache invalidation. |
| Outputs | EnvEntry for FFI functions/constants/structs, arity table entries registered with wisp_register_arity, g_ffi_link_libs accumulated for final link step, cached FFIContext snapshots. |
| Invariants | FFI cache is invalidated when header mtime changes; arity registration completes before Wisp expansion begins; FFI functions are injected with is_ffi=true marker; header parse errors are reported but non-fatal. |
| Test hooks | FFI header parse fixtures, include/parse round-trip tests, cache invalidation tests, arity registration coverage. |
| Failure symptoms | Header parse failure, C type to Monad type mapping mismatch, cache staleness (header newer than cache), arity registration collision, missing link library. |
| Context neighbors | wisp, reader, codegen-ops, language, env, buildsystem |

[DEC id:dec.ffi.category-contract-owner date:2026-06-18 by:big-pickle]
  Failures matching the symptoms above should start triage in this category, then
  follow the listed neighbors only when the local contract does not explain the
  issue.
