[DEC id:dec.language.wisp-bang-sugar date:2026-06-12 by:codex]
   The Wisp desugaring treats !-suffixed symbols as method-call sugar when the
   first argument is an untyped symbol: =(conj! s 3)= → =(s conj! 3)=. This is
   by design for method-call syntax but silently breaks calls to !-suffixed
   builtins when the variable has no type annotation. The workaround is to bind
   the set/list variable with a =:: Set= / =:: List= annotation, which prevents
   the Wisp sugar from firing. This affects =conj!=, =disj!=, =append!=, and
   any future !-suffixed builtins.
