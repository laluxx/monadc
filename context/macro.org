[OBS id:obs.context.macro.syntax-convention src:macro.h:7 conf:high]
  Macros are ordinary =define= forms whose lambda declares =-> Syntax= as
  the return type.  The expander registers them by scanning all top-level
  defines before the main expansion pass.  Examples:
  =(define (when condition . body -> Syntax) (if condition (begin . body) (begin)))=
  =(define (swap! a b -> Syntax) (let [tmp a] (set! a b) (set! b tmp)))=

  define syntax) ·
  `monadc.context.language.type-system-surface` (Syntax is a first-class
  type in the surface system)

[OBS id:obs.context.macro.quasi-output src:macro.h:45 conf:high]
  If the macro body contains =#(...)= the expander strips the marker and
  only the inner subtree becomes the macro output.  If no =#(...)= appears,
  the entire body is the output template.  Example:
  =(define (assert condition message -> Syntax)
    (if debug-mode
      #(when (not condition) (error message))
      #(begin)))=

[THINK id:think.context.macro.quasi-output-eval date:2026-06-13 by:codex]
  Full compile-time evaluation of code outside =#()= is TODO.  Currently the
  whole body is substituted and =#()= is stripped, so conditionals like
  =(if debug-mode ...)= are not evaluated at compile time — both branches
  survive into the output.  Future work: evaluate the non-=#()= portion at
  expansion time.

[OBS id:obs.context.macro.hygiene src:macro.h:36 conf:high]
  Names introduced inside a macro body (lambda params, =let= bindings, inner
  =define=s) that are NOT pattern variables are renamed to the pattern:
  =macroname__varname__N=, where N is a global monotonic counter.  Pattern
  variables and free references to outer definitions pass through unchanged.
  This prevents unintentional capture between macro-generated code and the
  surrounding scope.

[THINK id:think.context.macro.hygiene-counter date:2026-06-13 by:codex]
  The global counter N ensures uniqueness across all macro invocations
  within a single compilation unit.  The separator =__= (double underscore)
  is reserved by the compiler and cannot appear in user-written identifiers.
  On reset between units, =macro_clear()= resets the counter.

[OBS id:obs.context.macro.variadic-splicing src:macro.h:53 conf:high]
  A rest parameter (=is_rest=) captures zero or more trailing arguments.  In
  the template, if the rest name appears as a list element the captured nodes
  are *spliced* (inserted flat into the enclosing list).  In non-list position
  a single-element rest wraps the captured nodes in =(begin ...)=.

[THINK id:think.context.macro.rest-splicing-edge src:macro.h:55 conf:high]
  Splicing semantics: =(my-and . rest)= → =(if x (my-and . rest) False)=
  where =rest= splatted into the =if= tail produces N-ary chaining.
  Single-element capture in non-list position (e.g. =(list . rest)= with one
  arg) wraps in =(begin captured)= to preserve the single expression.

[OBS id:obs.context.macro.fixpoint src:macro.h:32 conf:high]
  After the initial registration pass, =macro_expand_all= walks every
  remaining AST, expands macro call sites, and repeats the walk to a
  fixpoint capped at 1 024 rounds.  This ensures macros that expand into
  other macro calls are fully resolved.  If the cap is reached the
  expansion is aborted (the compiler reports an error or returns the
  partially-expanded result).

[OBS id:obs.context.macro.func-expand-all src:macro.h:78 conf:high]
  =macro_expand_all(exprs, count)= — Entry point.  Takes a flat list of
  top-level ASTs from =parse_all= / =wisp_parse_all=.  Pipeline:
  1. Scan for defines whose lambda has =return_type= ="Syntax"= → register
     as macros and remove from the output (macros are compile-time only).
  2. Walk every remaining AST calling =macro_try_expand= at each list node.
  3. Repeat step 2 to fixpoint (capped at 1024 rounds).  Returns a new
     =ASTList=.  Caller owns the returned exprs array; individual AST nodes
     are live.  Macro define nodes consumed in step 1 are freed internally.

[THINK id:think.context.macro.expand-ownership date:2026-06-13 by:codex]
  The returned =ASTList= contains live AST nodes that are either original
  (non-macro) or freshly allocated for the macro output.  The caller must
  =free()= the array but must not free individual AST nodes that belong to
  the original input.  A flag or side table distinguishes owned vs borrowed
  nodes if needed.

[THINK id:think.context.macro.pattern-variables date:2026-06-13 by:codex]
  Pattern variables are the formal parameters of the macro lambda.  During
  expansion, actual arguments are substituted for pattern variables in the
  template.  Non-pattern variables (let bindings, inner lambdas) are
  hygienically renamed.  The current implementation does not support
  ellipsis ... patterns (Scheme-style) — variadic behaviour is via a single
  rest parameter.

[THINK id:think.context.macro.expansion-order date:2026-06-13 by:codex]
  Expansion is outermost-first: the expander walks the AST top-down and
  expands the outer list first, then recurses into the result.  This means
  a macro that produces another macro call will have that call expanded
  in a subsequent fixpoint round, not inline during the same walk.
