[DOC id:doc.garden.charter src:context/garden.org conf:high]
  The garden exists to make compiler work safer: a reader can move from a test
  failure to a language contract, from a source file to its design pressure, and
  from a decision to the tests that protect it.

[DEC id:dec.garden.no-decorative-context date:2026-06-16 by:llm]
  Context that is not connected to source, tests, decisions, or active design
  questions should be pruned or moved to an explicit archive. Decorative context
  is worse than no context because it creates false confidence.

#+CAPTION: Evidence classes
| Level | Record | Meaning | May use high confidence? |
|-------+--------+---------+--------------------------|
| Source observation | =OBS= | Read directly from source, test output, or command output | yes |
| User decision | =DEC= | Explicit project direction from the user | yes |
| Derived inference | =INF= | Reasoned consequence of observations/decisions | only with source links |
| Task | =TODO= | Work that should happen | yes for existence, not outcome |
| Opinion | =THINK= | Judgment, risk, or interpretation | usually medium |
| Proposal | =IDEA= | Not accepted unless promoted to =DEC= | medium/low |

[DEC id:dec.garden.obs-is-not-opinion date:2026-06-16 by:llm]
  Never put an opinion into =OBS=. If a record says what code does, cite the file
  and line range when known. If it says what code should do, use =DEC=, =TODO=,
  or =IDEA=.

#+CAPTION: Mutation policy
| Operation | Rule |
|-----------+------|
| Add | Use stable IDs and put the node in the nearest topic file. |
| Modify | Update =#+LAST_MODIFIED= and =:CONTEXT_UPDATED:= on touched nodes. |
| Delete | Delete only stale/generated/noisy content or records superseded elsewhere. |
| Supersede | Prefer a new =DEC= or =FIX= record naming the superseded record. |
| Move | Preserve the ID when moving a durable heading. |
| Split | Add a bridge link in the old file to the new file. |

[DEC id:dec.garden.append-or-supersede date:2026-06-16 by:human+llm]
  For architectural decisions, prefer append-or-supersede over in-place rewrite.
  For factual source observations that were simply wrong or stale, rewrite and
  mark the correction with a =FIX= record.

[DEC id:dec.garden.context-id-prefix date:2026-06-16 by:llm]
  New heading IDs must use =monadc.context.<topic>.<stable-name>=. Test records
  may point to any active heading ID. Avoid invented TEST-CONTEXT strings that
  do not resolve to an actual heading.

[DEC id:dec.garden.prune-generated-noise date:2026-06-16 by:llm]
  Generated run artifacts, local caches, and one-off debug transcripts should not
  live in the clean context tar. Summaries may live in =quality.org= or
  =debug-agenda.org= when they influence future work.

[DEC id:dec.garden.agent-must-leave-trail date:2026-06-16 by:human+llm]
  An agent that changes tests, compiler behavior, or project conventions must
  leave a trail: updated context links, a decision or fix record, and an honest
  validation note. The trail is part of the deliverable.

[DEC id:dec.garden.partial-truth-labels date:2026-06-16 by:llm]
  If validation is partial because the compiler binary or full repository is not
  present, say that in the context and in the user-facing answer. Do not convert
  partial validation into a green claim.

  Major topics should declare category contracts. A topic without an owned phase,
  invariants, test hooks, and failure symptoms is not yet a mature category.

  local contract and makes cross-file navigation semantic rather than merely
  alphabetical.
