[OBS id:obs.style.hybrid-c-signature-lisp-body src:user-request conf:high]
  MonadC supports a hybrid style where the function head/signature reads closer
  to C-like parameter and return declarations, while the body remains Lisp-like
  S-expression code. Example shape:

#+BEGIN_SRC monad
(define (push-vertex [x :: F32] [y :: F32]
                     [u :: F32] [v :: F32]
                     [c :: Color])
  (unless (g-vertex-count >= MAX-VERTICES)
    (define [vtx -> Vertex]
      (g-vertices g-vertex-count))
    (set! vtx.x x)
    (set! vtx.y y)
    (set! vtx.u u)
    (set! vtx.v v)
    (set! vtx.r c.r)
    (set! vtx.g c.g)
    (set! vtx.b c.b)
    (set! vtx.a c.a)
    (set! g-vertex-count (g-vertex-count + 1))))
#+END_SRC

[THINK id:think.style.hybrid-c-lisp-use src:user-request conf:high]
  Human framing: this style is useful because the type/ABI surface looks
  concrete and C-adjacent, while the body keeps Lisp's uniform expression
  syntax. It is especially relevant for low-level graphics/runtime code where
  fields, scalar types, buffers, and mutation are prominent.

[OBS id:obs.style.pure-wisp-prelude src:core/prelude/Coll.mon conf:high]
  Pure Wisp style appears in =core/prelude/Coll.mon=. It uses indentation,
  equation-style definitions, type signatures, pattern clauses, and guards
  instead of parenthesized Lisp forms for the main program shape.

#+BEGIN_SRC monad
define take :: Int -> [a] -> [a]
  _ []            -> []
  n [x|xs]
      | n > 0     -> (x ++ (take (n - 1) xs))
      | otherwise -> []
#+END_SRC

  source-level style that expands into ordinary S-expressions before reader and
