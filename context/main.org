[OBS id:obs.main.main-function src:main.c:1416-1431 conf:high]
  int main(int argc, char **argv):
      CompilerFlags flags = parse_flags(argc, argv);
      switch (flags.mode):
          CMD_REPL    → repl_run()
          CMD_NEW     → cmd_new(package_name)
          CMD_BUILD   → cmd_build()
          CMD_RUN     → cmd_run()
          CMD_CLEAN   → cmd_clean()
          CMD_INSTALL → cmd_install()
          CMD_TEST    → cmd_test(input_file)
          CMD_COMPILE → compile(&flags)

  All CLI commands except CMD_COMPILE delegate to cli.h functions.
  CMD_COMPILE calls compile() which calls compile_one().

[OBS id:obs.main.cli-dispatch src:main.c:1418-1429 conf:high]
  Each CommandMode maps to a function that either:
  - Delegates to a buildsystem action (CMD_NEW, CMD_BUILD, CMD_RUN, CMD_CLEAN, CMD_INSTALL)
  - Runs the REPL (CMD_REPL)
  - Runs tests (CMD_TEST)
  - Invokes the full compilation pipeline (CMD_COMPILE)

  compile() wraps compile_one() for the main module. It:
  - Initializes LLVM (LLVMInitializeAllTargetInfos, AllTargets, AllTargetMCs,
    AllAsmPrinters, AllAsmParsers)
  - Creates a fresh registry
  - Calls compile_one(source_path, flags, is_main_module=true)
  - Links the final executable using g_ffi_link_libs

  compile_one calls. Linking happens once in compile() after all modules

[OBS id:obs.main.mangling-convention src:main.c conf:high]
  Exported symbols are mangled as module_name::local_name (double-colon
  separator). This matches the MonadC import resolution scheme. LLVM symbol
  renaming happens in Phase 9 via LLVMSetValueName. Non-exported symbols
  retain their unqualified names.

[THINK id:think.main.deferred-object-linking conf:high]
  The deferred object linking architecture means compile_one returns before
  the final binary is linked. The compile() wrapper collects all .o paths
  from the registry and invokes the system linker (cc) with g_ffi_link_libs.
  This works because objects are standalone ELF relocatables — linking is
  purely additive (no LTO).
