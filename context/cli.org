[OBS id:obs.cli.file src:cli.h,cli.c conf:high]
  The CLI argument parser maps command-line arguments to a =CompilerFlags=
  struct and dispatches to the appropriate command handler. Header at
  =cli.h=, implementation in =cli.c= (≈730 lines). All =cmd_*= functions
  live in =cli.c= and delegate to the build system or the compiler
  pipeline in =main.c=.

[INF id:inf.cli.file-connect from:obs.cli.file conf:high]
  connects-to -> `monadc.context.main` (pipeline dispatch after flag parsing) ·
  `monadc.context.buildsystem` (cmd_build/cmd_run/cmd_clean delegate to
  build resolver + do_build)

[OBS id:obs.cli.parse-flags src:cli.c:193 conf:high]
  =CompilerFlags parse_flags(int argc, char **argv)= — The main flag
  parser. Walks =argv=, recognizes long options (=--emit-ir=,
  =--emit-json=, =--test-mode=, =--repl=, etc.), short options (=-o=,
  =-h=), and subcommands (=build=, =run=, =new=, =clean=, =install=,
  =test=, =repl=). Returns a fully populated =CompilerFlags= struct.
  Prints usage and calls =exit(0)= on =--help=.

[DEC id:dec.cli.flag-word-duality date:2026-06-18 by:human src:cli.c,completion.c,config.c,tests/test_cli_duality.py conf:high]
  User-facing CLI flags should have a matching bare word form when practical.
  Examples: =--emit-ir= and =emit-ir=, =-S= and =asm=, =-c= and =obj=,
  =-O= and =optimize=, =-v= and =verbose=, =-i= and =repl=. The default
  config flag reader should accept the same non-positional forms as the CLI.

[OBS id:obs.cli.eval-command src:cli.h,cli.c,main.c,completion.c conf:high]
  =CMD_EVAL= supports one-shot REPL evaluation. =monad -e "code"=,
  =monad --eval "code"=, and =monad eval "code"= all store the source string
  in =CompilerFlags.eval_code= and dispatch through =cmd_eval=, which evaluates
  it with =repl_eval_line= and exits with success/failure status.

[FIX id:fix.cli.eval-leading-imports-2026-07-16 date:2026-07-16 by:codex src:main.c,tests/test_repl.py conf:high]
  One-shot evaluation accepts leading Wisp =import= lines followed by one
  expression. Each import is evaluated in the same =REPLContext= before the
  remaining, potentially multiline expression, so =monad eval= can exercise
  core methods without requiring a separate interactive session.

[OBS id:obs.cli.cmd-build.handler src:cli.c:520 conf:high]
  =void cmd_build(void)= — Top-level build command. Calls
  =resolve_build_info= (line 415) to parse =monad.json= and locate
  sources, then =do_build= (line 505) which invokes =compile_one= from
  =main.c= on each source file and links the resulting objects with the
  Monad runtime into an executable.

[OBS id:obs.cli.cmd-run.handler src:cli.c:536 conf:high]
  =void cmd_run(void)= — Builds (via =cmd_build= internally through
  =do_build=) and executes the resulting binary. Looks up the executable
  path from the build info and runs it with any additional arguments.

[THINK id:think.cli.flag-proliferation from:obs.cli.parse-flags conf:medium]
  The number of emit flags is growing. Consider grouping them into a
  bitmask or a sub-struct. The =start_repl= flag mixed with emit flags
  is a design smell — it controls behavior, not output format.
