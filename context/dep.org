[OBS id:obs.context.dep.file src:context/dep.org,dep.h conf:high]
  This file records the complete reference for the dependent type checker kernel
  in dep.h — the Intensional Type Theory (ITT) implementation with Π-types,
  Σ-types, universe hierarchy, NbE, bidirectional checking, metavariable
  elaboration, and the interop bridge to the ground-type system.

[THINK id:think.context.dep.architectural-role conf:high]
  The dependent type checker runs as Phase 6.5 — a shadow pass between feature
  setup and LLVM codegen. It does not emit code. It validates dependent
  annotations, elaborates surface syntax to core terms, and back-propagates
  ground types to the AST for the HM inferencer and LLVM codegen to consume.
  This design ensures the fast HM path is the default and the full ITT path is
  opt-in per expression.

[OBS id:obs.dep.level-kind-enum src:dep.h:61-66 conf:high]
  LevelKind enum: LEVEL_CONST (concrete integer), LEVEL_VAR (universe-polymorphic
  variable), LEVEL_MAX (join of two levels), LEVEL_SUCC (successor offset).
  The predicative rule: Γ ⊢ Type u : Type (u+1); Π-formation uses max(u,v).

[OBS id:obs.dep.level-struct src:dep.h:68-74 conf:high]
  Level struct: kind discriminator; value (int for CONST/SUCC offset); var_id
  (unique ID for VAR); left and right pointers for composite expressions.
  Levels form a DAG of shared sub-expressions.

[OBS id:obs.dep.level-const src:dep.h:76 conf:high]

[OBS id:obs.dep.level-var src:dep.h:77 conf:high]

[OBS id:obs.dep.level-max src:dep.h:78 conf:high]
  level_max(Level *u, Level *v) — construct max(u, v), the join of two level expressions.

[OBS id:obs.dep.level-succ src:dep.h:79 conf:high]

[OBS id:obs.dep.level-clone src:dep.h:80 conf:high]
  level_clone(Level *u) — deep copy of a level expression DAG.

[OBS id:obs.dep.level-free src:dep.h:81 conf:high]

[OBS id:obs.dep.level-eval src:dep.h:82 conf:high]

[OBS id:obs.dep.level-leq src:dep.h:83 conf:high]

[OBS id:obs.dep.termkind-embed src:dep.h:152-153 conf:high]

[OBS id:obs.dep.term-struct-overview src:dep.h:168-240 conf:high]
  Term struct uses a common discriminator (kind: TermKind) with disjoint field
  groups for each variant. All pointer fields are owned (free on term_free).
  Source location (line, col) and source_ast pointer support error reporting
  and back-propagation of ground types to the HM inferencer.

[THINK id:think.dep.soundness-holes conf:medium]
  The NbE kernel has known soundness holes documented in TODO.org: (1) dep_conv
  allows implicit Int≡Float and Int≡Char coercions, (2) pattern unification is
  a stub with no Miller condition check, (3) the occurs check runs in the wrong
  environment, (4) VAL_PI→TYPE_ARROW conversion drops quantifiers, (5) universe
  levels use equality not ≤. These are deliberate stubs that test the thesis
  that a clean NbE architecture can be incrementally filled in without rewrites.

[THINK id:think.dep.locally-nameless conf:high]
  The locally nameless representation (De Bruijn indices for bound variables,
  free variable names for globals) eliminates α-equivalence bugs entirely while
  keeping error messages readable. This is the standard approach in
  production-quality dependently typed implementations (Coq, Agda, Idris).

[THINK id:think.dep.why-spine-form conf:high]
  TERM_APP uses a flat spine form (function + array of arguments) instead of
  nested left-associative applications. This speeds up WHNF reduction (no need
  to traverse a chain to find the head) and simplifies the evaluator's handling
  of neutrals (a single spine push vs. nested app eliminations).
