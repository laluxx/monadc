[DOC id:doc.category.catface.purpose src:context/category/catface.org conf:high]
  Catface is the canonical live interface for the context category.  It replaces
  the earlier Python curses prototype with a Zig terminal application designed
  around the actual category: objects, generating morphisms, Hom-set views,
  projections, search information, and fast link-following.

[DEC id:dec.category.catface.zig-canonical date:2026-06-16 by:llm]
  The live TUI should be written in Zig, not Python.  The desired interface has
  a custom terminal feel, low startup latency, explicit screen buffering,
  precise ANSI control, and an interaction model closer to a small editor than a
  generic curses list widget.  Zig matches that design better than Python.

[DOC id:doc.category.catface.interface-contract src:context/category/catface.org conf:high]
  The interface is dual-pane.  The left pane is a live fuzzy result stack.  The
  right pane is the focused object card and Hom-set inspector.  =Tab= switches
  the active pane; the active pane is rendered brighter.  =Enter= follows the
  selected object by rewriting the query to =?object-id=.  Category operations
  append compact glyphs to the query instead of opening modal forms.

[DOC id:doc.category.catface.keys src:context/category/catface.org conf:high]
  Core keys: =Tab= switch pane, =Enter= focus/follow, =j/k= or arrows move,
  =o= append outgoing = > =, =i= append incoming = < =, =n= append neighborhood
  =~=, =p= append projection =◗=, =b= history back, =?= help, =Ctrl-U= clear,
  =q= quit.

[DOC id:doc.category.catface.query-language src:context/category/catface.org conf:high]
  Catface Query Language is keyboard-small and stack/filter-like.  Plain words
  fuzzy-match objects.  =:Kind= narrows by type, =@path= narrows by source path,
  =?id= or =#id= narrows by object identity, =>= and =<= follow generating
  morphisms, =~= computes a neighborhood, and =◗= projects through taxonomy or
  functor-like conceptual edges.  The expression =(superset) ◗= is intentionally
  valid: a human-sized fuzzy term followed by a categorical projection.

[DEC id:dec.category.catface.not-sql date:2026-06-16 by:llm]
  The query language is not SQL-like.  It is optimized for live typing in a
  terminal: few punctuation keys, no keywords, no mandatory field names, and
  visible category glyphs for operations.

[DOC id:doc.category.catface.math src:context/category/catface.org conf:high]
  Catface does not merely search text.  It loads a typed directed multigraph G
  and reads the context as the free category =Ctx = Free(G)=.  Every object has
  an identity path.  Every generating edge has a source and target.  Composition
  is path concatenation when codomain and domain agree.  The UI exposes outgoing
  =Hom(A, -)= and incoming =Hom(-, A)= views for the focused object A.

[INF id:inf.category.catface.yoneda-instinct from:doc.category.catface.math conf:medium]
  The object card follows a Yoneda-like instinct: understand an object through
  the arrows into and out of it.  Catface does not claim a full Yoneda embedding;
  it uses representable views as an interface discipline.

[DOC id:doc.category.catface.contract src:context/category/catface.org conf:high]
  Catface is the live terminal interface for the context category layer.
  It loads Org files, builds the free category graph, and exposes Hom-set
  views through fuzzy search and categorical projections.

- *Owned Phase*: Context navigation and query (post-parse, not a compiler phase)
- *Inputs*: Org files and category/ TSV tables (objects.tsv, morphisms.tsv)
- *Outputs*: Terminal UI frames, JSON/Org/Markdown exports, --check exit code
- *Invariants*:
  - Query language is stack/filter-based with visible category glyphs
  - All generating morphisms have typed source/target in the model
  - Dual-pane layout: left = fuzzy result stack, right = object card + Hom-sets
  - No file writes outside explicit --export mode
- *Test Hooks*: =zig build test= runs law_tests; =--check= parses garden and verifies no unresolved edges
- *Failure Symptoms*:
  - Segfault on malformed Org: the org loader must handle all valid garden files
  - Missing objects.tsv entries: the parser will fail silently or skip
  - Cursor errors: =curs_set= failures are silently caught
- *Context Neighbors*: [[file:../category/query-language.org][query-language.org]], [[file:../category/tui.org][tui.org]], [[file:../category/mathematics.org][mathematics.org]], [[file:../category/type-system.org][type-system.org]]

  interface path.  It was useful for proving the data model but was visually weak
  and architecturally mismatched with the desired custom terminal experience.
  Batch Python scripts may remain for reports and validation; the live interface
  is now Zig-first.
