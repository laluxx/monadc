* Taxonomy as Preorder
:PROPERTIES:

:CONTEXT_DESCRIPTION: Taxonomy as a thin category / preorder

[DOC id:doc.category.taxonomy.preorder src:context/category/taxonomy.org conf:high]
  The concept taxonomy is a thin category: there is at most one morphism A -> B,
  read as "A refines B" or "A is a specialization of B".  Reflexivity gives
  identities.  Transitivity gives composition.

#+BEGIN_EXAMPLE
reader.tokenization -> reader
reader -> frontend
frontend -> compiler-phase
therefore reader.tokenization -> compiler-phase
#+END_EXAMPLE

[DEC id:dec.category.taxonomy.thin-category date:2026-06-16 by:llm]
  Taxonomy edges are preorder edges.  They must be acyclic unless an explicit
  equivalence class is declared.

[DOC id:doc.category.taxonomy.fca src:context/category/taxonomy.org conf:medium]
  A future formal concept analysis (FCA) view can treat context files/tests as
  objects and tags/record types/failure kinds as attributes.  A formal concept
  is a maximal pair (extent, intent): all objects sharing all attributes in the
  intent, and all attributes common to the extent.

#+BEGIN_EXAMPLE
extent  = {reader.org, wisp.org, tests/reader-atoms.tsv}
intent  = {frontend, syntax, parse, TEST-CONTEXT}
concept = syntax-front-end-regression-surface
#+END_EXAMPLE
