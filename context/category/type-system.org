[DOC id:doc.category.types.judgments src:context/category/type-system.org conf:high]
  The context type system assigns kinds to objects and morphisms.  A well-typed
  generating morphism has a source object whose kind is accepted by the morphism
  domain and a target object whose kind is accepted by the morphism codomain.

#+BEGIN_EXAMPLE
Γ ⊢ X : ObjectKind
Γ ⊢ f : X -> Y [MorphismKind]
Γ ⊢ path(f1,...,fn) : X0 -> Xn
#+END_EXAMPLE

* Morphism Type Rules
:PROPERTIES:
:ID: monadc.context.category.types.morphisms
:CONTEXT_DESCRIPTION: Morphism typing rules
:CUSTOM_ID: category-types-morphisms
:CONTEXT_KIND: type-system
:CONTEXT_STATUS: active
:SOURCE: context/category/type-system.org
:CONFIDENCE: high
:END:

| Rule | Domain | Codomain | Check |
|------+--------+----------+-------|
| contains | File/Script/Report | Heading/Record/Script/Report | object line/path lies inside file |
| file-link | File | File | target file exists or is explicitly external |
| id-link | File/Heading/Record | Heading | target ID resolves |
| supports | Record | Record/Heading | target resolves to claim or contract |
| supersedes | Record | Record/Heading | temporal/update relation resolves |
| verifies | Test/Record | Heading/Record | target is a contract or claim |
| blocks | Record | Heading/Record | target resolves to blocked contract |
| classifies-as | any | Concept | concept exists in taxonomy |
| refines | Concept | Concept | preorder edge is acyclic modulo equivalence |

[DEC id:dec.category.types.no-unknown-links date:2026-06-16 by:llm]
  Unknown target IDs are type errors in the category layer.  A prose note may be
  speculative, but a typed morphism should either resolve or be marked TODO with
  an explicit nonbinding status.

[DOC id:doc.category.types.metadata src:context/category/type-system.org conf:high]
  Metadata is a dependent record over context nodes.  Required fields depend on
  node kind.  For example, a contract heading should have CONTEXT_KIND=contract,
  owner/phase information, and links to tests or known TODOs; an observation
  record must have source and confidence.

| Field | Type | Required for |
|-------+------+--------------|
| :ID: | stable symbol | Heading |
| CONTEXT_KIND | enum | Heading |
| CONTEXT_STATUS | enum | Heading |
| SOURCE | path or source label | Heading/Record |
| CONFIDENCE | low/medium/high | Heading/Record |
| id: | stable symbol | Record |
| src: | path/source label | Record when evidence-bearing |
| from:/supports:/verifies:/blocks:/supersedes: | reference list | Record when edge-bearing |

[DOC id:doc.category.types.proof-obligations src:context/category/type-system.org conf:high]
  The verifier is not a theorem prover for MonadC.  It proves structural facts
  about the garden: uniqueness, endpoint resolution, identity availability,
  composable path typing, record-reference validity, and stable generated
  reports.  Semantic compiler claims still require tests and source inspection.
