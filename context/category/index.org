[DOC id:doc.category.index.purpose src:context/category/index.org conf:high]
  This directory turns the context garden into a mathematically typed semantic
  system.  The garden is modeled as a small category generated by context
  objects and explicitly typed edges.  Scripts live beside the theory because
  source code is also a context artifact: it can be linked, inspected, tested,
  and used as executable documentation.

[DEC id:dec.category.index.free-category-model date:2026-06-16 by:llm]
  The canonical model is the free category Ctx generated by the typed context
  graph.  Objects are files, headings, records, source regions, tests, and
  generated reports.  Generating morphisms are file containment, id links,
  record support links, verifies/exercises links, taxonomy edges, and explicit
  category tables.  Identity morphisms are empty paths.  Composition is path
  concatenation when codomain and domain agree.

* Working Definition

[DOC id:doc.category.index.definition src:context/category/index.org conf:high]
  A context category is a small category Ctx = Free(G) where G is the typed
  directed multigraph extracted from Org files and machine tables.  Ctx is
  small because every object and generating morphism is finite in a repository
  checkout.  It is free because no semantic equality between non-identical paths
  is assumed unless a future quotient/equational theory is explicitly added.

[INF id:inf.category.index.why-free from:dec.category.index.free-category-model conf:high]
  The free-category choice is conservative.  It makes all link composition
  legal and checkable while avoiding unsound claims such as "these two evidence
  paths are equal" when the prose has not stated such an equation.

[DOC id:doc.category.index.executable-contract src:context/category/index.org conf:high]
  The category is considered healthy when the verifier can parse the garden,
  build the graph, generate identity morphisms, compose sampled paths, validate
  endpoint types for known generating morphisms, and produce information metrics
  without unresolved IDs or duplicate node identities.
