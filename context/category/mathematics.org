* The Base Graph G

[DOC id:doc.category.math.base-graph src:context/category/mathematics.org conf:high]
  The base graph G has vertices for context objects and directed labeled edges
  for generating morphisms.  A vertex is typed by an object kind.  An edge is
  typed by a morphism kind with a declared domain-kind and codomain-kind.  The
  parser extracts vertices from files, headings, records, and tables; scripts may
  add generated objects such as reports.

#+BEGIN_EXAMPLE
G = (V, E, src, dst, kind)
V = finite set of context objects
E = finite set of typed links
src, dst : E -> V
kind : E -> MorphismKind
#+END_EXAMPLE

[DEC id:dec.category.math.free-category-is-canonical date:2026-06-16 by:llm]
  The canonical mathematical object for the garden is Free(G), the free category
  generated by the context graph.

[DOC id:doc.category.math.objects-morphisms src:context/category/mathematics.org conf:high]
  Objects of Ctx are vertices of G.  Morphisms of Ctx are finite directed paths
  in G.  The identity morphism id_X is the empty path at object X.  Composition
  is concatenation of paths when the target of the first path equals the source
  of the second path.

#+BEGIN_EXAMPLE
id_X = [] at X
(g ∘ f) = f followed by g
where f : X -> Y and g : Y -> Z
#+END_EXAMPLE

[INF id:inf.category.math.laws-hold-by-construction from:dec.category.math.free-category-is-canonical conf:high]
  Identity and associativity laws hold by construction because path
  concatenation has empty-list identities and associative list concatenation.

[DOC id:doc.category.math.typed-multigraph src:context/category/mathematics.org conf:high]
  The graph is a multigraph: multiple different links may share the same source
  and target but carry different meaning.  The type of a generating morphism is
  not only src -> dst; it is kind : DomainKind -> CodomainKind.  This prevents
  untyped edges such as "related" from carrying false evidence.

| Morphism kind | Mathematical reading |
|---------------+----------------------|
| contains | structural inclusion |
| file-link | explicit document navigation |
| id-link | pointer to stable heading object |
| supports | evidential implication candidate |
| supersedes | temporal preorder update |
| verifies | test-to-contract protection |
| blocks | obstruction relation |
| classifies-as | taxonomy membership |
| refines | preorder specialization |
| forgets-to | functor/projection witness |

[DOC id:doc.category.math.functors src:context/category/mathematics.org conf:high]
  A functor out of Ctx preserves identities and composition while forgetting or
  reinterpreting structure.  The garden uses functors as safe projections: a
  search interface can forget morphism labels, a report can map nodes to files,
  and a triage view can map nodes to compiler phases.

| Functor | Domain | Codomain | Meaning |
|---------+--------+----------+---------|
| U | Ctx | DirectedGraph | Forget identities/composite paths; keep nodes and generating edges. |
| File | Ctx | FileSet preorder | Map every object to its owning file. |
| Phase | Ctx | CompilerPhase preorder | Map contracts/tests to reader/wisp/infer/codegen/runtime/etc. |
| Evidence | Ctx | Trust lattice | Map records to confidence levels and provenance classes. |
| Info | Ctx | WeightedGraph | Map edges/nodes to information weights for search/ranking. |

[DEC id:dec.category.math.functor-rule date:2026-06-16 by:llm]
  Reports and scripts should name the projection/functor they are using.  This
  avoids confusing a lossy search view with the full context category.

[THINK id:think.category.math.natural-transformation-workflow src:context/category/mathematics.org conf:medium]
  A garden migration can be described as a natural transformation between two
  projections.  Example: old triage projection maps TEST-ID only to a file;
  improved triage maps TEST-ID to TEST-CONTEXT then category owner.  Naturality
  means the old and new routes agree where the old information was precise, and
  the new route refines the ambiguous cases rather than contradicting them.

[DOC id:doc.category.math.adjunction-shaped src:context/category/mathematics.org conf:medium]
  Some tooling has adjunction-like shape even when not implemented as formal
  category objects.  Search compresses a corpus into candidate nodes; expansion
  sends a node back to surrounding context.  The useful contract is that search
  left-adjoints should be recall-oriented, while expansion right-adjoints should
  be precision-oriented.

| Pair | Intuition |
|------+-----------|
| search ⊣ expand | many words collapse to candidates; a candidate expands to evidence. |
| summarize ⊣ cite | prose compresses; citations restore traceability. |
| test failure ⊣ triage context | failure maps to owner; owner expands to repair paths. |
