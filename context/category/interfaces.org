* Interface Principle
:PROPERTIES:

:CUSTOM_ID: category-interfaces-principle
:CONTEXT_KIND: interface

[DOC id:doc.category.interfaces.principle src:context/category/interfaces.org conf:high]
  The context should be readable as prose, navigable as Org, queryable as a
  graph, measurable as information, and executable as scripts.  Each interface is
  a functor/projection from the full category, and each projection should state
  what it forgets.

[DOC id:doc.category.interfaces.json src:context/category/interfaces.org conf:high]
  Scripts expose JSON output for programmatic use.  JSON reports should include
  counts, unresolved edges, entropy values, and top-ranked search results.  The
  prose layer should link to generated artifacts rather than copy giant JSON
  blobs into Org.

[DOC id:doc.category.interfaces.live-query-tui src:context/category/interfaces.org conf:high]
  The live interface layer consists of a compact query language plus the Zig Catface TUI.  Both are projections from the same parsed category model, so CLI search,
  JSON output, card rendering, and terminal exploration agree on object IDs,
  edge IDs, and category operations.
