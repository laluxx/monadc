[DOC id:doc.category.info.entropy src:context/category/information-theory.org conf:high]
  Shannon entropy measures how distributed a signal is.  In the garden, useful
  signals include record type distribution, file/tag distribution, and morphism
  kind distribution.  High entropy can mean broad coverage; low entropy can mean
  focused specialization or neglected dimensions.  Interpretation requires the
  category map.

[DOC id:doc.category.info.search src:context/category/information-theory.org conf:high]
  Search should prefer nodes that reduce uncertainty about a task.  A query over
  "Wisp typed define failing desugar" should increase posterior mass on Wisp,
  reader/desugar, tests, and known failure records.  The simple scripts use term
  weights and structural bonuses; future scripts can add BM25, embeddings, or
  probabilistic language models.

| Signal | Search effect |
|--------+---------------|
| exact ID match | maximum precision |
| title/heading match | high precision |
| record id/type match | high precision for triage |
| local neighborhood | context expansion bonus |
| rare term | high information gain |
| common tag | navigation but low surprise |

[DOC id:doc.category.info.surprise src:context/category/information-theory.org conf:medium]
  A record is surprising when it contradicts or supersedes nearby expectations,
  has low confidence but high centrality, or belongs to a sparsely covered
  category.  Generated metrics can rank these candidates for gardener review.

[IDEA id:idea.category.info.staleness-score src:context/category/information-theory.org conf:medium]
  Define staleness as a weighted function of last_modified age, superseded
  record edges, failing tests linked to a contract, and absence of recent source
  evidence.  Do not delete stale material automatically; route it to curation.

[THINK id:think.category.info.mdl src:context/category/information-theory.org conf:medium]
  MDL is a useful pruning principle: keep the smallest context representation
  that still predicts failures, explains decisions, and routes future changes.
  Repetitive prose with no new edge, invariant, or evidence should be merged or
  replaced by a generated index.
