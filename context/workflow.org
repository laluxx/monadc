[DEC id:dec.workflow.living-coverage date:2026-06-11 by:human]
  Explore MonadC as a living language system: keep adding small behavioral
  tests as language behavior is discovered, prefer codegen-backed compile/run
  fixtures when possible, and update context records continuously so the test
  suite and repository memory evolve together.

[DEC id:dec.workflow.coverage-over-green date:2026-06-11 by:human]
  Do not stop coverage expansion merely because a codegen-backed test exposes a
  deep compiler/runtime failure. Fix only trivial fixture syntax mistakes during
  coverage work; leave meaningful deep failures visible as executable pressure
  on the implementation.

[DEC id:dec.workflow.coverage-then-triage date:2026-06-11 by:human]
  After a coverage subset is large enough to reveal useful behavior, triage the
  failing tests instead of treating every failure the same. Classify each failure
  as an invalid fixture, a missing runner expectation mode, or a compiler/runtime
  defect; fix syntax/contract mistakes immediately and use the linked context
  record when choosing the next compiler fix.

[DEC id:dec.workflow.skip-uncertain-failures date:2026-06-11 by:human]
  During broad coverage expansion, do not spend long debugging an uncertain
  failure. If a fixture emits JSON and the failure is not clearly a syntax or
  test-contract mistake, keep the coverage and move on so reader, codegen,
  closure, HM, and dependent-type surfaces continue expanding.

[DEC id:dec.workflow.context-first date:2026-06-12 by:human]
  Before starting any work, read the relevant context files. After finishing,
  extend the context with what was done, discovered, decided, and left undone.
  Never leave the context behind — always extend it.

[DEC id:dec.workflow.context-recurring-todos date:2026-06-12 by:codex]
  Use =context/todo.org= for durable, cross-session TODOs discovered during
  work. Small ephemeral TODOs ("fix this one fixture") should go in the
  conversation summary, not in permanent context records.

[DEC id:dec.workflow.always-extend-context date:2026-06-12 by:human]
  Never forget to write back and extend the context. Everything the LLM does
  should leave a trace in the context corpus. Every discovery, decision,
  observation, TODO, and interaction that adds durable value must be recorded
  before the session ends. This is the core mechanism that turns ephemeral
  conversations into permanent collaborative memory.
