  targets (test, install, uninstall, clean). Uses GCC, llvm-config for
  LLVM-C flags, and pkg-config for libclang.

[OBS id:obs.makefile.target-all src:Makefile:29-30 conf:high]
  =make all= (default target). Builds with DEBUG_CFLAGS: =-g -DDEBUG=.
  Links against libmonad.a and all LLVM libraries. Produces the =monad=
  binary and =libmonad.a=.

[OBS id:obs.makefile.target-asan src:Makefile:32-34 conf:high]
  =make asan=. Builds with ASAN_CFLAGS: =-g -fsanitize=address
  -fno-omit-frame-pointer -DDEBUG=. Adds =-fsanitize=address= to LDFLAGS.
  Links binary with AddressSanitizer runtime.

[OBS id:obs.makefile.target-release src:Makefile:36-37 conf:high]
  =make release=. Builds with RELEASE_CFLAGS: =-DNDEBUG -O2=. Optimised
  binary, no debug symbols, no assertion checks.

[OBS id:obs.makefile.target-clean src:Makefile:55-56 conf:high]
  =make clean=. Removes all .o files (compiler and runtime), libmonad.a,
  and the monad binary.

[OBS id:obs.makefile.target-install src:Makefile:59-71 conf:high]
  =make install=. Installs the monad binary to PREFIX/bin, libmonad.a to
  PREFIX/lib, runtime.h to PREFIX/include/monad/, and core .mon modules
  to PREFIX/lib/monad/core/. Default PREFIX is /usr/local.

[OBS id:obs.makefile.target-uninstall src:Makefile:73-77 conf:high]
  =make uninstall=. Removes monad binary from BINDIR, libmonad.a from
  LIBDIR, the include directory (INCDIR), and the core module directory.

[OBS id:obs.makefile.target-test src:Makefile:79-83 conf:high]
  =make test=. Builds with =all= then runs =python3 tests/run.py=.

[OBS id:obs.makefile.target-core src:Makefile:86-87 conf:high]
  =make core=. Builds and runs =python3 tests/run_core.py=.

[OBS id:obs.makefile.mode-debug src:Makefile:12,29-30 conf:high]
  Debug mode (=make all=): =-g -O0 -DDEBUG=. Best debugging experience
  (DWARF symbols, no optimiser reordering). Slowest runtime. Default mode.

[OBS id:obs.makefile.mode-asan src:Makefile:13,32-34 conf:high]
  ASAN mode (=make asan=): =-g -fsanitize=address -fno-omit-frame-pointer
  -DDEBUG=. Catches buffer overflows, use-after-free, and memory leaks
  at runtime. Adds ~2x overhead; use during development and before release.

[OBS id:obs.makefile.mode-release src:Makefile:14,36-37 conf:high]
  Release mode (=make release=): =-DNDEBUG -O2=. Maximum performance,
  no assertions, no debug symbols. Use for production builds and benchmarks.

[OBS id:obs.makefile.runtime-objs src:Makefile:20-22 conf:high]
  RUNTIME_SRC = runtime.c arena.c ⇒ RUNTIME_OBJ = runtime.o arena.o.
  These are compiled with -fPIC and archived into libmonad.a. They are
  excluded from the compiler binary's direct object list to prevent
  parallel make race conditions.

[OBS id:obs.makefile.compiler-objs src:Makefile:25,27 conf:high]
  SRCS = $(filter-out $(RUNTIME_SRC), $(wildcard *.c)) — all .c files
  EXCEPT runtime.c and arena.c. Pattern filters out runtime sources.
  OBJS = $(SRCS:.c=.o) — the corresponding object files.

[OBS id:obs.makefile.libmonad src:Makefile:39-43 conf:high]
  libmonad.a: static archive built from runtime objects only.
  Compiled with -fPIC. Linked statically into the monad binary so no .so
  dependency at runtime. Used also by compiled MonadC programs that link
  against the runtime.

[OBS id:obs.makefile.monad-binary src:Makefile:46-47 conf:high]
  =$(TARGET): $(OBJS) $(RUNTIME_LIB)= — the monad binary links all compiler
  objects + libmonad.a + LDFLAGS. Uses -rdynamic so runtime symbols are
  available for FFI dlopen.

[OBS id:obs.makefile.ffi-o-rule src:Makefile:49-50 conf:high]
  Special compilation rule for ffi.o: uses FFI_CFLAGS (from pkg-config).
  All other .o files use the generic =%.o: %.c= pattern rule.

[OBS id:obs.makefile.generic-compile src:Makefile:52-53 conf:high]
  Generic pattern rule: =%.o: %.c= — compiles any .c to .o with CFLAGS.
  This matches all compiler sources not covered by a more specific rule.

[OBS id:obs.makefile.nprocs src:Makefile:16-17 conf:high]
  =NPROCS=$(shell nproc)= then =MAKEFLAGS += -j$(NPROCS)=. Enables
  parallel compilation across all available CPU cores.

[OBS id:obs.makefile.filter-race src:Makefile:25 conf:high]
  =$(filter-out $(RUNTIME_SRC), $(wildcard *.c))= ensures runtime.c and
  arena.c are NOT in the compiler SRCS list. This prevents make -j from
  compiling the same .c file concurrently via two different rules (one for
  libmonad.a with -fPIC and one for the binary without). The filter-out
  makes each runtime source appear in only one prerequisite chain.

[THINK id:think.makefile.architecture-simplicity conf:high]
  The Makefile is refreshingly simple for a project of this complexity.
  No recursive make, no generated files, no autoconf. The entire build
  is defined in variables at the top (CC, CFLAGS, LDFLAGS, SRCS, OBJS)
  with a single pattern rule for compilation and a single link rule.
  This simplicity is the result of deliberate restraint: the Makefile
  does only what it needs to (compile sources, link binary, install,
  clean) and nothing more. The three build modes (debug/asan/release)
  are handled by selective variable assignment. The entire build system
  is comprehensible in a single reading, which is the highest compliment
  you can give a Makefile.

[THINK id:think.makefile.llvm-config-dependency conf:medium]
  The Makefile's dependency on llvm-config for LLVM discovery is the
  most fragile part of the build. llvm-config output varies between LLVM
  versions and installation methods (system package vs from-source vs
  Homebrew). The Makefile captures flags at build time with =$(shell
  llvm-config ...)=, which means a missing or wrong llvm-config produces
  an inscrutable build error (missing symbols, wrong architecture).
  Adding a configure-time check (Makefile target =configure= or a
  config.sh that runs llvm-config and validates the output before the
  main build) would turn the opaque failure into a clear error message.
  The check could verify that llvm-config exists, returns a known LLVM
  version (19+), and produces the expected flag formats.

[THINK id:think.makefile.object-file-layout conf:medium]
  The decision to place .o files alongside their sources (rather than
  in a separate build/ directory) is a tradeoff between simplicity and
  cleanliness. In-source builds are simpler (no build directory creation,
  no path mapping) but clutter the source tree. For a small project this
  is manageable; for a project with 40+ source files, the build artifacts
  mixed with sources become noise. A separate build directory (build/debug/,
  build/release/, build/asan/) would keep the source tree clean and
  enable parallel debug/release builds. The cost is added Makefile
  complexity: pattern rules with path substitution (%_dir rules or VPATH).

[THINK id:think.makefile.header-dependency-tracking conf:medium]
  The Makefile does not track header dependencies automatically. If a
  .h file changes, =make= will not recompile the .c files that include
  it unless the .o file is explicitly declared as depending on the .h.
  This is a common Makefile pitfall that causes phantom build failures
  after header changes. The fix is =gcc -MMD= which generates .d files
  that Make can include. Adding =DEPFLAGS = -MMD -MP= to CFLAGS and
  =-include $(wildcard $(OBJS:.o=.d))= to the Makefile would add
  automatic header dependency tracking with about 3 lines of change.
  Until this is done, developers must =make clean= after any header
  change.

[THINK id:think.makefile.test-integration conf:medium]
  The test targets are well-integrated: =make test= runs the full suite,
  =make test-continuous= runs it in watch mode. The =make install= target
  correctly depends on =all= (the binary must be built before installing).
  The one missing integration is a =make fmt= or =make lint= target that
  runs code formatting and static analysis before testing. Adding
  clang-format and cppcheck as optional test dependencies would catch
  style issues and simple bugs before the human review stage. These
  should be non-failing (warnings only, the build continues) to avoid
  frustrating developers with formatting nits.
