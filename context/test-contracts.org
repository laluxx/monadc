[DEC id:dec.tests.default-gate-regression date:2026-06-16 by:llm]
  The default test gate should run only =TEST-TIER: regression=. Known-fail and
  future tests are valuable, but they should not hide fresh regressions behind a
  wall of expected red output.

[DEC id:dec.tests.red-tests-are-assets date:2026-06-16 by:llm]
  Red tests should not be deleted merely because they fail. They should be
  classified. Delete only malformed, duplicate, misleading, or obsolete tests.

#+CAPTION: Fixture metadata schema
| Field | Required | Meaning |
|-------+----------+---------|
| =TEST-ID= | yes | Globally unique stable test ID |
| =TEST-CONTEXT= | yes | Existing =monadc.context.*= heading ID |
| =TEST-PURPOSE= | yes | Human-readable contract |
| =TEST-EXPECT= | yes | Expected action/result class |
| =TEST-TIER= | yes | =regression=, =known-fail=, or =future= |
| =TEST-STATUS= | yes | =active=, =known-fail=, =future=, =archived= |
| =TEST-KNOWN-FAILURE= | for red tiers | Why it is quarantined |
| =TEST-COVERAGE= | recommended | Source functions/paths covered |

[DEC id:dec.tests.context-link-required date:2026-06-16 by:llm]
  Every durable test must connect to a real context heading. Dangling
  =TEST-CONTEXT= links should fail validation because they break the garden graph.

[DEC id:dec.tests.promote-with-proof date:2026-06-16 by:llm]
  To promote a =known-fail= or =future= test to =regression=, update the fixture
  metadata, run the specific test, run the regression gate, and record the
  compiler change in the nearest topic context file.
