#+CAPTION: Coverage dimensions
| Axis | Examples | Required artifact |
|------+----------+-------------------|
| Reader shape | path, quote, arrays, maps, Wisp layout | JSON AST golden |
| Desugar shape | define, match, for, pipes, method sugar | desugar golden |
| Inference | typed binds, dependent claims, bad arity | diagnostic + error class |
| Codegen | runtime value, collections, functions | stdout/run golden |
| Runtime | array/heap/list/map/set mutation | stdout + memory-safe execution |
| CLI/build | command flags, module builds | command output golden |

[DEC id:dec.coverage.stdout-is-not-enough date:2026-06-16 by:llm]
  Stdout-only tests are valid for runtime behavior but insufficient for reader
  and desugar contracts. High-risk syntax changes should include AST and desugar
  goldens.

#+CAPTION: Next coverage investments
| Topic | Why | Minimal new assertions |
|-------+-----+------------------------|
| Wisp =define= | User-facing syntax, high churn | Wisp source + desugar + run |
| Path literals | Reader/infer/runtime value boundary | AST path node + runtime show/use |
| Collection dispatch | Many sugar and runtime paths converge | desugar + run + negative misuse |
| Pattern matching | Clause shape and argument order are fragile | desugar + run |
| Diagnostics | Many red tests are diagnostic mismatch | stable error code/span |
| Test runner tiers | Protects daily workflow | unit tests + inventory validation |
