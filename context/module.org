[OBS id:obs.module.module-resolve-symbol src:module.h:100 conf:high]
  module_resolve_symbol searches the module's own exports and all imports to
  find which qualified name a bare symbol refers to. Returns a malloc'd string
  like "Math:sqrt" or NULL if the symbol is not accessible.

[DEC id:dec.module.symbol-resolution-ambiguity date:2026-06-13 by:codex]
  When the same bare symbol is exported by two different imports, resolution is
  ambiguous. The current implementation returns the first match in import order;
  future work should emit a diagnostic.

[OBS id:obs.module.manifest-create src:module_export.h:68 conf:high]
  manifest_create allocates an ExportManifest for the given module name.

[OBS id:obs.module.manifest-free src:module_export.h:69 conf:high]
  manifest_free frees all entries and the manifest struct.

[OBS id:obs.module.manifest-add-var src:module_export.h:70-72 conf:high]
  manifest_add_var appends a VAR entry with local name, mangled name, type kind,
  and optional array size/element-kind.

[OBS id:obs.module.manifest-add-func src:module_export.h:73-75 conf:high]
  manifest_add_func appends a FUNC entry with return kind and parameter descriptions.

[OBS id:obs.module.manifest-write src:module_export.h:78 conf:high]
  manifest_write serialises the manifest to "<module_name>.mexp" in plain text,
  one entry per line.

[OBS id:obs.module.manifest-read src:module_export.h:81 conf:high]
  manifest_read parses a .mexp file back into an ExportManifest struct.

[OBS id:obs.module.manifest-mangle src:module_export.h:84 conf:high]
  manifest_mangle concatenates module_name and local_name with "__" separator.
  E.g. "Math" + "phi" → "Math__phi". This mangled name is used for linker symbols.

[OBS id:obs.module.type-kind-constants src:module_export.h:10-22 conf:high]
  The MEXP_TYPE_* constants mirror the TypeKind enum from types.h. They are
  duplicated here so module_export.h has zero dependency on the rest of the
  compiler. The manifest stores types as plain integers.

[INF id:inf.module.mexp-format from:obs.module.export-manifest-struct,obs.module.manifest-write,obs.module.manifest-read conf:high]
  The mexp format bridges compilation units: module A writes Foo.mexp, module B
  reads it to emit correct extern declarations for imported symbols. This avoids
  re-parsing the original source and allows incremental compilation.
