[OBS id:obs.typeclass.instance-reg.fn src:typeclass.c:258-503 conf:high]
  =tc_register_instance(reg, ast, ctx)= = — Called when =AST_INSTANCE= is
  encountered (=codegen.c:13068=). Three-step process:
  1. Compile each method: look up body in AST, fall back to default, clone lambda,
     substitute type variables with concrete types (=type_var= → =type_name=,
     =assoc_names= → =assoc_values=), parse function signature via
     =my_type_parse_fn_arrow=, annotate lambda params, compile via =codegen_expr=
     as =(define __impl_Class_Type_method lambda)=, store in =method_funcs=.
  2. Build dictionary global: get or create =__dict_Class_Type= LLVM global struct
     of =method_count= function pointers. Bitcast each method function to =i8*= and
     store at the appropriate GEP slot.
  3. Register in env: for each compiled method, insert an =ENV_FUNC= entry (2-param
     closure ABI, types unknown) so =tc_is_method= dispatch works.

[OBS id:obs.typeclass.dispatch.dict-global src:typeclass.c:438-467 conf:high]
  The dictionary is an LLVM global variable of the struct type
  ="dict.ClassName"=. Each slot is initialized with =LLVMBuildBitCast= of the
  method function (from any calling convention to =i8*=). GEP + Store writes
  each function pointer into the struct.

[OBS id:obs.typeclass.dispatch.strcmp-dispatch src:codegen.c:8905-9121 conf:high]
  Typeclass dispatch in codegen's strcmp handler (layers 2/3) works as follows:
  1. Check if the operation symbol (=op=) is a known method via =tc_method_class=.
  2. If so, attempt to determine the concrete type from argument types.
  3. Find the instance via =tc_find_instance= and the method via =tc_get_method=.
  4. If the type is polymorphic and no instance is resolvable, emit an error
     referencing the missing instance.
