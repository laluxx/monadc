[OBS id:obs.build.test-target src:Makefile conf:high]
  =make test= builds the compiler (=all=) then runs =tests/run.py=. This exercises
  the full pipeline: reader parsing (atoms + sugar TSV entries), codegen
  compile/run fixtures, and language feature tests. It is a codegen test — every
  fixture is compiled to a binary and executed.

[OBS id:obs.build.test-core-target src:Makefile conf:high]
  =make test core= builds the compiler then runs =tests/run_core.py=. This
  compiles and runs the per-module =tests= sections in =core/**/*.mon= (prelude,
  List, Set, Math, text/, etc.). Each module is compiled to a binary and
  executed — this is also a codegen test, since the compiler must generate
  working code for core library definitions.

#+CAPTION: Make Test Targets
| Target    | Command                   | What It Tests                         |
|-----------+---------------------------+---------------------------------------|
| =test=      | =python3 tests/run.py=      | Codegen + reader (atoms, sugar, lang) |
| =test core= | =python3 tests/run_core.py= | Core module per-module tests          |

[DEC id:dec.build.make-fronts-python-tools date:2026-06-13 by:human+codex]
  Dynamic helper scripts in the repository should have Makefile front doors.
  Current script targets include =make generate-asm-tests= for
  =create_asm_tests.py=, =make test-runner= for =tests/test_run.py=,
  =make test-context-visualizer= for =tests/test_context_visualizer.py=, and
  =make fuzzing= / =make test-fuzzing= for =tests/fuzzing/fuzz_codegen.py=.
