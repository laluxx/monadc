[OBS id:obs.context.runtime.file src:runtime.h conf:high]
  This file documents every type, function, and macro in the MonadC runtime
  library (=runtime.h=, =runtime.c=). The runtime is the bridge between the
  compiler's LLVM codegen and the language semantics: it implements the
  concrete value representation, collection data structures (list/set/map),
  numeric towers (int/ratio/bignum), thunk-forcing evaluation, closure
  application, and printing. Every function declared in =runtime.h= has a
  corresponding =get_rt_*= LLVM declaration function in =codegen.c= that makes
  it callable from generated code.

[OBS id:obs.runtime.value-type-enum src:runtime.h:18-34 conf:high]
  =RuntimeValueType= is a 14-variant enum that tags each =RuntimeValue= with
  its concrete type. The enum is used in =switch= dispatch throughout the
  runtime (equality, hashing, printing, collection operations) and is emitted
  as an =i8= in LLVM IR via =get_rt_value_type()=. The tag is always at
  offset 0 of the =RuntimeValue= struct, matching C's first-member guarantee
  so that LLVM GEP with index 0 yields the type tag.

[OBS id:obs.runtime.type-rt-int src:runtime.h:19 conf:high]
  =RT_INT= — 64-bit signed integer. The default numeric representation.
  Constructed via =rt_value_int()=, unboxed via =rt_unbox_int()=.

[OBS id:obs.runtime.type-rt-float src:runtime.h:19 conf:high]
  =RT_FLOAT= — 64-bit IEEE 754 double-precision float.
  Constructed via =rt_value_float()=, unboxed via =rt_unbox_float()=.

[OBS id:obs.runtime.type-rt-char src:runtime.h:20 conf:high]
  =RT_CHAR= — Single character stored as C =char=.
  Constructed via =rt_value_char()=, unboxed via =rt_unbox_char()=.

[OBS id:obs.runtime.type-rt-string src:runtime.h:20 conf:high]
  =RT_STRING= — Heap-allocated C string (=char *=). Owned by the runtime value;
  freed when =rt_value_free()= is called.
  Constructed via =rt_value_string()= (which strdups), unboxed via =rt_unbox_string()=.

[OBS id:obs.runtime.type-rt-symbol src:runtime.h:21 conf:high]
  =RT_SYMBOL= — Interned symbol. Constructed via =rt_value_symbol()=.
  Symbols are heap-allocated and compared by pointer identity after interning.

[OBS id:obs.runtime.type-rt-keyword src:runtime.h:21 conf:high]
  =RT_KEYWORD= — Keyword value (self-evaluating symbolic constant, colon-prefixed
  in source). Constructed via =rt_value_keyword()=. Keywords are also interned and
  compared by pointer identity.

[OBS id:obs.runtime.type-rt-list src:runtime.h:22 conf:high]
  =RT_LIST= — Linked list wrapping a =ConsCell*= via the =RuntimeList= struct.
  Lists are the primary sequential collection in the language, supporting both
  strict and lazy (thunk-backed) cons cells.
  Constructed via =rt_value_list()=, unboxed via =rt_unbox_list()=.

[OBS id:obs.runtime.type-rt-ratio src:runtime.h:22 conf:high]
  =RT_RATIO= — Exact rational number stored as numerator/denominator pair of
  =int64_t=. Supports arithmetic via =rt_ratio_add/sub/mul/div()=.
  Constructed via =rt_value_ratio()=.

[OBS id:obs.runtime.type-rt-array src:runtime.h:23 conf:high]
  =RT_ARRAY= — Heap-allocated contiguous array of =RuntimeValue*= pointers.
  Indexed by =rt_array_get/set()= with bounds checking.
  Constructed via =rt_value_array()=.

[OBS id:obs.runtime.type-rt-nil src:runtime.h:23 conf:high]
  =RT_NIL= — The empty value, representing logical nothingness. Used as the
  terminator of proper lists and as the absence-of-value sentinel.
  Constructed via =rt_value_nil()=, tested via =rt_value_is_nil()=.

[OBS id:obs.runtime.type-rt-thunk src:runtime.h:24 conf:high]
  =RT_THUNK= — Lazy evaluation wrapper. A thunk carries a function pointer
  (=ThunkFn=) and an environment; the first access via =rt_force()= evaluates
  and caches the result.
  Constructed via =rt_value_thunk()=.

[OBS id:obs.runtime.type-rt-bignum src:runtime.h:24 conf:high]
  =RT_BIGNUM= — Arbitrary-precision integer backed by GMP's =mpz_t=.
  Used when integer operations overflow =int64_t=.
  Constructed via =rt_value_bignum_from_str/i64()=, arithmetic via =rt_bignum_*()=.

[OBS id:obs.runtime.type-rt-set src:runtime.h:25 conf:high]
  =RT_SET= — Unordered collection of unique =RuntimeValue*= entries, implemented
  as an open-addressing hash set.
  Constructed via =rt_value_set()=, unboxed via =rt_unbox_set()=.

[OBS id:obs.runtime.type-rt-map src:runtime.h:25 conf:high]
  =RT_MAP= — Key-value dictionary mapping =RuntimeValue*= keys to
  =RuntimeValue*= values, implemented as an open-addressing hash map.
  Constructed via =rt_value_map()=, unboxed via =rt_unbox_map()=.

[OBS id:obs.runtime.type-rt-closure src:runtime.h:26 conf:high]
  =RT_CLOSURE= — First-class function value carrying a compiled function pointer,
  captured environment, arity, and optional debug name.
  Constructed via =rt_value_closure()= / =rt_value_closure_named()=.

[OBS id:obs.runtime.runtime-value-struct src:runtime.h:66-92 conf:high]
  =RuntimeValue= is a tagged union of 14 variants. The =type= field is an
  =enum RuntimeValueType= (stored as =int=). The =data= union packs all
  representation types: scalars inline (=int64_t=, =double=, =char=),
  pointers to heap allocations (=char*=, =mpz_t=, collection structs), and
  nested structs for ratio and array. The struct is the universal currency
  of the runtime — every function accepts and returns =RuntimeValue*=.

[THINK id:think.runtime.runtime-value-layout conf:high]
  The tagged union layout is the simplest possible representation for a
  dynamically-typed runtime. LLVM codegen accesses the type tag at offset 0
  (matching C struct first-member rule) and uses =getelementptr= with the
  appropriate LLVM type (derived from =get_rt_value_type()=) to read/write
  the data union. The LLVM type for =RuntimeValue= is constructed in
  =codegen.c= as a packed struct matching the C layout exactly.

[OBS id:obs.runtime.runtime-thunk src:runtime.h:49-54 conf:high]
  =RuntimeThunk= implements lazy evaluation. The =fn= field is a =ThunkFn=
  (=RuntimeValue *(*)(void *)=) that computes the value. The =env= pointer
  carries the closure environment. On first call to =rt_force()=, the thunk
  evaluates =fn(env)=, stores the result in =value=, and sets =forced= to 1.
  Subsequent calls return the cached =value=.

[OBS id:obs.runtime.fn-rt-thunk-of-value src:runtime.h:171 conf:high]
  =rt_thunk_of_value(RuntimeValue *val)= — Wraps an already-computed value
  in a trivial thunk (an "eager thunk") whose =fn= immediately returns =val=.
  Used when a lazy position needs a value that is already known.

[OBS id:obs.runtime.fn-rt-thunk-create src:runtime.h:172 conf:high]
  =rt_thunk_create(ThunkFn fn, void *env)= — Allocates a new thunk with the
  given computation and environment. The thunk starts unforced; the first
  =rt_force()= triggers evaluation.

[OBS id:obs.runtime.fn-rt-force src:runtime.h:173 conf:high]
  =rt_force(RuntimeThunk *thunk)= — Forces evaluation of a thunk: if
  =thunk->forced= is 0, calls =thunk->fn(thunk->env)=, caches the result in
  =thunk->value=, sets =thunk->forced= to 1, and returns the value. If
  already forced, returns the cached =thunk->value= directly. This is the
  core of the lazy evaluation machinery.

[OBS id:obs.runtime.cons-cell src:runtime.h:116-126 conf:high]
  =ConsCell= is the fundamental building block of linked lists. Each cell
  inlines head and tail thunks directly (fn/env/val/forced quadruples)
  rather than storing pointers to separate thunk objects. A NULL =tail_val=
  with =tail_forced=1 signals the end of a list. Cells are arena-allocated
  from =g_eval_arena=, making them extremely fast to allocate and freeing
  them automatically at arena reset.

[THINK id:think.runtime.cons-cell-inlined-thunks conf:high]
  Inlining thunks into the cons cell (rather than storing pointers to
  separate =RuntimeThunk= structs) is an optimization that reduces pointer
  chasing and allocation overhead. A cons cell with both head and tail forced
  occupies ~64 bytes (8 ThunkFn* + 8 void* + 8 RuntimeValue* + 4 int for
  each of head and tail). This layout matches Baker's lazy-cons semantics
  where every field is potentially lazy.

[OBS id:obs.runtime.runtime-set-struct src:runtime.h:144-149 conf:high]
  =RuntimeSet= is an open-addressing hash set with linear probing. Buckets
  hold =RuntimeValue*= pointers. Capacity is always a power of two. The
  load factor threshold is 0.7; when crossed, the set rehashes into a larger
  bucket array. Tombstones mark deleted slots to preserve probe chains.
  Hashing uses =rt_hash_value()= (a static function, xorshift-based with
  type discrimination).

[OBS id:obs.runtime.runtime-map-struct src:runtime.h:161-166 conf:high]
  =RuntimeMap= is an open-addressing hash map with linear probing. Each entry
  is a =RuntimeMapEntry= (key/value pair of =RuntimeValue*= pointers). Same
  power-of-two sizing, 0.7 load factor, and tombstone strategy as
  =RuntimeSet=. Keys are hashed via the same =rt_hash_value()= function.

[OBS id:obs.runtime.fn-rt-hash-value src:runtime.h:211 conf:high]
  =rt_hash_value(RuntimeValue *v)= — Static hash function used by both set
  and map. Combines the =RuntimeValueType= tag with the value content via
  xorshift64. Integers hash by value, strings by content, collections by
  element hashes, etc.

  Value construction functions allocate a =RuntimeValue=, set the type tag,
  and initialize the data union. Arena-allocated types (int, float, char,
  list, nil, thunk) are allocated from =g_eval_arena= and are implicitly

  =rt_value_free()=. The comment in the header marks this distinction.

[THINK id:think.runtime.coverage-status conf:high]
  The runtime library covers the full set of value types required by the
  language specification. Every =RuntimeValueType= variant maps to a language
  construct or primitive type. The arena/heap allocation split is a performance
  architecture: hot-path values (int, float, char, list spines, nil, thunks)
  avoid malloc/free overhead through arena bump-allocation, while long-lived
  or variable-size values (strings, symbols, keywords, ratios, arrays, sets,
  maps) use explicit heap allocation with reference counting or explicit free.
  The LLVM declaration layer (~65 get_rt_* functions) ensures every runtime
  function is accessible from generated code through a consistent pattern.
