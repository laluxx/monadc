[OBS id:obs.asm.register-allocator-struct src:asm.h:10-15 conf:high]
  RegisterAllocator stores two fixed arrays of register name strings: int_regs[6]
  for general-purpose registers (%rdi, %rsi, %rdx, %rcx, %r8, %r9) and
  float_regs[8] for SSE registers (%xmm0-%xmm7), plus integer counters for each.

[OBS id:obs.asm.reg-alloc-init src:asm.h:18 conf:high]
  reg_alloc_init zeroes the allocator and fills in the SysV ABI register names.

[OBS id:obs.asm.reg-alloc-get src:asm.h:21 conf:high]
  reg_alloc_get returns the register name for a parameter at a given index and
  type. Integer-typed parameters consume from int_regs, float-typed from float_regs.
  Returns NULL if all registers of that class are exhausted.

[THINK id:think.asm.register-allocator-naked conf:high]
  The RegisterAllocator is used primarily for naked functions where the asm body
  directly names hardware registers. Non-naked functions use LLVM constraint
  strings ("r", "=r", ",0") instead, leaving register allocation to LLVM's
  register allocator. This split reflects the two asm modes: bare-metal control
  (naked) vs portable constraint-based inline asm (non-naked).

[OBS id:obs.asm.preprocess-asm src:asm.h:41 conf:high]
  preprocess_asm takes an AST_ASM node and an AsmContext, then:
  1. Splits the flat token stream into individual instructions by mnemonic
  2. Replaces parameter-name symbols with registers (naked) or LLVM placeholders
  3. Escapes numeric immediates as raw text
  4. Preserves string operands (e.g. syscall) as-is
  Returns a malloc'd array of AsmInstruction with count in out_count.

[OBS id:obs.asm.free-asm-instructions src:asm.h:44 conf:high]
  free_asm_instructions frees every operand and mnemonic string in the instruction
  array, then frees the array itself.

[OBS id:obs.asm.codegen-inline-asm src:asm.h:46-53 conf:high]
  codegen_inline_asm lowers preprocessed AsmInstruction arrays into LLVM IR via
  LLVMGetInlineAsm. For non-naked functions it generates constraint strings:
  return value gets "=r" (int-like) or "=x" (float), first input is tied to
  output with ",0", remaining inputs use ",r". For naked functions, instructions
  are emitted directly without LLVM constraint wiring and the caller must handle
  prologue/epilogue.

[OBS id:obs.asm.codegen-size-suffix src:asm.h:46-53 conf:high]
  Bare integer mnemonics in non-naked mode infer size suffixes (b, w, l, q) from
  LLVM operand widths. Two-operand arithmetic/logical instructions with
  placeholder destinations are reordered for AT&T syntax.

  assembly processing and LLVM integration.

#+CAPTION: Inline Assembly Category Contract
| Field | Contract |
|-------+----------|
| Owned phase | ASM string preprocessing, constraint parsing, register allocation for inline asm operands, LLVM inline asm CallInst generation. |
| Inputs | AST_ASM nodes with asm string, constraint lists (input/output/clobber), operand expressions, naked flag. |
| Outputs | LLVM inline assembly CallInst with constraint wiring, LLVMGetInlineAsm handle, side-effect marker for non-naked asm. |
| Invariants | Register constraints parsed correctly from LLVM constraint syntax; input/output operands matched 1:1 to constraints; naked asm emits no prologue/epilogue; size suffixes propagate correctly for non-naked asm. |
| Test hooks | Inline assembly codegen tests, constraint parsing tests, naked vs non-naked emission tests, LLVM IR verification for asm blocks. |
| Failure symptoms | LLVM constraint violation at codegen, operand count mismatch, naked asm register conflict with compiler-generated prologue, size suffix inference failure. |
| Context neighbors | codegen-ops, language, codegen, tests |

[DEC id:dec.asm.category-contract-owner date:2026-06-18 by:big-pickle]
  Failures matching the symptoms above should start triage in this category, then
  follow the listed neighbors only when the local contract does not explain the
  issue.
