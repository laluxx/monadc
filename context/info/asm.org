[[file:index.org][<- Back to Index]] · [[file:ffi.org][C FFI]] · [[file:layouts.org][Layout Types]] · [[file:performance.org][Performance]] · [[file:from-c.org][From C]] · [[file:types.org][Types]] · [[file:control.org][Control Flow]]

Monad provides inline x86-64 assembly blocks for low-level control
through the =asm.c= preprocessing and codegen pipeline
([[file:../asm.c]], [[file:../asm.h]]). The reader recognizes =(asm
mnemonic operand ...)= inside an S-expression and creates an
=AST_ASM= node ([[file:../reader.c::3889]]). Wisp-style definitions
expand to the same form.

Test coverage: 60 test fixtures in [[file:../tests/codegen/asm/][tests/codegen/asm/]]
covering syscall operands, int/float arithmetic, SSE, conditional moves,
shifts, signed division, xchg, NOP, multi-parameter constraints, and
Wisp-ASM mixing. Generated by =create_asm_tests.py= and
=create_asm_tests_extra.py=. Non-naked asm is verified by
=rt_asm_function_nonaked.mon= printing 30 for =(add 10 20)=.
The test architecture is documented at
[[file:../context/tests-arch.org][context/tests-arch.org]].

For the C FFI approach to interop see [[file:ffi.org][C FFI]]. For the
underlying LLVM codegen see [[file:../context/codegen.org][context/codegen.org]].

* 1. Basic Syntax

#+BEGIN_SRC monad
(asm mov 1 %rax add %rbx %rax ret)
#+END_SRC

#+BEGIN_SRC wisp
asm mov 1 %rax add %rbx %rax ret
#+END_SRC

Multiple instructions in one block. The preprocessor splits them by
mnemonic boundaries. Each =(asm ...)= form is lowered to an
=LLVMGetInlineAsm= call at codegen time
([[file:../asm.h::46]]).

* 2. Operand Types

| Syntax | Description                                   |
|--------+-----------------------------------------------|
| =%reg=   | Named register (=%rax=, =%rdi=, =%rbx=, ...)        |
| =$N=     | Immediate integer value (escaped as raw text) |
| =name=   | Monad function parameter (LLVM placeholder)   |
| ="str"=  | Raw string operand (passed through as-is)     |

Parameter-name symbols in non-naked functions are mapped to LLVM
constraint placeholders (=$0=, =$1=, …). In naked functions they are
mapped to SysV ABI registers instead. Numeric immediates (e.g. =1=)
are escaped (prefixed with =$=). String operands (e.g. ="syscall"=)
are preserved verbatim.

See [[file:../context/asm.org][context/asm.org]] for the operand preprocessing
algorithm and [[file:../asm.h][asm.h]] for the =AsmOperand= type.

* 3. Architecture ([[file:../asm.h]])

The assembly pipeline is:

1. =preprocess_asm= ([[file:../asm.h::41]]) takes an =AST_ASM= node
   and an =AsmContext=, splits the flat token stream into individual
   instructions by mnemonic recognition, maps parameter symbols to
   registers or LLVM placeholders, escapes numeric immediates, and
   preserves string operands as raw text.
2. =codegen_inline_asm= ([[file:../asm.h::46]]) lowers the preprocessed
   instruction array to LLVM IR via =LLVMGetInlineAsm=.

** 3.1 AsmInstruction ([[file:../asm.h::24]])

One instruction: a mnemonic string (e.g. "add", "mov") and a dynamic
array of operand strings (registers, immediates, or raw text).

** 3.2 AsmContext ([[file:../asm.h::31]])

Carries the function's =EnvParam= array, parameter count, optional
=RegisterAllocator=, and a boolean =naked= flag controlling
prologue/epilogue generation.

** 3.3 RegisterAllocator ([[file:../asm.h::10]])

Maps function parameters to SysV AMD64 ABI registers:

- =int_regs[6]=: =%rdi=, =%rsi=, =%rdx=, =%rcx=, =%r8=, =%r9=
- =float_regs[8]=: =%xmm0=–=%xmm7=

See [[file:../context/asm.org::asm-register-allocator][context/asm.org — RegisterAllocator]]
for the full register mapping and allocation strategy.

* 4. Naked Functions

Use =:naked True= to suppress function prologue/epilogue. A naked body
must contain at least one =(asm ...)= block. Parameters map directly
to SysV ABI registers.

#+BEGIN_SRC monad
(define [sys-write :: Int -> String -> Int -> Int])
  :naked True
  fd buf len -> (asm mov 1 %rax mov fd %rdi
                      (mov buf %rsi mov len %rdx)
                      (syscall ret)
#+END_SRC

#+BEGIN_SRC wisp
define [sys-write :: Int -> String -> Int -> Int]
  :naked True
  fd buf len -> (asm mov 1 %rax mov fd %rdi
                      mov buf %rsi mov len %rdx
                      syscall ret
#+END_SRC

Parameter mapping for naked functions follows the =RegisterAllocator=
(%rdi for arg 0, %rsi for arg 1, %rdx for arg 2, %rcx for arg 3,
%r8 for arg 4, %r9 for arg 5). Float-typed parameters use %xmm0–%xmm7
([[file:../asm.h::10]]).

KNOWN BUG ([[file:../context/language.org::317]]): Naked asm compiles
but does not propagate results through LLVM constraints. The raw asm
output has no constraint wiring — =add y %rax= returns the first
parameter only, not the sum. Naked asm is suitable for syscall
invocations where the return value comes from %rax but not for
arithmetic that needs result propagation.

See [[file:../context/asm.org][context/asm.org]] for implementation details and
[[file:../context/language.org][context/language.org]] for the known bug
tracking.

* 5. Non-naked Functions

Non-naked asm lowers through =LLVMGetInlineAsm= with LLVM constraint
strings ([[file:../asm.h::46]]):

- Return value: ="=r"= for integer-like, ="=x"= for float
- First input: tied to output with =,0=
- Remaining inputs: =,r=

#+BEGIN_SRC monad
(define [add-one :: Int -> Int])
  x -> (asm inc x %rax
            (mov %rax x)
#+END_SRC

#+BEGIN_SRC wisp
define [add-one :: Int -> Int]
  x -> (asm inc x %rax
            mov %rax x
#+END_SRC

The first parameter is tied to the output (=,0=), later parameters use
=,r=. Two-operand arithmetic/logical instructions with placeholder
destinations are reordered for AT&T syntax.

See [[file:../context/codegen.org][context/codegen.org]] for the LLVM
constraint lowering and [[file:syntax.org][Syntax]] for the =(asm ...)=
special form.

** 5.1 Size-suffix Inference

Bare integer mnemonics in non-naked mode infer size suffixes from LLVM
operand widths ([[file:../context/language.org::328]]):

| Operand Width | Suffix | Example |
|---------------+--------+---------|
| 8 bits        | =b=      | =addb=    |
| 16 bits       | =w=      | =addw=    |
| 32 bits       | =l=      | =addl=    |
| 64 bits       | =q=      | =addq=    |

For =Int= parameters (64-bit on x86-64) this emits =q=, so =(asm add
x y)= lowers to =addq $2, $1= under constraints =r,0,r=
([[file:../context/language.org::331]]). Verified by
=rt_asm_function_nonaked.mon= printing 30 for =(add 10 20)=.

#+BEGIN_SRC monad
(define [add :: Int -> Int -> Int])
  x y -> (asm add %rax x y)
#+END_SRC

#+BEGIN_SRC wisp
define [add :: Int -> Int -> Int]
  x y -> asm add %rax x y
#+END_SRC

For =Int= args this emits =addq $2, $1= automatically.

* 6. Multi-instruction Splitting

The preprocessor splits flat =(asm ...)= token streams into individual
instructions by recognizing mnemonic boundaries. The 60 test fixtures
in [[file:../tests/codegen/asm/]] cover:

- Syscall-shaped =String= operands
- =Int= placeholder arithmetic
- Immediate escaping
- Multi-instruction splitting
- Unary =inc=/=dec=/=neg=/=not=
- Shifts and rotates with immediate counts
- Six-argument placeholder constraints
- =Char= and =Float= return types
- SSE scalar add/multiply
- Conditional moves
- Signed division
- =xchg=
- No-op placement

See [[file:../context/asm.org][context/asm.org]] for the splitting algorithm
and [[file:../context/language.org][context/language.org]] for instruction
recognition rules.

* 7. Pointer Sugar for Syscall Buffers

=type_from_name= ([[file:../context/types.org::1547]]) accepts
star-prefixed pointer sugar such as =*U8= and converts it to
=Pointer :: U8=:

#+BEGIN_SRC monad
(define [sys-write2 :: Int -> *U8 -> Int -> Int])
  :naked True
  fd buf len -> (asm mov 1 %rax mov fd %rdi
                      (mov buf %rsi mov len %rdx)
                      (syscall ret)
#+END_SRC

#+BEGIN_SRC wisp
define [sys-write2 :: Int -> *U8 -> Int -> Int]
  :naked True
  fd buf len -> (asm mov 1 %rax mov fd %rdi
                      mov buf %rsi mov len %rdx
                      syscall ret
#+END_SRC

String literals still type-check as =String=, so direct
String-to-=*U8= call coercion remains a separate FFI bridge concern
([[file:../context/language.org::397]]). See [[file:ffi.org][C FFI]] for
the FFI approach and [[file:casting.org][Casting]] for type conversion
functions.

* 8. Wisp Form

#+BEGIN_SRC monad
(define [add-one :: Int -> Int])
  x -> asm inc x %rax
           (mov %rax x)
#+END_SRC

#+BEGIN_SRC wisp
define [add-one :: Int -> Int]
  x -> asm inc x %rax
           mov %rax x
#+END_SRC

In Wisp syntax the =(asm ...)= parentheses are omitted — the =asm=
keyword introduces a block whose body is parsed as a flat token stream
until the next definition or closing form. See [[file:wisp.org][Wisp
Syntax]] and [[file:advanced-wisp.org][Advanced Wisp]] for full Wisp
integration details.

* Implementation Notes

The inline assembly pipeline touches three compiler components:

- **Reader**: =reader.c= around line 3889 recognises =(asm ...)= forms
  and produces =AST_ASM= nodes
  ([[file:../context/asm.org][context/asm.org — ASM AST node]]).
- **Preprocessor**: =preprocess_asm= (=asm.h:41=) splits the flat token
  stream into individual =AsmInstruction= entries, maps parameter symbols
  to registers or LLVM placeholders, and escapes numeric immediates
  ([[file:../context/asm.org][context/asm.org — preprocessing]]).
- **Codegen**: =codegen_inline_asm= (=asm.h:46=) lowers the instruction
  array to =LLVMGetInlineAsm= with constraint strings
  ([[file:../context/codegen.org][context/codegen.org — inline asm codegen]]).
- **Register allocator**: =RegisterAllocator= (=asm.h:10=) maps the first
  6 integer params to =%rdi=–=%r9= and 8 float params to =%xmm0=–=%xmm7=
  following the SysV AMD64 ABI
  ([[file:../context/asm.org::asm-register-allocator][context/asm.org — register allocator]]).

Key data structures: =AsmInstruction= (=asm.h:24=) holds a mnemonic and
operand array; =AsmContext= (=asm.h:31=) carries the =EnvParam= array,
naked flag, and register allocator.

* 9. Known Gaps

- Naked asm does not propagate results (no LLVM constraint wiring)
  — tracked in [[file:../context/language.org::317][context/language.org]].
- String-to-=*U8= call coercion for syscall convenience is not
  implemented — use an explicit FFI bridge
  ([[file:../context/language.org::397]]).
- No support for =asm goto= or =asm volatile= qualifiers.
- Size-suffix inference only works in non-naked mode.
- No ARM or AArch64 support — x86-64 only (SysV AMD64 ABI).

* Key Points

- =(asm ...)= blocks support inline x86-64 assembly with register naming.
- **Naked mode** (=:naked True=): no prologue/epilogue, direct SysV
  register mapping — best for syscalls. Does not propagate results.
- **Non-naked mode**: LLVM constraint wiring (="=r"=, =,0=, =,r=),
  size-suffix inference (=b=, =w=, =l=, =q=), result propagation.
- Operand types: =%reg= (named register), =$N= (immediate), =name=
  (parameter symbol), ="str"= (raw text).
- Pointer sugar =*U8= enables syscall buffer types.
- Tested by 60 fixtures in =tests/codegen/asm/=.
