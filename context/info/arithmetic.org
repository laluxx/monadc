All arithmetic operators work on Int, Float, and mixed Int/Float
operands. Most are variadic (1..N arguments). For comparison operators,
see [[file:equality.org][Equality]]. For numeric type details, see
[[file:numeric.org][Numeric Types]].

MonadC's arithmetic follows mathematical convention: operations
promote "up" the numeric hierarchy (Int → Float) when types are mixed,
and most operators accept multiple arguments for concise expression.

** Complete example:

#+BEGIN_SRC monad
;; Save as arithmetic-demo.mon && monadc arithmetic-demo.mon && ./arithmetic-demo
(module Main)

;; Basic arithmetic
(show (+ 1 2 3))         ;; 6
(show (- 10 3 2))        ;; 5
(show (* 2 3 4))         ;; 24
(show (/ 17 5))          ;; 3 (integer division)
(show (% 10 3))          ;; 1

;; Mixed Int/Float promotes to Float
(show (+ 1 2.5))         ;; 3.5
(show (* 3 0.5))         ;; 1.5

;; Unary negation
(show (- 5))             ;; -5

;; Bitwise operations
(show (& 0xFF 0x0F))     ;; 15
(show (<< 1 8))          ;; 256
(show (~ 0))             ;; -1
#+END_SRC

** Common Pitfalls

- **Integer division truncates**: =(/ 17 5)= yields =3=, not =3.4=.
  Use float literals or =(Float x)= casting for float division.
- **Modulo works on Int only**: =(% 3.5 2.0)= is a type error.
- **Superscript =^= has no codegen**: =x²= desugars at reader time
  but =^= is not callable from paren-mode. Use explicit =(* x x)= or
  the [[file:math-module.org][Math Module]].
- **Bitwise shifts by negative amounts or > bit width are
  implementation-defined**: The compiler does not guard against
  undefined behavior in shift operations.
- **Zero-argument =(+)= returns 0, =(*)= returns 1**: This follows
  mathematical convention (identity elements), which may surprise
  users expecting an error.

* 1. Basic Operations

| Operator | Example  | Description    | Arity |
|----------+----------+----------------+-------|
| +        | =(+ 1 2)=  | Addition       | 0..N  |
| -        | =(- 5 2)=  | Subtraction    | 1..N  |
| *        | =(* 3 4)=  | Multiplication | 0..N  |
| /        | =(/ 10 2)= | Division       | 1..N  |
| %        | =(% 10 3)= | Modulo         | 2     |

#+BEGIN_SRC monad
(+ 1 2 3)                       ;; 6 — variadic sum
(- 10 3 2)                      ;; 5 — 10 - 3 - 2
(* 2 3 4)                       ;; 24
(/ 10 2)                        ;; 5

(- 5)                           ;; -5 — negation
(/ 7 2)                         ;; 3 (integer division)
(/ 7.0 2)                       ;; 3.5 (float division)
#+END_SRC

#+BEGIN_SRC wisp
+ 1 2 3
- 10 3 2
* 2 3 4
/ 10 2

- 5
/ 7 2
/ 7.0 2
#+END_SRC

Mixed Int/Float promotes to Float:
#+BEGIN_SRC monad
(+ 1 2.5)                        ;; 3.5 — Float
(* 3 0.5)                       ;; 1.5 — Float
#+END_SRC

#+BEGIN_SRC wisp
+ 1 2.5
* 3 0.5
#+END_SRC

Unary negation =(- x)= works on both Int and Float. Division follows
the type of the first argument: integer division truncates toward zero.
For casting between numeric types, see [[file:casting.org][Casting]].

** Implementation Notes

- All arithmetic dispatch via strcmp handlers in codegen
  ([[file:../context/codegen-ops.org::124]])
- Promotion rules: Int + Float → Float
  ([[file:../context/codegen-ops.org::124]])
- Variadic operations chain left-to-right with the same associativity

** Key Points

- Zero-argument =(+)= returns 0; zero-argument =(*)= returns 1
- =(/ x)= divides 1 by x (reciprocal)
- =(/ a b c)= = =(/ (/ a b) c)= (left-associative)
- Modulo works on Int operands only

* 2. Bitwise Operations

| Operator   | Example           | Description        |
|------------+-------------------+--------------------|
| &          | =(bit-and x 0xFF)= | Bitwise AND        |
| bit-or / ∨ | =(bit-or a b)=    | Bitwise OR         |
| bit-xor / ⊕| =(bit-xor a b)=   | Bitwise XOR        |
| ~          | =(~ x)=           | Bitwise NOT        |
| <<         | =(<< x 2)=        | Left shift         |
| >>         | =(>> x 2)=        | Right shift        |
| >>>        | =(>>> x 2)=       | Unsigned right shift |

#+BEGIN_SRC monad
(& 0xFF 0x0F)                    ;; 0x0F
(<< 1 8)                        ;; 256
(~ 0)                           ;; -1 (all bits set)
(bit-or 0xF0 0x0F)              ;; 0xFF
(bit-xor 0xFF 0x0F)             ;; 0xF0
#+END_SRC

#+BEGIN_SRC wisp
& 0xFF 0x0F
<< 1 8
~ 0
bit-or 0xF0 0x0F
bit-xor 0xFF 0x0F
#+END_SRC

Bitwise operators work on Int operands only. For hexadecimal literal
syntax, see [[file:numeric.org][Numeric Types]].

** Implementation Notes

- Bitwise dispatch via strcmp
  ([[file:../context/codegen-ops.org::159]])
- Shifts by negative amounts or beyond bit width produce
  implementation-defined results

* 3. Superscript Sugar

| Written | Desugars to |
|---------+-------------|
| =x²=      | =(^ x 2)=     |
| =x³=      | =(^ x 3)=     |
| =xⁿ=      | =(^ x n)=     |

=^= has no strcmp handler — not callable from paren-mode
([[file:../context/language.org::404]]). See
[[file:../context/language.org][context/language.org]] for the full superscript
dispatch table.

** Key Points

- Superscript notation is sugar recognised by the reader, not by the
  codegen dispatch layer
- Only works in wisp mode; superscript =^= is not available as a
  paren-mode function call

* 5. Advanced Arithmetic

Beyond the basic operators, MonadC provides advanced arithmetic in the
[[file:math-module.org][Math Module]]:

#+BEGIN_SRC monad
(use Math)

(Math/gcd 12 8)                  ;; 4
(Math/factorial 5)               ;; 120
(Math/sin 0.0)                   ;; 0.0
(Math/pi)                        ;; 3.14159...
#+END_SRC

#+BEGIN_SRC wisp
use Math

Math/gcd 12 8
Math/factorial 5
Math/sin 0.0
Math/pi
#+END_SRC
