Test coverage: =show= is exercised by every runtime test fixture — it
is the standard output mechanism used across the 423 codegen tests.

=show= prints any value to stdout. Through the =Show= typeclass, every
type — builtin, ADT, layout, or function — has a canonical string
representation. You can literally show anything.

For the =Show= typeclass internals, see [[file:../context/codegen-ops.org][context/codegen-ops.org]].
For the output semantics, see [[file:../context/language.org][context/language.org]].

This page covers the =show= function, the =Show= typeclass, =assert-eq=,
and display behavior for all Monad types including ADTs, layouts, and paths.

* 1. show (universal)

Print one or more values to stdout, separated by spaces:

#+BEGIN_SRC monad
(show 42)                        ;; "42"
(show "hello world")             ;; "hello world"
(show (+ 1 2) "is three")       ;; "3 is three"
(show x y z)                    ;; variadic, any types
#+END_SRC

#+BEGIN_SRC wisp
show 42
show "hello world"
show (+ 1 2) "is three"
show x y z
#+END_SRC

Variadic — takes 1..N arguments of any types. Prints each followed by
a space, then a newline.

*** Key Points
- Universal: works on every type with no manual instance required
- Variadic: accepts any number of arguments
- Prints each value followed by a space, then a trailing newline

See also: [[file:strings.org][Strings]]

* 2. All types are showable

=show= dispatches through the =Show= typeclass, which every type
satisfies. This covers every value in the language:

| Type     | Example output            |
|----------+---------------------------|
| Int      | =42=, =-7=                |
| Float    | =3.14=, =1e10=            |
| Char     | =a=, =\n=                 |
| String   | =hello world= (no quotes) |
| Bool     | =True=, =False=           |
| Symbol   | =foo=, =:keyword=         |
| Keyword  | =:keyword=                |
| List     | =(1 2 3)=                 |
| Array    | =[1 2 3]=                 |
| Set      | ={1 2 3}=                 |
| Map      | =#{:a 1 :b 2}=            |
| Ratio    | =1/3=, =50%=              |
| Hex      | =0xFF=                    |
| Oct      | =0o77=                    |
| Bin      | =0b1010=                  |
| Path     | File content *or* =ls -la= of directory |
| Data/ADT | =(Just 42)=, =Nothing=    |
| Layout   | =(Point 3.0 4.0)=         |

See [[file:types.org][Types]], [[file:collections.org][Collections]],
[[file:numeric.org][Numeric Types]], and [[file:paths.org][Path Literals]].

** 2.1 ADTs print in constructor syntax

Algebraic data types display their constructor name and fields:

#+BEGIN_SRC monad
(show (Just 42))                 ;; "(Just 42)"
(show Nothing)                   ;; "Nothing"
(show (Pair 1 "one"))            ;; "(Pair 1 one)"
(show Red)                       ;; "Red"
(show (Node Leaf 1 Leaf))        ;; "(Node Leaf 1 Leaf)"
#+END_SRC

#+BEGIN_SRC wisp
show (Just 42)
show Nothing
show (Pair 1 "one")
show Red
show (Node Leaf 1 Leaf)
#+END_SRC

Every constructor and every field is included — no manual show
instance needed. See [[file:data.org][Data Types]] and [[file:maybe.org][Maybe]].

*** Key Points
- ADTs print as =(ConstructorName field1 field2 ...)=
- =Nothing= (no fields) prints without parentheses
- No manual show instances required — derived automatically from the ADT definition

** 2.2 Layouts show field names

Layout types display their type name and field values:

#+BEGIN_SRC monad
(define p (Point 3.0 4.0))
(show p)                         ;; "(Point 3.0 4.0)"

(define r (Rect (Point 0 0) (Point 10 10)))
(show r)                         ;; "(Rect (Point 0 0) (Point 10 10))"
#+END_SRC

#+BEGIN_SRC wisp
define p (Point 3.0 4.0)
show p

define r (Rect (Point 0 0) (Point 10 10))
show r
#+END_SRC

See [[file:layouts.org][Layout Types]].

** 2.3 Functions show their type

#+BEGIN_SRC monad
(show +)                         ;; "<function: +>"
(show id)                        ;; "<function: id>"
#+END_SRC

See [[file:lambda.org][Lambda]] and [[file:core.org][Core Library]].

** 2.4 Paths show file contents or directory listing

Calling =show= on a =Path= value reads the filesystem:

#+BEGIN_SRC monad
(show (path "./README.org"))     ;; prints file contents to stdout
(show (path "./src/"))           ;; prints directory listing (ls -la style)
#+END_SRC

#+BEGIN_SRC wisp
show path "./README.org"
show path "./src/"
#+END_SRC

For a regular file, =show= writes its contents verbatim to stdout. For
a directory, it prints an =ls -la=-style listing with permissions,
size, and modification time for each entry.

Non-existent paths print an error diagnostic. See [[file:paths.org][Path Literals]]
and [[file:error-handling.org][Error Handling]].

*** Key Points
- =show= on a file path =cat=s the content to stdout
- =show= on a directory path produces an =ls -la= style listing
- Non-existent paths produce an error diagnostic

* 3. assert-eq

Test assertion that prints both values on failure via =show=:

#+BEGIN_SRC monad
(assert-eq expected actual)
(assert-eq (+ 1 1) 2 "1+1=2")
#+END_SRC

#+BEGIN_SRC wisp
assert-eq expected actual
assert-eq (+ 1 1) 2 "1+1=2"
#+END_SRC

Codegen evaluates both arguments and calls =rt_equal_p=. On failure,
prints both values via =show= ([[file:../codegen.c::6431]]). See
[[file:testing.org][Testing]].

*** Key Points
- =assert-eq= compares two values; prints both via =show= on failure
- Equivalent to xUnit assertEqual
- =show= is the backbone of the testing infrastructure

See also: [[file:testing.org][Testing]]
