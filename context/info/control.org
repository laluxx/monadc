control forms begins at [[file:../codegen.c::7471][codegen.c:7471]].

Test coverage: control flow is exercised by =rt_control_all.mon=,
=rt_control_comp.mon=, and 35+ diagnostic error fixtures.

This page provides an overview of Monad's control flow forms. Each
group has a dedicated reference page with detailed documentation.

* 3. Expression-Oriented Control

Every control form returns a value, making them composable:

#+BEGIN_SRC monad
(define result
  (if (> x 0)
    (begin (show "positive") x)
    (begin (show "non-positive") (- x))))
#+END_SRC

#+BEGIN_SRC wisp
define result
  if (> x 0)
    begin
      show "positive"
      x
    begin
      show "non-positive"
      - x
#+END_SRC

This expression orientation is consistent across all forms — see
[[file:../context/language.org::control-flow-constructs][context/language.org — Control Flow Constructs]].

#+BEGIN_QUOTE
Key Points:
- All control forms are expressions (they return values)
- Forms with strcmp handlers work in paren-mode
- Composition: nest control forms to build complex logic
- =if=, =when=, =unless=, =while=, =for=, =begin=, =do=, =with= all have handlers
#+END_QUOTE

* 4. Known Codegen Gaps

The following forms have no strcmp dispatch handler:

| Form   | Example                         | Status                                    |
|--------+---------------------------------+-------------------------------------------|
| cond   | =(cond [t1 e1] [t2 e2] [else e])= | No strcmp dispatch — not yet implemented. |
| let    | =(let [x 1 y 2] (+ x y))=         | No strcmp dispatch — not yet implemented. |
| let*   | =(let* [x 1 y (+ x 1)] (+ x y))=  | No strcmp dispatch — not yet implemented. |
| letrec | =(letrec [f (lambda ...)] (f 0))= | No strcmp dispatch — not yet implemented. |
| lambda | =(lambda (x) body)=               | No strcmp dispatch — not yet implemented. |

These are =register_builtins=-only symbols
([[file:../codegen.c::12960][codegen.c:12960]]). See
[[file:../context/language.org::codegen-gaps][context/language.org — Known Codegen Gaps]]
for details.

Workaround: use Wisp syntax (the reader processes these forms before
codegen sees them) or use the forms that have strcmp handlers (e.g.
=if= instead of =cond=, =with= instead of =let=).

* 5. Null-Coalescing (??)

=(?? expr default)= ([[file:../codegen.c::5763]]) returns =expr= if not =nil=,
=default= otherwise. Scalar types skip the nil check.

#+BEGIN_SRC monad
(?? (lookup key dict) "default")
(?? (safe-div a b) 0)
#+END_SRC

#+BEGIN_SRC wisp
?? (lookup key dict) "default"
?? (safe-div a b) 0
#+END_SRC

See [[file:conditionals.org][Conditionals — ??]] for full documentation
and [[file:maybe.org][Maybe]] for option types.

* 6. Combining Control Forms

Control forms compose naturally thanks to expression orientation:

#+BEGIN_SRC monad
(define (classify n)
  (if (< n 0) "negative"
    (if (= n 0) "zero"
      (for [i 1 n]
        (show i))
      "positive")))

(while (>= n 0)
  (when (= n 0) (show "hit zero"))
  (set! n (- n 1)))
#+END_SRC

#+BEGIN_SRC wisp
define classify n
  if (< n 0) "negative"
    if (= n 0) "zero"
      for [i 1 n]
        show i
      "positive"

while (>= n 0)
  when (= n 0)
    show "hit zero"
  set! n (- n 1)
#+END_SRC

* 7. Runnable Example

#+BEGIN_SRC monad
;; run with: monadc
(define [x :: Int] -3)
(define result
  (if (> x 0)
    (begin (show "positive") x)
    (begin (show "non-positive") (- x))))
(show "result: _" result)
#+END_SRC

* 8. Gotchas

- **No =cond= dispatch**: =cond= has no strcmp handler in paren-mode.
  Use nested =if= or Wisp syntax (the reader desugars =cond= to =if=).
- **=let=, =lambda= also missing**: These 5 forms (=cond=, =let=,
  =let*=, =letrec=, =lambda=) are =register_builtins=-only. Paren-mode
  treats them as variable references.
- **Workaround**: All 8 forms with strcmp handlers work in paren-mode.
  For the missing 5, use Wisp syntax (reader processes before codegen).
- **=unless= return**: =unless= return convention is still evolving.
  Prefer =when= or =if= when the return value matters.
