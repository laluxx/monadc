Monad uses S-expressions and code-as-data — Lisp programmers will feel
at home, but the type system and compilation model change how you write code.
For the design philosophy behind dual syntax, see
[[file:../context/philosophy.org][context/philosophy.org]].
For the macro system architecture (hygiene, fixpoint, variadic splicing),
see [[file:../context/macro.org][context/macro.org]].
For the arena allocator replacing GC, see [[file:../context/arena.org][context/arena.org]].
For the monomorphization engine, see [[file:../context/codegen.org][context/codegen.org]].

Known gaps vs Common Lisp/Scheme are documented at
[[file:../context/language.org::287][context/language.org — Known codegen gaps]].
Clone of S-expression familiarity: =car=, =cdr=, =cons=, =quote=,
=quasiquote=, =define=, =lambda=, =if=, =cond=, =and=, =or=, =not=,
=let=, =let*=, =letrec= all work with familiar semantics where
codegen dispatch exists.

1. Compilation: LLVM Native Code, Not Interpreted

| Lisp / Scheme | Monad |
+------------------------+----------------------------------+
| Interpreted or JIT-compiled | Ahead-of-time native via LLVM               |
| =compile-file= = load         | =monad file.mon= — single step                |
| Incremental development     | Compiler pipeline (parse → infer → codegen) |

Monad compiles to native machine code through LLVM — not interpreted, not
JIT'd. The compilation pipeline ([[file:../context/language.org]]):
1. Reader (S-expression or Wisp → AST)
2. Macro expansion (fixpoint, up to 1024 rounds)
3. HM type inference + monomorphization
4. ITT dependent type checking (shadow pass)
5. LLVM codegen → native executable

See [[file:build.org][Build]] for compiler usage, and [[file:../context/build.org][context/build.org]]
for the build system architecture.

2. Lisp is the AST

S-expressions are not just syntax — they ARE the AST. What you write in
parens IS the parse tree the compiler walks:

#+BEGIN_SRC monad
(if (> x 0) (print "pos") (print "neg"))
#+END_SRC

#+BEGIN_SRC wisp
if (> x 0) (print "pos") (print "neg")
#+END_SRC

The reader directly produces AST nodes ([[file:../reader.h::40]]). No separate
parse tree, no CST, no intermediate representation. This means you can
always drop into raw S-expressions. Wisp (indentation syntax) is a
pre-expansion pass ([[file:../wisp.c]]) that generates the same AST.

You never lose the ability to write raw Lisp. Mix wisp and s-exprs
freely in the same file — both syntaxes are first-class
([[file:../context/philosophy.org::92][context/philosophy.org — syntax is instrument]]):

#+BEGIN_SRC monad
;; Wisp
if (> x 0)
  (print "pos")     ;; raw s-expr where clarity matters
  print "neg"       ;; back to wisp
#+END_SRC

#+BEGIN_SRC wisp
;; Wisp
if (> x 0)
  (print "pos")     ;; raw s-expr where clarity matters
  print "neg"       ;; back to wisp
#+END_SRC

See [[file:syntax.org][Syntax]] for the reader/parser grammar and [[file:wisp.org][Wisp Syntax]]
for indentation-based syntax.

3. Type System: HM Inference + Dependent Types

| Lisp/Scheme              | Monad                                 |
+--------------------------+---------------------------------------+
| Dynamic typing               | Hindley-Milner static inference        |
| (or gradual in Typed Racket) | + dependent types (ITT via NbE)        |
| Type annotations optional    | Type annotations optional (HM infers)  |
| No compile-time checking     | Compile-time static checking           |
| Runtime type dispatch        | Monomorphization — no runtime dispatch |

Monad uses a dual type system ([[file:../context/language.org::501]]):
- **Hindley-Milner** — fast type inference for ground types, unification,
  constraint generation, generalization ([[file:../context/language.org::489]])
- **Intuitionistic Type Theory** — dependent types with Pi/Sigma types,
  Normalization by Evaluation kernel, bidirectional checking
  ([[file:../context/language.org::686]])

The HM path handles the common case at O(n) speed; the ITT path validates
dependent annotations as a shadow pass ([[file:../context/language.org::1398]]).

Refinement types provide subset predicates on ground types
([[file:../context/language.org::597]], [[file:refinements.org]]):

#+BEGIN_SRC monad
(type Pos { x ∈ Int | (> x 0) })
(type Even { x ∈ Int | (= (mod x 2) 0) })
#+END_SRC

#+BEGIN_SRC wisp
type Pos { x ∈ Int | (> x 0) }
type Even { x ∈ Int | (= (mod x 2) 0) }
#+END_SRC

See [[file:types.org][Types]], [[file:type-inference.org][Type Inference]], and
[[file:dependent-types.org][Dependent Types]] for the full type system reference.

4. Monomorphization, Not Objects

In Lisp, every parameter is a boxed object (pointer + type tag). In
Monad:

- If you annotate a parameter with a type, it uses that type directly.
- If you DON'T annotate, the type is inferred and the function is
  **monomorphized** for each concrete call site.

#+BEGIN_SRC monad
(define (id x) x)              ;; polymorphic — monomorphized per use
(define [id-int :: Int -> Int] ;; monomorphic — Int directly
  x -> x)
#+END_SRC

#+BEGIN_SRC wisp
define id x
  x              ;; polymorphic — monomorphized per use
define [id-int :: Int -> Int] ;; monomorphic — Int directly
  x -> x
#+END_SRC

Monomorphization means =(id 42)= and =(id "hi")= generate separate
specialized functions (=id_Int=, =id_String=) with zero boxing
overhead. No type tags, no dispatch at runtime
([[file:../context/codegen.org]]).

The =mono_cache= ([[file:../codegen.c::109]]) caches specialized versions by
function name + concrete type args. When codegen encounters a
polymorphic call, it looks up the cache, substitutes concrete types via
=mono_apply_subst= ([[file:../codegen.c::189]]), generates a new LLVM function if not
cached, and emits a direct call.

| Lisp              | Monad (monomorphized)         |
|-------------------+-------------------------------|
| =(defun id (x) x)=  | =(define (id x) x)=             |
| =(id 42)= — boxed   | =(id 42)= — Int, no boxing      |
| =(id "hi")= — boxed | =(id "hi")= — String, no boxing |
| Dynamic dispatch  | Static specialization         |

See [[file:../context/philosophy.org::162][context/philosophy.org — monomorphization]] for the design rationale.
See [[file:performance.org][Performance]] for the performance implications.

5. Arena Allocation, Not GC

| Lisp / Scheme          | Monad                            |
+------------------------+----------------------------------+
| Garbage collector      | Arena allocation (bump-pointer)   |
| =malloc/=GC allocation | No manual free — arena reset      |
| Boxed cons cells       | Cons cells in arena (boxed) but no GC pauses |

Monad does not have a garbage collector. Values are allocated in
compiler-managed arena regions — bump-pointer allocation, deterministic
lifetime, no GC pauses ([[file:../context/philosophy.org::99]],
[[file:../context/arena.org][context/arena.org]]).

See [[file:arena.org][Arena Allocation]] for the arena API, and [[file:heap.org][Heap]]
for heap-allocated arrays and structures.

6. Macros: Functions Returning Syntax

| Common Lisp             | Monad ([[file:../context/macro.org]])                     |
+-------------------------+---------------------------------------------------+
| =(defmacro swap! (a b)  | =(define (swap! a b -> Syntax)                     |
|   `(let ((tmp ,a))     |   #(let [tmp a] (set! a b) (set! b tmp)))=        |
|      (setf ,a ,b)      |                                                   |
|      (setf ,b tmp)))=  |                                                   |

Monad macros are =define= forms whose return type is =Syntax=
([[file:../context/macro.org::56]]). Key differences from =defmacro=:

- Macros are **compile-time functions**: they are ordinary =define= forms
  whose lambda declares =-> Syntax= as the return type
  ([[file:../context/macro.org::75]])
- **=#(...)= quasi-output syntax** delimits the template output region:
  code outside =#()= runs at compile time (when implemented), code inside
  =#()= is the generated output ([[file:../context/macro.org::88]])
- **Hygiene**: names introduced inside the macro body (not pattern variables)
  are renamed to =macroname__varname__N= to prevent capture
  ([[file:../context/macro.org::123]])
- **Fixpoint expansion** (up to 1024 rounds): macros that expand into other
  macro calls are fully resolved ([[file:../context/macro.org::187]])
- **Variadic splicing**: rest parameters splice captured nodes flat in
  list position ([[file:../context/macro.org::156]])

Example macro:

#+BEGIN_SRC monad
;; Swap macro: returns Syntax
(define (swap! a b -> Syntax)
  #(let [tmp a] (set! a b) (set! b tmp)))

;; Usage
(define x 10)
(define y 20)
(swap! x y)  ;; expands to (let [tmp x] (set! x y) (set! y tmp))
#+END_SRC

#+BEGIN_SRC wisp
# Swap macro: returns Syntax
define (swap! a b -> Syntax)
  #(let [tmp a] (set! a b) (set! b tmp))

# Usage
define x 10
define y 20
swap! x y  # expands to (let [tmp x] (set! x y) (set! y tmp))
#+END_SRC

See [[file:macros.org][Macros]] and [[file:macro-patterns.org][Macro Patterns]] for the full macro reference.
See [[file:../context/macro.org::56][context/macro.org — Syntax convention]] for the Syntax type definition.

7. Wisp Syntax as Alternative

Monad offers two syntaxes ([[file:../context/philosophy.org::92]], [[file:wisp.org]]):

| S-expression       | Wisp (indentation)           |
+--------------------+------------------------------+
| =(define (f x) (if (> x 0) x (- x)))= | =define f x<br>  if > x 0<br>    x<br>    - x= |
| =(let [x 1] (+ x 1))=    | =let [x 1]<br>  + x 1=          |
| =(set! x val)=           | =set! x val=                  |

Both syntaxes share the same AST. Mix freely in the same file
([[file:wisp.org]]). The wisp-to-AST desugaring is implemented in
[[file:../wisp.c]] ([[file:../context/wisp.org][context/wisp.org]]).

See [[file:advanced-wisp.org][Advanced Wisp]] for indentation syntax patterns and edge cases.

8. Key Differences

**8.1 Same (identical syntax)**

- =car=, =cdr=, =cons=
- =quote=, =quasiquote=
- =define=, =lambda=
- =if=, =cond=, =and=, =or=, =not=
- =let=, =let*=, =letrec=
- Macros (code-as-data, =define ... -> Syntax=)

**8.2 Same, different syntax**

| Common Lisp        | Monad                      |
|--------------------+----------------------------|
| =(defun f (x) body)= | =(define (f x) body)=        |
| =(let ((x 1)) body)= | =(let [x 1] body)=           |
| =(setq x val)=       | =(set! x val)=               |
| ='()=                | =nil= or ='()=                 |
| =(format t "~a" x)=  | =(show x)=                   |
| =(defstruct ...)=    | =(layout ...)=               |
| =(defmacro ...)=     | =(define ... -> Syntax ...)= |

**8.3 New (no Lisp equivalent)**

- Static types with HM inference + dependent types
- Algebraic data types (=(data Maybe a = Nothing | Just a)=)
- Refinement types (=(type Pos { x ∈ Int | (> x 0) })=)
- Layout types (C-compatible structs, [[file:layouts.org]])
- Inline assembly (=(asm ...)=, [[file:asm.org]])
- C FFI via =(include "header.h")= ([[file:ffi.org]])
- Wisp indentation syntax ([[file:wisp.org]])
- Type classes (=(class ...)= / =(instance ...)=)
- Monomorphization of polymorphic functions
- Arena-based memory management

**8.4 Not yet implemented**

| Feature                  | Status ([[file:../context/language.org::287]])                     |
+--------------------------+-----------------------------------------------------+
| =map= / =mapcar=           | Type sig in infer.c, no codegen dispatch (see gap: closures) |
| =|> pipe                  | Reader parses, codegen fails                        |
| =fn= shorthand            | No codegen dispatch                                 |
| =cond=, =when=, =do=      | No strcmp handler — treated as ordinary variable    |
| Tail-call optimization    | Planned                                             |
| Closure allocation        | Partially implemented (closure ABI exists, =fn=/=lambda= strcmp dispatch missing) |
| =let=, =let*=, =letrec=   | =register_builtins=-only, no strcmp dispatch        |
| Wisp multi-line =define= with =->= arrow | Does not create correct lambda bindings (use s-expr form) |

Workarounds for common gaps:

#+BEGIN_SRC monad
;; Instead of map: use recursion
(define (add1 xs -> [Int])
  ((match xs)
    []     -> []
    [x|xs] -> (++ (+ x 1) (add1 xs))))

;; Instead of cond: use nested if
(define (classify x -> String)
  (if (> x 0) "positive"
    (if (< x 0) "negative"
      "zero")))
#+END_SRC

#+BEGIN_SRC wisp
# Instead of map: use recursion
define (add1 xs -> [Int])
  (match xs)
    []     -> []
    [x|xs] -> (++ (+ x 1) (add1 xs))

# Instead of cond: use nested if
define (classify x -> String)
  if (> x 0) "positive"
    if (< x 0) "negative"
      "zero"
#+END_SRC

See [[file:recursion.org][Recursion]] and [[file:patterns.org][Pattern Matching]] for alternative patterns.

Implementation Notes

Key internals behind the Lisp-to-Monad mapping:

- **Reader as AST producer**: S-expressions are parsed directly into AST
  nodes (=AST_CALL=, =AST_DEFINE=, etc.) by =reader.c= — no separate CST
  or parse tree ([[file:../context/reader.org][context/reader.org — AST production]]).
- **Wisp desugaring**: the wisp expander (=wisp.c=) converts
  indentation-based syntax to S-expressions in a pre-pass before the
  reader; both paths produce identical ASTs
  ([[file:../context/wisp.org][context/wisp.org — architecture]]).
- **Macro expansion**: =macro_expand_all= walks the AST outermost-first,
  expanding =Syntax=-returning =define= forms in a fixpoint (up to 1024
  rounds) before type inference
  ([[file:../context/macro.org][context/macro.org — fixpoint expansion]]).
- **Monomorphization**: =mono_cache= (=codegen.c=) specialises polymorphic
  functions per concrete call site, eliminating runtime dispatch
  ([[file:../context/codegen.org][context/codegen.org — monomorphization]]).
- **Arena allocation**: =arena_alloc= (=arena.c=) provides bump-pointer
  allocation — no GC, no =malloc= / =free=
  ([[file:../context/arena.org][context/arena.org — bump-pointer allocator]]).

Key Points

- S-expressions **are** the AST — what you write in parens is what the
  compiler walks. No separate parse tree.
- Wisp and S-expression syntax mix freely; both produce identical ASTs.
- Macros are =define= forms returning =Syntax= — no separate =defmacro=
  keyword, no quasiquote needed (though supported).
- Static typing via HM inference + ITT dependent types — a major shift
  from dynamic Lisp. Type annotations optional.
- Monomorphization replaces boxing: generic functions specialise per
  concrete call site.
- Arena allocation replaces GC: bump-pointer, deterministic, no pauses.
- =car=, =cdr=, =cons=, =quote=, =define=, =lambda=, =if=, =cond=,
  =and=, =or=, =not=, =let=, =let*=, =letrec= all work (where codegen
  dispatch exists).
