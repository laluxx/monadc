* 1. Quick Reference

| Form                                      | Description                      |
|-------------------------------------------+----------------------------------|
| =(type Name Base)=                        | Transparent type alias           |
| =(type T { x ∈ Int | pred })=             | Refinement with predicate        |
| =T?=                                       | Generated predicate function     |
| =(type T { x ∈ T | pred } :doc "..." )=   | Refinement with metadata         |

Refinement types annotate a base type with a predicate, forming the
subset of values that satisfy it. They are a lightweight static
contract system — predicates are evaluated at compile time when the
value is a literal, and deferred to runtime otherwise
([[file:../context/types.org::1321][context/types.org — refinement internals]]).

Think of a refinement type as "this Int, but only the ones that are
positive" — a compile-time checkable subset of the base type's values.

Test coverage: [[file:../tests/reader-refinements.tsv][tests/reader-refinements.tsv]] (106 parser rows).
Experiments: [[file:../how_to/RefinementType.mon][how_to/RefinementType.mon]]. The =refinement_check_literal=
static evaluator is documented at [[file:../context/types.org::1367][context/types.org]].

For an overview of all advanced type features, see
[[file:advanced-types.org][Advanced Types]].

* 2. Simple Type Alias

A =type= without braces is a transparent alias — no predicate, no
predicate function generated:

#+BEGIN_SRC monad
(type Code List)
(type UserId Int)
(type UserName String)
(type Rows List)
(type Dict Map)
(type Bytes Arr)
(type Count Int "integer count")
(type Slug String :doc "url slug")
(type Email String :doc "email text" :alias Contact)
(type Bytes Arr :alias Buffer)
#+END_SRC

#+BEGIN_SRC wisp
type Code List
type UserId Int
type UserName String
type Rows List
type Dict Map
type Bytes Arr
type Count Int "integer count"
type Slug String :doc "url slug"
type Email String :doc "email text" :alias Contact
type Bytes Arr :alias Buffer
#+END_SRC

Types MUST start with a capital letter. The alias inherits everything
from the base type, including docstring. Aliases are resolved
transitively through the =g_aliases= linked list up to 32 steps
([[file:../context/types.org::1547]]).

** Key Points

- Simple aliases are zero-cost — they disappear during type resolution
- Aliases are transitive: =type A B; type C A= means C resolves to B
- Docstrings and metadata can be attached with =:doc= and =:alias= keys

* 3. Refinement With Predicate

Set-builder notation with ∈ (U+2208, element of):

#+BEGIN_SRC monad
type Positive    { x ∈ Int | x > 0  }
type NonNegative { x ∈ Int | x >= 0 }
type Negative    { x ∈ Int | x < 0  }
type NonZero     { x ∈ Int | x != 0 }
type Even        { x ∈ Int | x % 2 = 0 }
type Odd         { x ∈ Int | x % 2 = 1 }
#+END_SRC

#+BEGIN_SRC wisp
type Positive    { x ∈ Int | x > 0  }
type NonNegative { x ∈ Int | x >= 0 }
type Negative    { x ∈ Int | x < 0  }
type NonZero     { x ∈ Int | x != 0 }
type Even        { x ∈ Int | x % 2 = 0 }
type Odd         { x ∈ Int | x % 2 = 1 }
#+END_SRC

Each refinement generates a predicate function: =Positive?=,
=NonNegative?=, =Even?=, etc. For the numeric operations used in
predicates, see [[file:arithmetic.org][Arithmetic]].

** 3.1 Floats

#+BEGIN_SRC monad
(type PositiveFloat { x ∈ Float | (> x 0.0) })
(type UnitFloat { x ∈ Float | (and (>= x 0.0) (<= x 1.0)) })
(type NotNaN { x ∈ Float | (= x x) })
#+END_SRC

#+BEGIN_SRC wisp
type PositiveFloat { x ∈ Float | > x 0.0 }
type UnitFloat { x ∈ Float | and (>= x 0.0) (<= x 1.0) }
type NotNaN { x ∈ Float | = x x }
#+END_SRC

** 3.2 Strings

#+BEGIN_SRC monad
(type NonEmptyString { s ∈ String | (> (count s) 0) })
(type ShortString { s ∈ String | (< (count s) 10) })
(type HttpUrl { s ∈ String | (starts-with? s "http") })
(type MarkdownFile { s ∈ String | (ends-with? s ".md") })
(type NoSpaces { s ∈ String | (not (contains? s " ")) })
(type CleanName { s ∈ String | (and (> (count s) 0) (not (contains? s "/")) (not (contains? s "\\\\"))) })
#+END_SRC

#+BEGIN_SRC wisp
type NonEmptyString { s ∈ String | > (count s) 0 }
type ShortString { s ∈ String | < (count s) 10 }
type HttpUrl { s ∈ String | starts-with? s "http" }
type MarkdownFile { s ∈ String | ends-with? s ".md" }
type NoSpaces { s ∈ String | not (contains? s " ") }
type CleanName { s ∈ String | and (> (count s) 0) (not (contains? s "/")) (not (contains? s "\\\\")) }
#+END_SRC

For string operations, see [[file:strings.org][Strings]].

** 3.3 Collections

#+BEGIN_SRC monad
(type NonEmptyList { xs ∈ List | (> (count xs) 0) })
(type SmallList { xs ∈ List | (< (count xs) 8) })
(type NonEmptySet { s ∈ Set | (> (count s) 0) })
(type SingletonSet { xs ∈ Set | (= (count xs) 1) })
(type ConfigMap { m ∈ Map | (and (> (count m) 0) (contains? m :config)) })
(type NonEmptyArr { xs ∈ Arr | (> (count xs) 0) })
#+END_SRC

#+BEGIN_SRC wisp
type NonEmptyList { xs ∈ List | > (count xs) 0 }
type SmallList { xs ∈ List | < (count xs) 8 }
type NonEmptySet { s ∈ Set | > (count s) 0 }
type SingletonSet { xs ∈ Set | = (count xs) 1 }
type ConfigMap { m ∈ Map | and (> (count m) 0) (contains? m :config) }
type NonEmptyArr { xs ∈ Arr | > (count xs) 0 }
#+END_SRC

For collection types, see [[file:collections.org][Collections]].

** 3.4 Ranges

#+BEGIN_SRC monad
(type ByteRange { n ∈ Int | (and (>= n 0) (< n 256)) })
(type Port { n ∈ Int | (and (> n 0) (< n 65536)) })
(type Year { y ∈ Int | (and (>= y 1970) (< y 3000)) })
(type SafeIndex { n ∈ Int | (and (>= n 0) (< n 1024) (!= n 666)) })
#+END_SRC

#+BEGIN_SRC wisp
type ByteRange { n ∈ Int | and (>= n 0) (< n 256) }
type Port { n ∈ Int | and (> n 0) (< n 65536) }
type Year { y ∈ Int | and (>= y 1970) (< y 3000) }
type SafeIndex { n ∈ Int | and (>= n 0) (< n 1024) (!= n 666) }
#+END_SRC

** Key Points

- The =∈= symbol (U+2208) is read as "element of" — the variable name
  before it is the bound variable used in the predicate
- Any expression that returns Bool can be used in the predicate
- Predicates can reference other refinement predicates via =T?=
  (e.g., =(Positive? n)= in a compound predicate)

* 4. Metadata

Metadata follows the type form, on the same line or indented with
=:doc= and =:alias= keywords:

#+BEGIN_SRC monad
(type PositiveAlias { x ∈ Int | (> x 0) } :alias Natural)
(type DocumentedPositive { x ∈ Int | (> x 0) } :doc "positive ints")
(type NameText { s ∈ String | (> (count s) 0) } :doc "names" :alias Identifier)
#+END_SRC

#+BEGIN_SRC wisp
type PositiveAlias { x ∈ Int | > x 0 } :alias Natural
type DocumentedPositive { x ∈ Int | > x 0 } :doc "positive ints"
type NameText { s ∈ String | > (count s) 0 } :doc "names" :alias Identifier
#+END_SRC

* 5. Predicate Functions

Each refinement type =T= generates =T?= at compile time:

#+BEGIN_SRC monad
Positive?  ;; Int -> Bool
Even?      ;; Int -> Bool
NonEmptyString?  ;; String -> Bool
Port?      ;; Int -> Bool  — generated from Port refinement
#+END_SRC

#+BEGIN_SRC wisp
Positive?  ;; Int -> Bool
Even?      ;; Int -> Bool
NonEmptyString?  ;; String -> Bool
Port?      ;; Int -> Bool  — generated from Port refinement
#+END_SRC

* 6. Compound Predicates

Arbitrary boolean expressions with =and=, =or=, =not=:

#+BEGIN_SRC monad
(type RangedScore { n ∈ Int | (and (>= n 0) (<= n 100) (!= n 13)) })
(type ZeroOrOne { n ∈ Int | (or (= n 0) (= n 1)) })
(type NotFortyTwo { n ∈ Int | (not (= n 42)) })
#+END_SRC

#+BEGIN_SRC wisp
type RangedScore { n ∈ Int | and (>= n 0) (<= n 100) (!= n 13) }
type ZeroOrOne { n ∈ Int | or (= n 0) (= n 1) }
type NotFortyTwo { n ∈ Int | not (= n 42) }
#+END_SRC

Predicates can use arithmetic, comparison, and type predicates:

#+BEGIN_SRC monad
(type SumBound { n ∈ Int | (< (+ n 1) 10) })
(type MultipleOfThree { n ∈ Int | (= (% n 3) 0) })
(type Numeric { x ∈ Number | (Number? x) })
(type Collectionish { x ∈ Coll | (or (List? x) (Map? x)) })
#+END_SRC

#+BEGIN_SRC wisp
type SumBound { n ∈ Int | < (+ n 1) 10 }
type MultipleOfThree { n ∈ Int | = (% n 3) 0 }
type Numeric { x ∈ Number | Number? x }
type Collectionish { x ∈ Coll | or (List? x) (Map? x) }
#+END_SRC

For boolean logic and short-circuit evaluation, see
[[file:booleans.org][Booleans]]. For arithmetic operators, see
[[file:arithmetic.org][Arithmetic]].

** Key Points

- Compound predicates compose freely: arithmetic, comparison, logic,
  and even other refinement predicates can be combined
- =and= and =or= short-circuit, which matters when predicates have
  side effects or expensive checks

** Common Pitfalls

- **Refinement checking only works on literals at compile time**:
  =(define [x :: Positive] (some-func n))= will NOT be checked at
  compile time — it's deferred to runtime (or silently passes if the
  predicate can't be evaluated).
- **Types MUST start with a capital letter**: =(type positive Int)=
  is parsed as a function call, not a type alias. Always capitalize
  type names.
- **Predicate functions are named =T?= automatically**: =(type
  Foo ...)= generates =Foo?=. If you define a second refinement with
  the same name, the predicate is overwritten silently.
- **Alias chains are limited to 32 steps**: Deep alias chains
  (=(type A B), (type C A)= etc.) will fail after 32 resolutions.
- **Intersection types are aspirational, not implemented**: You cannot
  write =(type PositiveEven = Positive ∩ Even)=. Write a compound
  predicate instead.

* 7. Compile-time Checking

When a refinement type is used with a literal value, the compiler
evaluates the predicate at compile time using =refinement_check_literal=
([[file:../context/types.org::1367]]):

1. Walks the alias chain (up to 32 steps) resolving through
   =TypeAlias= entries
2. Finds the =RefinementEntry= with a stored =predicate_ast=
3. Invokes =static_eval_pred= to recursively evaluate the AST

| Result | Meaning | Action |
|--------+---------+--------|
| 1      | Predicate passes | Compilation continues |
| 0      | Predicate fails | Compile error emitted |
| -1     | Cannot evaluate | Deferred to runtime check |

The static evaluator (=static_eval_pred=,
[[file:../context/types.org::1382]]) handles:

- Arithmetic: =+ -= *= %= (mod)
- Comparison: ===, =!=, =>=, >=, =<=, =<=
- Logical: =and= (short-circuit), =or= (short-circuit), =not=
- Recursive refinement lookup: calls =Name?= by stripping trailing
  =?= and looking up the registered refinement

Returns 1 (true = passes predicate), 0 (false = fails predicate), or
-1 (cannot evaluate at compile time → deferred to runtime).

#+BEGIN_SRC monad
;; Compile-time check — passes
(define x :: Positive 5)

;; Compile-time check — fails with error
(define y :: Positive -1)            ;; compile error

;; Runtime check — deferred (not a literal)
(define z :: Positive (some-func n))
#+END_SRC

#+BEGIN_SRC wisp
;; Compile-time check — passes
define x :: Positive 5

;; Compile-time check — fails with error
define y :: Positive -1              ;; compile error

;; Runtime check — deferred (not a literal)
define z :: Positive (some-func n)
#+END_SRC

For type annotation syntax, see [[file:type-inference.org::*7. Type Annotations][Type Annotations]].

** Key Points

- Literal values are checked at compile time — a bug is caught before
  the program runs
- Non-literal values get runtime checks via the =T?= predicate function
- The static evaluator returns -1 for anything it cannot decide (e.g.,
  variables, function calls)

* 8. Internal Representation

Each refinement is stored as a =RefinementEntry=
([[file:../context/types.org::849]]) in the global =g_refinements=
linked list:

| Field           | Description                                  |
|-----------------+----------------------------------------------|
| =name=          | Refinement type name (e.g. "Even")           |
| =pred_name=     | Predicate function name (e.g. "even?")       |
| =base_type=     | Underlying base type (e.g. "Int")            |
| =predicate_ast= | Owned clone of predicate AST for static eval |
| =var=           | Bound variable name in predicate (e.g. "x")  |

Registered via =refinement_register= ([[file:../types.c::45]]). For the
full type representation, see [[file:../context/types.org][context/types.org]].

* 9. Intersection Types (Aspirational)

The "types are sets" philosophy implies that type intersection should
be expressible:

#+BEGIN_SRC monad
;; TODO: aspirational — not yet implemented
(type PositiveEven = Positive ∩ Even)
#+END_SRC

#+BEGIN_SRC wisp
;; TODO: aspirational — not yet implemented
type PositiveEven = Positive ∩ Even
#+END_SRC

Currently, combined constraints must be written as a single predicate:

#+BEGIN_SRC monad
(type PositiveEven { n ∈ Int | (and (Positive? n) (Even? n)) })
#+END_SRC

#+BEGIN_SRC wisp
(type PositiveEven { n ∈ Int | and (Positive? n) (Even? n) })
#+END_SRC

For union types, see [[file:data.org::*1. Data Declaration Syntax][Sum Types]]. For the
relationship between refinement and dependent types, see
[[file:dependent-types.org][Dependent Types]].

** Complete example:

#+BEGIN_SRC monad
;; Save as refinements-demo.mon && monadc refinements-demo.mon && ./refinements-demo
(module Main)

;; Refinement types with compile-time checking
(type Positive { n :: Int | (> n 0) })
(type Even { n :: Int | (= (% n 2) 0) })
(type ShortString { s :: String | (< (count s) 10) })

;; Compile-time check — passes
(define [x :: Positive] 5)
(define [y :: Even] 42)
(define [name :: ShortString] "hi")

;; These would fail at compile time:
;; (define [bad :: Positive] -1)

;; Predicate functions for runtime checking
(show (Positive? 5))        ;; True
(show (Positive? -1))       ;; False
(show (Even? 42))           ;; True
(show (ShortString? "hello"))  ;; True

;; Using refinement in function signature
(define (safe-divide [n :: Int] [d :: Positive] -> Int)
  (/ n d))

(show (safe-divide 10 3))   ;; 3
#+END_SRC

* Notes

- All examples above are from [[file:../tests/reader-refinements.tsv][reader-refinements.tsv]] (parser tests)
- Predicate functions exist at the syntax level; their compilation
  depends on the compiler phase
- Types MUST start with a capital letter
- The =∈= symbol is U+2208, ∈
- Constructors in =types.h=: =type_int()=, =type_float()=, etc.
  ([[file:../context/types.org::975]])

  enabling composable constraints
