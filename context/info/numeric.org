MonadC provides a comprehensive numeric type hierarchy spanning arbitrary-precision concepts down to fixed-width machine types for C interop. The hierarchy is organized from most-general to most-specific: =Number= (any numeric) → =Int= / =Float= / =Ratio= (primary types) → fixed-width subtypes.

** Complete example:

#+BEGIN_SRC monad
;; Save as numeric-demo.mon && monadc numeric-demo.mon && ./numeric-demo
(module Main)

;; All numeric literal forms
(define i 42)              ;; Int
(define f 3.14)            ;; Float
(define r 1/3)             ;; Ratio
(define h 0xFF)            ;; Hex (255)
(define b 0b1010)          ;; Binary (10)
(define o 0o777)           ;; Octal (511)
(define p 50%              ;; Percentage (= 1/2)

;; Type predicates
(show (Int? 42))           ;; True
(show (Float? 3.14))       ;; True
(show (Number? 42))        ;; True
(show (Number? "hi"))      ;; False

;; Type conversions
(show (Int 3.14))          ;; 3
(show (Float 42))          ;; 42.0
(show (Hex 255))           ;; "0xFF"
(show (Char 65))           ;; 'A'
#+END_SRC

** Common Pitfalls

- **=Int= defaults to 64-bit, not arbitrary precision**: Integer
  overflow wraps silently (two's complement on x86-64). For safe
  arithmetic, validate ranges with [[file:refinements.org][Refinement Types]].
- **Fixed-width types do NOT auto-promote**: =(+ (I8 3) (I8 4))= is a
  type error. Use explicit [[file:casting.org][Casting]].
- **=Hex=, =Bin=, =Oct= are subtypes of Int at the type level**: They
  are NOT separate types at runtime. =(= 0xFF 255)= is True.
- **Ratio literals are parsed as single tokens, NOT division**: =1/3=
  is an =AST_RATIO= token, not sugar for =(/ 1 3)=.
- **Percentage syntax is reader-only**: =50%= desugars to =1/2= at
  parse time. There is no runtime percentage operation.
- **=Int?= checks for Int specifically, NOT for any numeric type**:
  =(Int? 3.14)= returns False. Use =Number?= to test any numeric type.

* 1. Primary Numeric Types

| Type | Description | Example | Precision |
|------+-------------+---------+-----------|
| =Int= | Architecture-width signed | =42=, =-7= | 64-bit (x86-64) |
| =Float= | IEEE 754 double | =3.14=, =1e10= | 64-bit |
| =Ratio= | Exact rational | =1/3=, =50%= | Arbitrary (num/den) |

See [[file:rationals.org][Rationals]] for the =Ratio= type.
See [[file:arithmetic.org][Arithmetic]] for operations on these types.
See [[file:type-classes.org][Type Classes]] for the =Num=, =Integral=, and =Floating= class hierarchy.

** Key Points

- =Int= is 64-bit on all supported platforms
- =Float= is IEEE 754 double-precision (binary64)
- =Ratio= is exact — no floating-point rounding, automatically reduced

* 2. Literal Notations

All numeric literals are read by the tokeniser ([[file:../reader.c][reader.c]]) and assigned their type at parse time.

| Notation | Description | Example |
|----------+-------------+---------|
| =42=, =-7= | Decimal Int | =42= |
| =3.14=, =1e10= | Float | =3.14= |
| =1/3=, =50%= | Ratio | =1/3= |
| =0xFF= | Hex | =0xFF= |
| =0b1010= | Binary | =0b1010= |
| =0o777= | Octal | =0o777= |

(42                 ;; Int)
(-7                 ;; Int (negated))
(3.14               ;; Float)
(1e10               ;; Float (scientific notation))

** Implementation Notes

- Numeric literal tokenisation in reader
  ([[file:../reader.c]])
- Hex, binary, octal are stored as subtypes of Int at the type level
  ([[file:../context/types.org::107]])
- Ratio literals parsed as single AST_RATIO tokens (not division)
  ([[file:../reader.c::265]])

* 3. Fixed-Width Numeric Types

For C interop and bit-level control ([[file:types.org][Types]], [[file:ffi.org][C FFI]]):

| Type | Width | Description |
|------+-------+-------------|
| =I8= | 8 | Signed byte |
| =U8= | 8 | Unsigned byte |
| =I16= | 16 | Short |
| =U16= | 16 | Unsigned short |
| =I32= | 32 | Int |
| =U32= | 32 | Unsigned int |
| =I64= | 64 | Long |
| =U64= | 64 | Unsigned long |
| =I128= | 128 | Quad word |
| =U128= | 128 | Unsigned quad |
| =F32= | 32 | IEEE 754 single |
| =F80= | 80 | x86 extended precision |

Note: =Int= maps to the machine word (i64 on x86-64). Use fixed-width types for explicit layout control or FFI.

These types are detailed in [[file:../context/types.org][context/types.org]].

** Key Points

- =Int= is the default integer type; fixed-width types are for FFI and
  layout control
- =F80= provides x86 extended precision (80-bit) for numerical computing
- Fixed-width types do NOT participate in automatic promotion — use
  explicit [[file:casting.org][Casting]]

* 4. Type Conversions ([[file:../codegen.c::13144]])

Numeric types are first-class casting functions (see also [[file:casting.org][Casting]]):

#+BEGIN_SRC monad
(Int   40.3)          ;; 4)
(Float 20)            ;; 20.)
(Hex   65)            ;; 0x4)
(Oct   63)            ;; 0o7)
(Bin   42)            ;; 0b10101)
(Char  65)            ;; 'A)
(Arr   20)            ;; [20)
#+END_SRC

* 5. Arithmetic Operations ([[file:core.org][Core Library]])

All working on Int, Float, and mixed operands (see [[file:arithmetic.org][Arithmetic]]):

| Operation | Example | Variadic |
|-----------+---------+----------|
| =+= | =(+ 1 2 3)= | Yes (1..N) |
| =-= | =(- 10 3)= | Yes (1..N) |
| =*= | =(* 2 3 4)= | Yes (1..N) |
| =/= | =(/ 10 3)= | Yes (1..N) |
| =%= | =(% 10 3)= | No (binary) |

* 6. Bitwise Operations

| Operation | Example | Variadic |
|-----------+---------+----------|
| =&= | =(& 0xFF 0x0F)= | Yes |
| =bit-or=, =∨= | =(bit-or 1 2 4)= | Yes |
| =bit-xor=, =⊕= | =(bit-xor 5 3)= | Yes |
| =~= | =(~ 0xFF)= | Unary |
| =<<= | =(<< 1 4)= | Binary |
| =>>= | =(>> 16 2)= | Binary |
| =>>>= | =(>>> -1 1)= | Binary |

Bitwise operations only work on integer types (Int and fixed-width integers).

* 7. Comparison Operations

All comparisons return =Bool= (see [[file:booleans.org][Booleans]]):

| Operation | Example | Variadic |
|-----------+---------+----------|
| ==== | =(= 1 1 1)= | Yes (2..N) |
| =!= | =(!= 1 2)= | Yes |
| =<= | =(< 1 2 3)= | Yes |
| =<== | =(<= 1 2)= | Yes |
| =>= | =(> 3 2 1)= | Yes |
| =>== | =(>= 3 3)= | Yes |

See also [[file:equality.org][Equality]].
