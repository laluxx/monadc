Sets are hash sets stored as open-addressing hash tables at runtime.
O(1) average insert/lookup/delete. Elements are boxed (heap-allocated)
to support hashing and equality dispatch.

See [[file:collections.org][Collections]] for an overview of all collection
types and [[file:../context/codegen-ops.org][context/codegen-ops.org]] for
operation dispatch.

* 1. Literal Syntax

#+BEGIN_SRC monad
{1 2 3}                 ;; set of Ints
{"a" "b"}               ;; set of Strings
{}                      ;; empty set
(set 1 2 3)             ;; set function call
#+END_SRC

#+BEGIN_SRC wisp
{1 2 3}
{"a" "b"}
{}
set 1 2 3
#+END_SRC

The ={}= syntax is shared with refinement types — the parser peeks
ahead for =∈= after the first symbol: refinement if found, set literal
otherwise ([[file:../reader.c::5998]]).

See [[file:refinements.org][Refinement Types]] for the shared syntax and
[[file:../context/reader.org][context/reader.org]] for parsing details.

** 1.1 Known gap: untyped set construction

Literal sets like ={1 2 3}= require an explicit =:: Set= annotation
to avoid garbage at runtime. Without it, elements are unboxed and
corrupt the hash table
([[file:../context/language.org::codegen-gaps][context/language.org]]).
This is the same class of bug as untyped map literals — the codegen
path does not box elements without a type annotation.

Key Points:
- Set literal syntax: ={1 2 3}= for a set of Ints
- Empty set: ={}=
- Syntactic ambiguity with refinement types — parser disambiguates by peeking for =∈=
- Untyped set literals require =:: Set= annotation to avoid runtime corruption

* 2. Core Operations

| Function    | Example             | Result    |
|-------------+---------------------+-----------|
| conj        | =(conj {1 2} 3)=      | {1 2 3}   |
| disj        | =(disj {1 2 3} 2)=    | {1 3}     |
| conj!       | =(conj! s 3)=         | mutates s |
| disj!       | =(disj! s 2)=         | mutates s |
| contains?   | =(contains? {1 2} 1)= | True      |
| count       | =(count {1 2 3})=     | 3         |
| Set?        | =(Set? {1 2})=        | True      |
| collection? | =(collection? {1 2})= | True      |

See [[file:core.org][Core Library]] for builtin implementations.

** 2.1 conj / disj

#+BEGIN_SRC monad
(conj {1 2} 3)               ;; {1 2 3}
(disj {1 2 3} 2)             ;; {1 3}
#+END_SRC

#+BEGIN_SRC wisp
conj {1 2} 3
disj {1 2 3} 2
#+END_SRC

=conj= and =disj= copy the set before modifying (pure).
=conj!= and =disj!= mutate in place. Both share the scheme
=∀a. Set → a → Set=.

See [[file:mutation.org][Mutation]] for mutable vs immutable semantics.

** 2.2 contains?

#+BEGIN_SRC monad
(contains? {1 2 3} 2)        ;; True
(contains? {1 2 3} 99)       ;; False
#+END_SRC

#+BEGIN_SRC wisp
contains? {1 2 3} 2
contains? {1 2 3} 99
#+END_SRC

Key Points:
- =conj= and =disj= are pure (copy + modify)
- =conj!= and =disj!= are mutable (in-place)
- =contains?= returns boolean membership test

* 3. Set-as-Function

A set variable can be used as a function:

#+BEGIN_SRC monad
(define s {1 2 3})
(s 2)                       ;; 2 — element found
(s 99)                      ;; nil — not found
#+END_SRC

#+BEGIN_SRC wisp
define s {1 2 3}
s 2
s 99
#+END_SRC

When used as a function, a set returns the element if found, or =nil=
if absent. Inference skips =TYPE_ARROW= unification when the function
position has =TYPE_SET= ([[file:../codegen.c::9880]]).

See [[file:collections.org][Collections]] for the collection-as-function protocol.

Key Points:
- Sets are callable: =(s 2)= returns 2 if present, =nil= if absent
- Inference special-cases =TYPE_SET= to avoid =TYPE_ARROW= unification
- No default-value argument (unlike maps)

* 4. Mutable vs Immutable

| Operation | Semantics       | Use Case                 |
|-----------+-----------------+--------------------------|
| conj      | Copy + insert   | Pure, no aliasing        |
| disj      | Copy + remove   | Pure, no aliasing        |
| conj!     | In-place insert | Performance, owned refs  |
| disj!     | In-place remove | Performance, owned refs  |

The choice is purely about aliasing, not enforced by the compiler.
Use pure variants by default; switch to mutable variants when
profiling identifies a bottleneck.

Key Points:
- Pure operations are safe with aliasing; mutable are faster
- Compiler does not enforce aliasing discipline
- Prefer pure (=conj=, =disj=) by default

* 5. Iteration

Sets can be iterated with =for=:

#+BEGIN_SRC monad
(for [x {1 2 3}]
  (show x))                  ;; prints 1, 2, 3
#+END_SRC

#+BEGIN_SRC wisp
(for [x {1 2 3}]
  show x)
#+END_SRC

Iteration order is not guaranteed (hash table traversal).

See [[file:iteration.org][Iteration]] for for-loop semantics and
[[file:../context/language.org][context/language.org]] for set iteration dispatch.

Key Points:
- =for= iterates over set elements in unspecified order
- Use =conj= to build sets incrementally

* 6. Runnable Example

#+BEGIN_SRC monad
;; run with: monadc set-demo.mon

;; Always annotate set literals to avoid garbage
(define s :: (Set Int) {1 2 3 4 5})

(show (contains? s 3))            ;; prints True
(show (contains? s 99))           ;; prints False
(show (count s))                  ;; prints 5

(define s2 (conj s 6))
(show (count s2))                 ;; prints 6

(define s3 (disj s2 1))
(show (count s3))                 ;; prints 5

;; Set-as-function — returns element or nil
(show (s 3))                      ;; prints 3
(show (s 99))                     ;; prints nil

;; Iteration
(for [x s] (show x))              ;; prints elements
#+END_SRC

* 7. Gotchas

Common mistakes and surprising behaviours with sets:

- **Untyped set literal ={}= produces garbage** — always annotate =:: (Set T)=. Same root cause as maps: the codegen path does not box elements without a type annotation ([[file:../context/language.org::codegen-gaps]]).

- **={\}= syntax is shared with refinement types** — the parser peeks for =∈= after the first symbol. If you get a parse error on a set literal, check that it isn't being interpreted as a refinement.

- **Iteration order is unspecified** — hash table traversal means =for= and =conj=/=disj= order is not preserved.

- **Set-as-function has no default-value argument** — unlike maps, =(s missing)= returns =nil= with no way to supply a fallback.

- **Mutable ops (=conj!=, =disj!=) do not copy** — aliasing issues apply; prefer pure =conj=/=disj= by default.
