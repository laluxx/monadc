The Monad standard library consists of core modules (=Math=, =Coll=, =Test=, =Ascii=) imported via =(import ...)= plus built-in functions always available. See [[file:core.org][Core Library]] for builtins.

1. Quick Reference

| Module     | Key Exports                                    |
|------------+------------------------------------------------|
| Core       | Built-in functions (always available)          |
| Math       | =gcd=, =lcm=, =factorial=, =sin=, =sqrt=, =pi= |
| Coll       | =map=, =foldl=, =foldr=, =take=, =drop=        |
| Test       | =assert-eq=, =deftest=, =run-tests=             |
| Ascii      | =upper=, =lower=                                 |

Test coverage: =make test-core= runs per-module tests via
=python3 tests/run_core.py=. Individual modules are tested by=rt_core_*.mon=
fixtures. Main.mon coverage is documented under [[file:../context/main-mon.org][context/main-mon.org]].
The test architecture is documented at
[[file:../context/tests-arch.org][context/tests-arch.org]].

The Monad standard library is currently composed of two layers:

1. *Builtins* — functions dispatched directly in =codegen.c= (always
   available, no import needed). See [[file:core.org][Core Library]].

2. *Core modules* — source files under =core/= that contain reusable
   definitions compiled as needed.

See [[file:../context/core-libs.org][context/core-libs.org]] for the library architecture.

2. Core Modules

The =core/= directory contains per-module library sources:

| Module | File                              | Working Status                                                                                                                                                                                                                                                |
|--------+-----------------------------------+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Math   | =core/Math.mon= (400 lines)         | Partially working. Exports 30+ functions/constants including =gcd=, =lcm=, =factorial=, =fib=, =prime?=, =clamp=, =sin=, =sqrt=, =pow=, =floor=, =ceiling=, =round=, constants =pi=, =e=, =tau=, =phi=, infinite sequences =naturals=, =positives=, =evens=, =odds=. Known issue: variadic =gcd= fails. |
| Coll   | =core/prelude/Coll.mon= (100 lines) | Partially working. Exports =take=, =drop=, =takeWhile=, =dropWhile=, =map=, =all?=, =foldl=, =foldr=. Known issue: =takeWhile= inference fails during compilation.                                                                                                              |

**2.1 Usage examples**

#+BEGIN_SRC monad
(import Math)

(show (gcd 12 8))       ; 4
(show (even? 42))       ; True
(show (factorial 5))    ; 120
(show pi)               ; 3.14159...
(show (clamp 0 100 150)); 100
#+END_SRC

#+BEGIN_SRC wisp
import Math

show (gcd 12 8)       ; 4
show (even? 42)       ; True
show (factorial 5)    ; 120
show pi               ; 3.14159...
show (clamp 0 100 150); 100
#+END_SRC

#+BEGIN_SRC monad
(import Coll)

(show (take 3 '(1 2 3 4 5)))       ; (1 2 3)
(show (drop 2 '(1 2 3 4 5)))       ; (3 4 5)
(show (all? even? '(2 4 6)))       ; True
(show (foldl + 0 '(1 2 3 4)))      ; 10
#+END_SRC

#+BEGIN_SRC wisp
import Coll

show (take 3 '(1 2 3 4 5))       ; (1 2 3)
show (drop 2 '(1 2 3 4 5))       ; (3 4 5)
show (all? even? '(2 4 6))       ; True
show (foldl + 0 '(1 2 3 4))      ; 10
#+END_SRC

Note: Core prelude modules are not yet automatically imported at
startup. Explicit =import= statements are required. Known compilation
issues exist (context/language.org:417-424): =import Coll= currently
fails while compiling =takeWhile= in =Coll.mon=; variadic =gcd= in
=Math.mon= fails.

**2.2 Test module**

The =core/prelude/Test.mon= module provides =assert-eq=, =assert-true=,
=assert-false=, =deftest=, and =run-tests= for embedded testing:

#+BEGIN_SRC monad
(import Test)

(deftest arithmetic
  (assert-eq (+ 1 1) 2 "1+1=2")
  (assert-true (= 2 (+ 1 1)) "2=1+1"))
#+END_SRC

#+BEGIN_SRC wisp
import Test

deftest arithmetic
  assert-eq (+ 1 1) 2 "1+1=2"
  assert-true (= 2 (+ 1 1)) "2=1+1"
#+END_SRC

See [[file:testing.org][Testing]] for the full test framework reference.

**2.3 String module**

The =core/text/= directory contains =Ascii.mon= and =String.mon= (WIP):

#+END_SRC

#+BEGIN_SRC wisp

#+END_SRC

3. Main.mon

The file =Main.mon= (1244 lines) serves as both the compiler's primary
test surface and a de facto standard library reference. It demonstrates
real usage of every language feature:

- **Variables and types**: =define= with inferred and annotated types
  (=Int=, =Float=, =Char=, =String=, =Hex=, =Bin=, =Oct=).
- **Arithmetic**: mixed-type operations across all numeric types.
- **Functions**: typed function definitions with multi-param arrows,
  documentation strings, aliases.
- **Lists**: =list=, =make-list=, =append!=, =list-length=, =list-empty?=,
  =car=, =cons=, =++=.
- **Arrays**: fixed-size arrays (=Arr :: T :: N=), heap arrays (=~[]=),
  typed and inferred element types.
- **Layouts**: =layout Point= with construction and access, =where=
  clause methods.
- **Inline assembly**: naked (=:naked True=) and non-naked (=:naked
  False=) asm functions with SysV register usage.
- **Maps and sets**: =set=, =conj=, =disj=, =contains?=, map construction
  with =#{"key" val}= notation, =assoc=, =dissoc=, =keys=, =vals=,
  =merge=, =find=, =get= with default, map-as-function.
- **Control flow**: =for= loops with and without step, =if= expressions,
  =quote=, =show=.
- **Conditional compilation**: =#+LINUX=, =#+X86-64= feature flags.
- **Type aliases**: =type Code List=.
- **Ratios**: =1/3=, rational arithmetic.
- **Casting**: type-as-function casting across all builtin types.
- **Infix sugar**: =`into`= backtick infix notation.

4. Retired old-core staging area

The former =old-core/= directory held aspirational modules from a previous
compiler era. It was retired on 2026-07-16 after durable behavior gained active,
tested owners under =core/=:

| File               | Contents                             |
|--------------------+--------------------------------------|
| =old-core/List.mon=   | List functions (aspirational)        |
| =old-core/Set.mon=    | Set functions (wisp =tests= block parsing issue) |
| =old-core/text/=      | Ascii, String modules (old =::= syntax)          |
| =old-core/Math.mon=   | Math functions (superseded by =core/Math.mon=)   |
| =old-core/prelude/    | Coll, Test modules (old versions)                |
| =old-core/json.mon=   | JSON parser/emitter                  |
| =old-core/Random.mon= | Random number generation             |
| =old-core/Assert.mon= | Assertion utilities                  |

These historical paths describe provenance only. Current owners are =Math=,
=Data.Set=, =Data.List=, =Data.String=, =Data.Char=, =Coll=, =Function=, and
=Text.Parser=; see =context/core-libs.org= for the migration record.

5. Modules

The module system ([[file:modules.org]]) provides =(import ModuleName)= syntax.
Module resolution searches =core/= for library sources. See
[[file:../context/module.org][context/module.org]] for the module resolution algorithm and
[[file:../context/module-export.org][context/module-export.org]] for the export system.

Implementation Notes

- **Library architecture**: core modules are compiled on demand when
  =(import ...)= is encountered; the module registry caches compiled
  modules by name to avoid recompilation
  ([[file:../context/core-libs.org][context/core-libs.org — architecture]]).
- **Module resolution**: =module_name_to_path= converts dot-separated
  module names to filesystem paths (="Std.Math"= → ="Std/Math.mon"=);
  the 4-step resolution chain searches registry, local, =MONAD_CORE=,
  and installed paths ([[file:../context/module.org][context/module.org — resolution]]).
- **Main.mon**: serves as both test surface and de facto stdlib reference;
  its 1244 lines exercise every language feature and are verified by
  =make test-core= ([[file:../context/main-mon.org][context/main-mon.org]]).
- **Old-core**: aspirational modules from a previous compiler era; they
  use older syntax and are not compiled by the current compiler.

Key Points

- Two layers: **builtins** (always available, see [[file:core.org][Core Library]])
  and **core modules** (importable via =(import ...)=).
- Core modules live in =core/=: =Math.mon= (400 lines), =Coll.mon=
  (100 lines), =Test.mon=, =Ascii.mon=.
- =Main.mon= (1244 lines) is the primary test surface demonstrating every
  language feature.
- Explicit =import= required — no automatic prelude import yet.
- Known compilation issues: =takeWhile= in =Coll.mon= fails, variadic
  =gcd= in =Math.mon= fails.
- Retired =old-core/= designs survive only where active core tests establish a contract.
