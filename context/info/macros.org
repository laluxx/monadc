[[file:index.org][<- Back to Index]] · [[file:macro-patterns.org][Macro Patterns]] · [[file:syntax.org][Syntax]] · [[file:quote.org][Quote]] · [[file:define.org][Define]] · [[file:lambda.org][Lambda]] · [[file:wisp.org][Wisp Syntax]]

Monad supports compile-time metaprogramming via macros ([[file:../macro.c]]).
There is no separate =macro= keyword — macros are ordinary =define= forms
whose lambda declares =-> Syntax= as the return type
([[file:../context/macro.org::macro-syntax-convention][context/macro.org — Syntax convention]]).

Implementation: [[file:../macro.c][macro.c]] (expander, 765+ lines),
[[file:../context/macro.org][context/macro.org]] (architecture, hygiene, variadic splicing).
Test coverage: macro expansion is tested by codegen fixtures that use
macro-defined forms; the fixpoint cap (1024 rounds) at
[[file:../macro.c::32][macro.c:32]] is exercised by recursive macro tests.
The test architecture is documented at
[[file:../context/tests-arch.org][context/tests-arch.org]].

* 1. How Macros Work

1. The reader ([[file:../reader.c]]) parses source into S-expressions (AST)
   ([[file:../context/reader.org][context/reader.org]]).
2. The macro expander =macro_expand_all= ([[file:../macro.c::78]]) scans every
   top-level define whose lambda return type is ="Syntax"=, registers them as
   macros, and removes the define forms from the output (macros are compile-time
   only and do not emit LLVM IR).
3. The expander walks the remaining AST outermost-first, looking for calls to
   registered macros.
4. When found, it calls the macro function with the unevaluated argument nodes
   and substitutes the returned =Syntax= tree in place of the call.
5. The walk repeats to a fixpoint (capped at 1024 rounds;
   [[file:../macro.c::32]]), so macros that expand into other macro calls are
   fully resolved. If the cap is reached, expansion is aborted.

* 2. Defining Macros

Define a macro as a function returning =Syntax=:

#+BEGIN_SRC monad
;; A macro that discards its argument and returns 42
(define (my-macro x -> Syntax)
  42)
#+END_SRC

#+BEGIN_SRC wisp
;; A macro that discards its argument and returns 42
define my-macro x -> Syntax
  42
#+END_SRC

Variadic macros use =. name= for the rest argument:

#+BEGIN_SRC monad
(define when :: condition . body -> Syntax
  (if condition (begin . body)))
#+END_SRC

#+BEGIN_SRC wisp
define when :: condition . body -> Syntax
  if condition (begin . body)
#+END_SRC

The =Syntax= return type is what distinguishes macros from functions. The
type-checker uses =macro_is_registered= ([[file:../macro.c::86]]) to skip
macro define forms during codegen.

See [[file:define.org][Define — Function Definitions]] for general =define= syntax and
[[file:lambda.org][Lambda]] for the function return type annotation syntax.

* 3. #() Quasi-Output Syntax

If the macro body contains =#(...)= the expander strips the marker and only the
inner subtree becomes the macro output. If no =#(...)= appears, the entire body
is the output template ([[file:../context/macro.org::marco-quasi-output]]):

#+BEGIN_SRC monad
(define (assert condition message -> Syntax)
  (if debug-mode
    #(when (not condition) (error message))
    #(begin)))
#+END_SRC

#+BEGIN_SRC wisp
define assert condition message -> Syntax
  if debug-mode
    #(when (not condition) (error message))
    #(begin)
#+END_SRC

Currently, code outside =#()= is not evaluated at compile time — both branches
of the conditional survive into the output. Full compile-time evaluation of the
non-=#()= portion is future work ([[file:../context/macro.org::think.context.macro.quasi-output-eval]]).

* 4. How Syntax Works

=Syntax= is the type of unevaluated S-expressions. A macro receives its
arguments as =Syntax= values and must construct a valid S-expression to return.
Construction uses plain function calls — no quasiquotation needed:

#+BEGIN_SRC monad
(define (double-it x -> Syntax)
  (list '* 2 x))

(double-it 5)  ;; expands to (* 2 5)
#+END_SRC

#+BEGIN_SRC wisp
define double-it x -> Syntax
  list '* 2 x

double-it 5  ;; expands to (* 2 5)
#+END_SRC

=Syntax= is a first-class type in the surface type system but does not exist at
runtime — macro defines are consumed during expansion and no LLVM IR is emitted
for them.

For details on the =Syntax= type and its place in the type hierarchy, see
[[file:syntax.org][Syntax Reference]] and [[file:types.org][Types]].

* 5. Pattern Matching on Arguments

Use =match= to destructure the argument list
([[file:../context/pmatch.org][context/pmatch.org]] for the pattern-match compiler):

#+BEGIN_SRC monad
(define (my-and . args -> Syntax)
  []       -> True
  [x]      -> x
  [x|rest] -> (if x (my-and . rest) False))
#+END_SRC

#+BEGIN_SRC wisp
define my-and . args -> Syntax
  []       -> True
  [x]      -> x
  [x|rest] -> (if x (my-and . rest) False)
#+END_SRC

For more pattern-matching recipes see [[file:patterns.org][Pattern Matching]] and
[[file:macro-patterns.org][Macro Patterns]].

* 6. Expansion Phase

Macro expansion happens after reader parsing but before type inference
([[file:../context/infer.org][context/infer.org]]). Macros work on the raw AST, not on
typed representations. The expander is outermost-first: it expands the outer list
first, then recurses into the result. A macro that produces another macro call
is expanded in a subsequent fixpoint round
([[file:../context/macro.org::think.context.macro.expansion-order]]):

#+BEGIN_EXAMPLE
Source → Reader → Macro Expansion → Type Inference → Codegen
         ↑          |
         └──────────┘
       (raw AST)
#+END_EXAMPLE

After expansion completes, the resulting AST passes to type inference
([[file:../context/infer.org][context/infer.org]]) and then LLVM codegen
([[file:../context/codegen.org][context/codegen.org]]).

* 7. Hygiene

Macro-generated names are hygienically renamed to prevent unintentional capture
([[file:../context/macro.org::macro-hygiene]]). Names introduced inside a macro
body (lambda parameters, =let= bindings, inner =define=s) that are NOT pattern
variables are renamed to the pattern: =macroname__varname__N=, where N is a
global monotonic counter reset between compilation units by =macro_clear()=
([[file:../macro.c::94]]):

#+BEGIN_SRC monad
;; The 'tmp' binding below is hygienically renamed
(define (swap-hygienic a b -> Syntax)
  (let [tmp a]
    (set! a b)
    (set! b tmp)))
;; Expands with: tmp → swap-hygienic__tmp__0
#+END_SRC

#+BEGIN_SRC wisp
define swap-hygienic a b -> Syntax
  let [tmp a]
    set! a b
    set! b tmp
#+END_SRC

Pattern variables (the formal parameters of the macro lambda) and free
references to outer definitions pass through unchanged. The double-underscore
separator =__= is reserved by the compiler and cannot appear in user-written
identifiers.

For unhygienic macro patterns and intentional capture, see
[[file:macro-patterns.org][Macro Patterns — Hygiene]].

* 8. Variadic Splicing

A rest parameter (=. name=) captures zero or more trailing arguments. In the
template, if the rest name appears as a list element the captured nodes are
spliced (inserted flat into the enclosing list). In non-list position a
single-element rest wraps in =(begin ...)=
([[file:../context/macro.org::macro-variadic-splicing]]):

#+BEGIN_SRC monad
(define (my-and . args -> Syntax)
  [x|rest] -> (if x (my-and . rest) False))
;; rest spliced into the 'if' tail produces N-ary chaining
#+END_SRC

#+BEGIN_SRC wisp
define my-and . args -> Syntax
  [x|rest] -> (if x (my-and . rest) False)
;; rest spliced into the 'if' tail produces N-ary chaining
#+END_SRC

Splicing vs. wrapping:

#+BEGIN_SRC monad
(define (splice-demo . args -> Syntax)
  (list . args))    ;; splices: (list 1 2 3)
(define (wrap-demo . args -> Syntax)
  (list args))      ;; wraps: (list (1 2 3))
#+END_SRC

#+BEGIN_SRC wisp
define splice-demo . args -> Syntax
  list . args    ;; splices: (list 1 2 3)
define wrap-demo . args -> Syntax
  list args      ;; wraps: (list (1 2 3))
#+END_SRC

See [[file:macro-patterns.org][Macro Patterns — Variadic Macro Semantics]] for more variadic patterns.

* 9. Important Notes

- Macros run at compile time only.
- Macros can see other macros defined earlier in the same compilation unit.
- Recursive macros must avoid infinite expansion (fixpoint cap at 1024 rounds).
- =Syntax= return type distinguishes macros from functions; codegen skips
  macro defines entirely via =macro_is_registered=.
- =Syntax= does not exist at runtime — macro forms compile away entirely.
- =macro_clear()= frees all registered macros and resets the hygiene counter
  between compilation units.
- No ellipsis =...= patterns (Scheme-style) — variadic behaviour is via a
  single rest parameter ([[file:../context/macro.org::think.context.macro.pattern-variables]]).
- The =#()= quasi-output syntax marks template output regions.
- Macro expansion is outermost-first with a 1024-round fixpoint.
- The double-underscore =__= separator in hygiene renames is reserved.

Implementation Notes

The macro expander (=macro.c=, 765+ lines) integrates between the reader
and the type inference phase:

- **Registration**: =macro_is_registered= (=macro.c:86=) checks whether a
  define form has return type =Syntax=; if so, it's registered as a macro
  and removed from the codegen stream (no LLVM IR emitted)
  ([[file:../context/macro.org::macro-syntax-convention][context/macro.org — syntax convention]]).
- **Expansion order**: =macro_expand_all= (=macro.c:78=) walks the AST
  outermost-first, recursing into results. A fixpoint (capped at 1024
  rounds) ensures macros expanding to other macro calls are fully resolved
  ([[file:../context/macro.org::macro-fixpoint][context/macro.org — fixpoint]]).
- **Hygiene**: names introduced in macro bodies (not pattern variables)
  are renamed to =macroname__varname__N= with a global monotonic counter
  reset by =macro_clear()= between compilation units
  ([[file:../context/macro.org::macro-hygiene][context/macro.org — hygiene]]).
- **Variadic splicing**: rest parameters (=. name=) splice captured nodes
  flat in list position; in non-list position a single-element rest wraps
  in =(begin ...)= ([[file:../context/macro.org::macro-variadic-splicing][context/macro.org — variadic splicing]]).
- **Quasi-output =#()=**: the expander strips the =#()= wrapper and uses
  only the inner subtree as output; code outside is reserved for future
  compile-time evaluation
  ([[file:../context/macro.org::marco-quasi-output][context/macro.org — quasi-output]]).

Key Points

- Macros are =define= forms whose return type is =Syntax= — no separate
  =macro= keyword.
- Arguments are received **unevaluated** as =Syntax= values.
- Template construction uses ordinary function calls (no quasiquotation
  required, though supported via =⌜…⌟=).
- =#(...)= quasi-output syntax delimits the template output region.
- Hygienic renaming prevents capture: =macroname__varname__N=.
- Variadic macros use =. name= for rest args; splicing vs wrapping
  depends on list vs non-list position.
- Fixpoint expansion (1024-round cap) resolves nested macro calls.
- Macros are compile-time only — no LLVM IR emitted.
