=Path= is a first-class literal type for filesystem paths. Paths are valid
Unix path tokens that the reader recognizes as =AST_PATH= nodes
([[file:../reader.c]]). See [[file:syntax.org][Syntax]] for the reader's
atom classification rules.

* 1. Path Literals

#+BEGIN_SRC monad
~/xos/projects/c/monadc/context/     ; absolute with home
/tmp/monadc-fuzz                     ; absolute
./context                            ; relative
../monadc                            ; parent
#+END_SRC

Path literals are valid as values in both S-expression and Wisp syntax:

#+BEGIN_SRC monad
(define context-dir ~/xos/projects/c/monadc/context/)
(show context-dir)                   ; prints directory listing
#+END_SRC

#+BEGIN_SRC wisp
define context-dir ~/xos/projects/c/monadc/context/
show context-dir
#+END_SRC

#+BEGIN_SRC monad
;; Reading a source file
(define src ~/monad/main.mon)
(show src)

;; Working directory relative
(define build-dir ./build/)
#+END_SRC

#+BEGIN_SRC wisp
;; Reading a source file
define src ~/monad/main.mon
show src

;; Working directory relative
define build-dir ./build/
#+END_SRC

See [[file:show.org][Show]] for path display semantics and
[[file:../context/reader.org][context/reader.org]] for =AST_PATH= tokenization.

* 2. Path Type

=Path= is a primitive ground type (=TYPE_PATH=) in the type system
([[file:../types.h::8]]).

| Aspect         | Detail              |
|----------------+---------------------|
| Type kind      | =TYPE_KIND_GROUND=    |
| Type enum      | =TYPE_PATH=           |
| Runtime tag    | =RT_PATH=             |
| Representation | =char*= (path string) |

See [[file:types.org][Types → Primitive Types]] for the ground type
overview and [[file:../context/types.org][context/types.org]] for the
=TYPE_PATH= enum definition and runtime tagging.

* 3. Path Operations

Paths support =show= for display and comparison via ==== and =!=:

#+BEGIN_SRC monad
(= ./context ./context)             ; True
(!= ./context /tmp)                 ; True
#+END_SRC

#+BEGIN_SRC wisp
= ./context ./context
!= ./context /tmp
#+END_SRC

#+BEGIN_SRC monad
;; Path comparison in conditionals
(if (= ./config.json ./config.prod.json)
  (show "production config")
  (show "development config"))
#+END_SRC

#+BEGIN_SRC wisp
;; Path comparison in conditionals
if = ./config.json ./config.prod.json
  show "production config"
  show "development config"
#+END_SRC

See [[file:equality.org][Equality]] for the full equality protocol and
[[file:show.org][Show]] for the display protocol. Path operations at the
codegen level are documented in
[[file:../context/codegen.org][context/codegen.org]].

* 4. Path vs String

| Aspect         | Path             | String               |
|----------------+------------------+----------------------|
| Type           | =Path= (=TYPE_PATH=) | =String= (=TYPE_STRING=) |
| Literal syntax | =~/path=, =./path=   | ="text"=               |
| Quote needed   | No               | No                   |
| Use case       | Filesystem paths | General text         |

A path literal is not a string — it has its own type and cannot be used
where a =String= is expected without explicit conversion.

#+BEGIN_SRC monad
;; Path literal (type Path)
(define p ~/data/file.txt)

;; String literal (type String)
(define s "~/data/file.txt")

;; These are NOT the same type
(= p s)              ; type error: Path vs String
#+END_SRC

#+BEGIN_SRC wisp
;; Path literal (type Path)
define p ~/data/file.txt

;; String literal (type String)
define s "~/data/file.txt"

;; These are NOT the same type
= p s                # type error: Path vs String
#+END_SRC

See [[file:strings.org][Strings]] for the string type and
[[file:casting.org][Casting]] for type conversion between path and
string representations.

* 5. Paths in Context

Paths interact with several compiler subsystems:

- **C FFI**: paths used for C library include directories
  ([[file:ffi.org][C FFI]], [[file:../context/ffi.org][context/ffi.org]]).
- **Modules**: paths for module resolution and import
  ([[file:modules.org][Modules]],
  [[file:../context/module.org][context/module.org]]).
- **CLI**: command-line path arguments
  ([[file:cli.org][CLI Reference]],
  [[file:../context/cli.org][context/cli.org]]).
- **Build system**: source and output paths
  ([[file:build.org][Build System]],
  [[file:../context/buildsystem.org][context/buildsystem.org]]).
- **Reader**: path token recognition
  ([[file:../reader.c][reader.c]]).

Implementation Notes

- **Reader tokenisation**: the reader (=reader.c=) identifies path tokens
  by their leading =~/=, =/=, =./=, or =../= prefix and produces
  =AST_PATH= nodes ([[file:../context/reader.org][context/reader.org — path tokens]]).
- **Type system**: =TYPE_PATH= (=types.h:8=) is a primitive ground type
  with runtime tag =RT_PATH=
  ([[file:../context/types.org][context/types.org — TYPE_PATH]]).
- **Codegen**: path literals lower to LLVM global string constants with
  the path text; equality compiles to =memcmp=
  ([[file:../context/codegen.org][context/codegen.org — path codegen]]).
- **No implicit conversion**: paths and strings are separate types;
  casting between them requires explicit conversion.

Key Points

- Path literals start with =~/=, =/=, =./=, or =../= — they have their
  own type (=Path=) separate from =String=.
- Use =~/path= for home-relative, =./path= for cwd-relative,
  =/abs/path= for absolute, =../= for parent directory.
- Paths support ==== and =!= comparison but cannot be used where strings
  are expected without explicit conversion.
- The reader distinguishes path tokens from symbols by the leading path
  prefix characters.
