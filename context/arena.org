[OBS id:obs.context.arena.file src:arena.h conf:high]
  This file records the full arena allocator API: a fast bump-pointer allocator
  organised as a singly-linked list of fixed-size blocks. Allocation is O(1);
  freeing the entire arena is O(blocks). Supports scratch (reset/reuse) and
  permanent (free once) usage patterns plus checkpoint/restore for stack-like
  rollback.

[OBS id:obs.arena.default-block-size src:arena.h:36-38 conf:high]
  ARENA_DEFAULT_BLOCK is 4,194,304 bytes (4 MiB). Can be overridden before
  including arena.h by defining the macro.

[OBS id:obs.arena.alignment src:arena.h:41 conf:high]
  ARENA_ALIGN is 8 bytes, matching max scalar alignment on the target
  architecture. All allocations are rounded up to this alignment.

[OBS id:obs.arena.arena-init src:arena.h:66 conf:high]
  arena_init initialises an Arena struct. If block_size is 0, ARENA_DEFAULT_BLOCK
  (4 MiB) is used. No OS memory is allocated until the first arena_alloc call.

[OBS id:obs.arena.arena-free src:arena.h:70 conf:high]
  arena_free frees ALL memory owned by the arena by walking the block linked list
  and calling free on each block. The Arena struct itself is not freed — callers
  may reuse it with arena_init after freeing.

[OBS id:obs.arena.arena-reset src:arena.h:74 conf:high]
  arena_reset rewinds all blocks to empty (used = 0) but keeps OS memory mapped.
  This is the fast path for scratch arena reuse: zero mallocs in the common case.

[OBS id:obs.arena.arena-alloc src:arena.h:80 conf:high]
  arena_alloc bumps the pointer by `size` bytes (rounded up to ARENA_ALIGN).
  If the current block cannot satisfy the request, a new block is allocated and
  prepended to the list. Never returns NULL — calls abort() on OOM (same contract
  as xmalloc).

[OBS id:obs.arena.arena-calloc src:arena.h:83 conf:high]
  arena_calloc allocates `size` bytes and zeroes them via memset.

[OBS id:obs.arena.arena-memdup src:arena.h:86 conf:high]
  arena_memdup allocates a copy of `src` of `size` bytes.

[OBS id:obs.arena.arena-strdup src:arena.h:89 conf:high]
  arena_strdup duplicates a NUL-terminated string into the arena. Uses strlen +
  memcpy internally.

[OBS id:obs.arena.arena-strndup src:arena.h:92 conf:high]
  arena_strndup duplicates at most `n` bytes of a string. Always NUL-terminates
  the result, so the allocated buffer is at most n+1 bytes.

[OBS id:obs.arena.arena-sprintf src:arena.h:95-96 conf:high]
  arena_sprintf formats a string using printf-style varargs into arena memory.
  Returns a pointer to the NUL-terminated result. The underlying buffer is
  allocated with arena_alloc after determining the required length via vsnprintf.

[OBS id:obs.arena.arena-checkpoint src:arena.h:101 conf:high]
  arena_checkpoint saves the current block pointer and used offset into an
  ArenaCheckpoint struct.

[OBS id:obs.arena.arena-restore src:arena.h:106 conf:high]
  arena_restore rewinds the arena to a previously saved checkpoint: restores
  the active block and its used offset, then frees any blocks that were allocated
  after the checkpoint. Only safe when no blocks were freed between save and
  restore (which is always true for this bump-pointer allocator — free is
  whole-arena only).
