[OBS id:obs.navigation.purpose src:context/navigation.org conf:high]
  This file is a routing table for LLMs. Given a context ID like
  =monadc.context.reader.sugar.percent=, read the prefix to find which file
  contains the node. Read this file first before writing new records.

* ID Prefix → File Map

The second segment of an ID (after =monadc.context.=) maps directly to a topic
file in =context/*.org=:

| ID prefix (monadc.context.<prefix>) | File                   | Contents                |
|-------------------------------------+------------------------+-------------------------|
| =index=                               | [[file:index.org][context/index.org]]      | Schema, format, index   |
| =reader=                              | [[file:reader.org][context/reader.org]]     | Lexer, parser, AST      |
| =wisp=                                | [[file:wisp.org][context/wisp.org]]       | Wisp syntax expander    |
| =language=                            | [[file:language.org][context/language.org]]   | Type system, semantics  |
| =build=                               | [[file:build.org][context/build.org]]      | Build, install          |
| =tests=                               | [[file:tests.org][context/tests.org]]      | Test metadata, coverage |
| =guidelines=                          | [[file:guidelines.org][context/guidelines.org]] | Rules + workflow        |
| =todo=                                | [[file:todo.org][context/todo.org]]       | TODO items              |
| =visualizer=                          | [[file:visualizer.org][context/visualizer.org]] | Context visualizer      |
| =opinions=                            | [[file:opinions.org][context/opinions.org]]   | LLM critique            |
| =philosophy=                          | [[file:philosophy.org][context/philosophy.org]] | Project philosophy      |
| =source-map=                          | [[file:source-map.org][context/source-map.org]] | Source file map         |
| =style=                               | [[file:style.org][context/style.org]]      | MonadC code styles      |

* How to Use

1. To find a node: match the ID's second segment against the table above.
2. To add a new record: use the prefix that matches the topic's file.
3. If no prefix matches, use =context/index.org= and add a new prefix entry here.
4. For new topics: add a row to this table first, then create the file.
