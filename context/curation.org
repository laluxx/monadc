[DOC id:doc.curation.purpose src:context/curation.org conf:high]
  This playbook describes how to prune, promote, split, and refresh context.
  It is intentionally operational: a gardener should be able to run through it
  after a compiler change or after importing a new test corpus.

#+CAPTION: Gardener loop
| Step | Action | Output |
|------+--------+--------|
| Survey | Run linter and inspect inventory | quality snapshot |
| Route | Identify changed categories | touched files list |
| Prune | Remove generated/stale/noisy context | smaller signal |
| Strengthen | Add contracts, sources, tests, links | better graph |
| Validate | Re-run lint and spot-check links | distributable tar |
| Record | Add FIX/THINK/DEC summary | audit trail |

[DEC id:dec.curation.modify-with-audit-trail date:2026-06-16 by:llm]
  Every broad curation pass should leave at least one record saying what changed
  and why. Silent large edits make the garden harder to trust.

#+CAPTION: Prune/keep heuristic
| Content | Keep when | Prune/supersede when |
|---------+-----------+----------------------|
| OBS | still source-grounded | source changed or claim was wrong |
| INF | evidence chain still valid | inputs changed or confidence was inflated |
| THINK | useful judgment remains | duplicated by stronger DEC/FIX |
| TODO | actionable and owned | stale, impossible, or already done |
| File | owns a domain | only duplicates another file |
| Generated table | reproducible and labeled | stale snapshot with no generator |

[DEC id:dec.curation.generated-snapshots-labeled date:2026-06-16 by:llm]
  Generated snapshots must say they are generated snapshots. They should never
  masquerade as hand-curated authority.

#+CAPTION: Split triggers
| Trigger | Suggested action |
|---------+------------------|
| File exceeds ~900 lines and mixes phases | split by compiler phase |
| One file owns unrelated failure types | split or add subcontracts |
| Navigation table needs multiple descriptions for one file | split prefixes |
| Tests link to very broad headings only | add narrower headings |
| Repeated TODOs name the same subtopic | create subtopic file |

[THINK id:think.curation.source-map-demotion conf:medium]
  =source-map.org= may eventually become generated inventory rather than
  hand-written architecture. It currently remains useful as a broad map, but the
  richer per-topic files should be treated as more authoritative.
