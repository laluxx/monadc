The context corpus records stable, source-grounded knowledge about the MonadC
compiler. It is designed for humans and LLM agents. Every durable claim should
point to a source location, a command result, or a clearly marked inference.

#+BEGIN_CENTER
*=Context = Org tree + metadata drawers + fixed LLM records + source links=*
#+END_CENTER

- =context.org= is the root entry point.
- =context/index.org= is the navigable index and formal notation reference.
- Topic files live under =context/= and use kebab-case names.
- Cross-file links use Org file links: =[[file:reader.org::*AST and Token Header][...]]=.
- Stable node references use =ID= properties prefixed with =monadc.context.=.

** LLM Notation
:PROPERTIES:
:ID: monadc.context.format.llm-notation
:CUSTOM_ID: llm-notation
:CONTEXT_KIND: format
:CONTEXT_VERSION: 1
:CONTEXT_CREATED: 2026-06-11
:CONTEXT_UPDATED: 2026-06-11
:SOURCE: context/index.org
:CONFIDENCE: high
:END:

LLM-oriented statements use bracketed records with fixed keys:

#+BEGIN_EXAMPLE
[OBS id:<id> src:<file:line> conf:<high|medium|low>] <direct observation>
[INF id:<id> from:<obs-id[,obs-id...]> conf:<high|medium|low>] <inference>
[DEC id:<id> date:<YYYY-MM-DD> by:<human|codex>] <decision>
[TODO id:<id> owner:<human|codex|unknown> status:open] <task>
#+END_EXAMPLE

Rules:

- =OBS= is reserved for facts read from files or command output.
- =INF= must name the observation IDs it depends on.
- =DEC= records repository-local choices that future work should preserve.
- =TODO= records pending work only when it is specific and actionable.
- IDs are immutable once published.
- Prefer one claim per record.
- Do not silently rewrite history; append a superseding record when facts change.
